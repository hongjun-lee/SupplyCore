📦 物料编码体系设计

阜矿集团物资管理系统 · 基础数据规范 · 2025年12月

一、编码设计原则
  • 唯一性：每种物料有且只有一个编码，一码对应一种物料，杜绝"一物多码""多物一码"
  • 稳定性：编码一旦启用不得修改，停用物料保留编码仅标记状态为"已停用"
  • 可扩展：各分类预留足够顺序号空间，支持新增物料品类而不重构体系
  • 可读性：编码带有明显分类含义，人工核对时可快速判断物料归属大类
  • 统一性：全集团统一编码标准，各矿禁止自行编码，由物资公司统一维护主数据

小类码（2位数字）中类下的三级分类01 = 采煤机截齿
顺序码（4位数字）同小类内唯一流水号0023 = 第23号
规格特征码（2位）区分同品种不同规格A1 = 规格型号A1

五、规格特征码设计

同一品种物资因规格、型号、材质不同而有不同编码，规格特征码是区分关键。

规格特征码含义适用场景
A1~A9型号序列A，规格1~9截齿、钻头等
B1~B9型号序列B，规格1~9不同厂家同类型
S1~S9小规格（Small）螺栓、垫片等
M1~M9中规格（Medium）轴承、链轮等
L1~L9大规格（Large）皮带、圆环链
XL超大规格重型备件
00规格不区分通用型物资
示例说明：

ZH·01·02·0015·M2
→ 支护类 / 锚杆类 / 树脂锚杆 / 第15号 / 中号规格2
→ 树脂锚杆 φ22×2400mm（M2规格）

ZH·01·02·0015·L1
→ 同品种，φ22×2800mm（L1规格）

注：规格不同 = 不同编码，库存分开管理

7.2 禁止行为
  • 禁止各矿、各区队自行创建编码
  • 禁止修改已启用编码的任何位段
  • 禁止将同一物料以不同名称重复申请
  • 禁止停用编码后重新分配给新物料

八、物料编码与各模块的关系
📋 采购计划
申报时必须填入物料编码，杜绝用文字描述带来的歧义；系统自动带出物料名称、单位、参考单价
📦 库存管理
以物料编码为唯一键管理库存，支持条码/二维码扫描入出库，编码 = 物料身份证
💰 财务管理
成本核算按物料编码归集，可精确统计各矿、各区队物资消耗结构
🏭 供应商管理
供应商对应其可供物料编码清单，采购时自动匹配合格供应商
📊 数据分析
按编码统计消耗趋势、价格波动、库存周转率，支撑决策
🔧 检修计划
检修工单关联备件编码，系统自动检查库存是否充足，不足则触发采购
⬆️ 下一步建议：在物料编码体系确定后，进入各模块功能清单设计，或直接做UI原型（库存看板、编码查询界面、出入库单）
