# SupplyCore

## 项目定位

SupplyCore 是面向集团型煤炭企业的**物资供应管理系统需求与方案资料仓库**。

项目核心目标是把物资业务从“分散处理”变成“统一标准、流程闭环、业财协同、集团可视”。

可以将该系统理解为一个**以库存实物流转为主线、以财务接口为落点、以权限审批和报表预警为支撑**的物资业务平台。

当前阶段以**需求梳理、业务沟通、规则统一、招标准备、开发指导**为主，暂不进入代码实现阶段。

## 当前编写目的

本仓库中的内容主要用于：

- 与管理人员、业务人员沟通，统一业务认识
- 明确系统建设范围、功能边界、业务流程与关键规则
- 形成项目招标、需求说明和供应商沟通基础材料
- 为后续产品设计、系统开发、项目实施和验收提供依据

## 当前阶段重点

当前阶段重点不是技术实现，而是先把以下内容说清楚、写明白、定下来：

## 文档分层与优先级

```
集团统筹层（最高优先级）
    ↓ 约束
项目主文档层（需求与总体方案收敛源）
    ↓ 收敛形成
招标正文层（正式对外文本）
    ↓ 引用细化
专项附件层（补充展开，不替代正文）
```

原则上：
- 编制与收敛阶段：按 **集团统筹 > 项目主文档 > 招标正文 > 专项附件** 理解和组织
- 正式发标后：对供应商的外部约束，以正式发出的**招标正文及其配套附件**为准，但不得突破集团统筹层原则

详见 `docs/需求梳理/00-文档分层与引用关系说明-V1.0.md`。

## 建设原则

- 先规范业务，再设计系统
- 先统一规则，再讨论实现
- 先明确范围，再控制扩展
- 先形成可沟通、可招标、可指导开发的文档，再进入编码实施
- 物资系统负责业务与实物流转，财务系统负责价值核算，两者通过接口协同
- 遵循集团统一建设原则，纳入统一权限、统一接口、统一AI入口、统一运维体系
