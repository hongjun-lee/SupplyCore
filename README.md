# SupplyCore

## 项目定位

SupplyCore 是面向集团型煤炭企业的**物资供应管理系统需求与方案资料仓库**。

当前阶段以**需求梳理、业务沟通、规则统一、招标准备、开发指导**为主，暂不进入代码实现阶段。

## 当前编写目的

本仓库中的内容主要用于：

- 与管理人员、业务人员沟通，统一业务认识
- 明确系统建设范围、功能边界、业务流程与关键规则
- 形成项目招标、需求说明和供应商沟通基础材料
- 为后续产品设计、系统开发、项目实施和验收提供依据

## 当前阶段重点

当前阶段重点不是技术实现，而是先把以下内容说清楚、写明白、定下来：

- 项目目标与建设范围
- 当前系统存在问题及内部调研结论
- 核心业务流程与单据体系
- 主数据对象与编码规则
- 与财务系统（NC）的接口边界与管理规则
- 集团统一建设原则与 AI 入口接入要求
- 待确认事项、责任分工与上线前决策清单

## 建设原则

- 先规范业务，再设计系统
- 先统一规则，再讨论实现
- 先明确范围，再控制扩展
- 先形成可沟通、可招标、可指导开发的文档，再进入编码实施
- 物资系统负责业务与实物流转，财务系统负责价值核算，两者通过接口协同
- 遵循集团统一建设原则，纳入统一权限、统一接口、统一AI入口、统一运维体系

## 适用对象
