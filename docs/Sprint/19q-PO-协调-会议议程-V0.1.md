## 一、会议目的

**单一目的**：建立 SupplyCores 业务方反馈协调机制 — 19q-19s 解锁 mock → real 业务价值

**会议成果**：5 业务方对接人花名册填完 + 反馈窗口建立 + 反馈 deadline 商定 + 催办机制确认

---

## 二、参会人

- **cici**（SupplyCores 架构师 / 主持）
- **PO**（产品负责人 / 业务方协调主导）
- **可选邀请**：财务总监（如可能）

**不需要邀请**：业务方对接人本身（PO 后续 1-on-1 落实）

---

## 三、会议议程（60 min 严格控时）

### 3.1 开场 5 min（cici）

- 介绍 SupplyCores 19a-19p 累计交付（19 Sprint / ~85+ PD）
- **关键展示**：admin / 1q2w3E* 登录 + /supplycores/home → React 页面 mount（30 秒 demo）
- 现状陈述："34 页 React 化 / 70% 演示数据 / 5 月业务方协调失败"

### 3.2 现状分析 + 业务影响 15 min

#### 3.2.1 70% Mock 分布（5 min）

cici 用 `[19q-PO-协调-业务价值清单-V0.1.md](./19q-PO-协调-业务价值清单-V0.1.md)` §二 表：
- 财务方 10 页（最高 ROI）
- 设备方 5 页 + 仓储方 7 页（中 ROI）
- SQA + 质保 + 项目 + 全业务方 8 页（低 ROI）

#### 3.2.2 阻塞业务流（10 min）

cici 用业务价值清单 V0.1 §四：
- 🔴 月结 4 核心流程全手工（财务部月底 3-5d 加班痛点）
- 🟡 采购前置链堵塞
- 🟢 仓储 / 设备域部分接通

**PO 反馈**：业务方真正痛点是哪个？

### 3.3 协调机制建立 30 min（核心环节）

#### 3.3.1 业务方对接人花名册（15 min）

cici 与 PO **现场填表**（用 `[19q-PO-协调-反馈跟踪表-V0.1.md](./19q-PO-协调-反馈跟踪表-V0.1.md)` §一）：
- 财务方 / 设备方 / 仓储方 / SQA / 质保 / 项目方 → 各 1 主对接人 + 1 备份
- **必填**：姓名 + 部门 + 联系方式

如 PO 无法当场决定：会议结束 ≤ 1 周内补完（19q D5 前）

#### 3.3.2 反馈窗口 + deadline 商定（10 min）

- 反馈提交方式（飞书 / 邮件 / 紧急电话）
- 反馈 deadline：每个 endpoint 1 周内（保守）
- 优先级排序：财务方 10 页（30 min 第 1 优先）→ 设备 + 仓储 12 页（20 min 第 2）→ SQA + 其他 8 页（10 min 第 3）

#### 3.3.3 催办机制 + 升级路径（5 min）

cici 用反馈模板 V0.1 §四：
- 第 5 天 cici 提醒
- 第 7 天 cici 升级 PO
- 第 10 天 PO 升级分管副总

### 3.4 19q-19s 路线图同步 5 min（cici）

- 19q：协调机制建立 + 3 业务方对接人接洽 + 1-3 endpoint 反馈到位
- 19r：财务方 10 页前 5 页 mock → real
- 19s：财务方 10 页后 5 页 + 设备 / 仓储入手

### 3.5 会议总结 + Action Items 5 min（PO + cici）

PO 现场承诺：
1. 业务方对接人花名册 ≤ 1 周内确认完整
2. 催办机制启用 / cici 每周 1h 同步给 PO 进度

cici 现场承诺：
1. 业务方对接人接洽后 1-3 个 endpoint demo 1 周内交付
2. 反馈到位后 mock → real ≤ 2 周内完成
