# 19q PO 协调 - 业务方反馈模板 V0.1

**项目：** 阜矿物资供应管理系统 / SupplyCores
**版本：** V0.1（cici 与 PO 协调材料 3/5）
**日期：** 2026-05-16
**文档性质：** 实施层 · PO 协调材料 · 业务方反馈标准化模板

---

## 一、为什么需要模板

5 月业务方协调失败核心原因之一：**业务方不知道要反馈什么** — cici 单点询问 / 业务方回复模糊 / 反复来回浪费时间。

**标准化模板降低反馈门槛**：业务方按模板填写 → 主代理 a / 子代理可直接转 DTO 改 / Repository 改 / Controller 真接通。

---

## 二、通用反馈模板（适用所有 endpoint）

业务方填 1 个 endpoint 用此模板（5-10 min/endpoint）：

```markdown
# [业务方] 反馈 — [详设码] [页面名]

**业务方**：（财务方 / 设备方 / 仓储方 / SQA 方 / 质保方）
**对接人**：（姓名 + 部门 + 联系方式）
**反馈日期**：（YYYY-MM-DD）
**详设码**：（如 F-01 / P-02 / S-05）
**页面名**：（如 three-way-match / purchase-orders）

## 一、字段反馈（与 mock DTO 字段对照）

| Mock DTO 字段 | 业务实际字段 | 是否需要 | 备注 |
|---|---|---|---|
| supplierName | 供应商名称 | ✓ 保留 | 长度可能 > 50 |
| amount | 金额 | ✓ 保留 | 精度 2 位小数 |
| createdAt | 创建时间 | ✓ 保留 | - |
| ... | （新字段 X）| ➕ 新增 | 业务必填 |
| oldField | ❌ 不需要 | - | 不再使用 |

## 二、业务规则反馈

1. **触发条件**：（什么情况下走这个流程）
2. **数据校验**：（必填字段 / 格式校验 / 业务约束）
3. **异常分支**：（数据异常时怎么处理 — reject / 重审 / 人工介入）
4. **审批流路由**：（按金额 / 按部门 / 按物料分级审批）

## 三、性能 / 数据规模反馈

- 数据量级：（每日 / 每月 records 数）
- 实时性要求：（实时 / 近实时 / T+1 / 异步）
- 报表口径：（按时段 / 按维度）

## 四、其他

- 与其他系统集成：（NC / 用友 / 银行系统 / 等）
- 既有数据迁移：（是否需要导入历史数据）
- 培训需要：（用户培训 / 文档需求）
```

## 三、3 个示例（cici 带给业务方参考）

### 示例 1：F-01 三单匹配反馈模板填写

```markdown
# 财务方反馈 — F-01 three-way-match 三单匹配

**业务方**：财务方
**对接人**：张总监 / 财务部 / 13800138001
**反馈日期**：2026-05-20

## 一、字段反馈

| Mock 字段 | 业务字段 | 是否需要 | 备注 |
|---|---|---|---|
| poNumber | 采购订单号 | ✓ | 格式 PO-YYYY-NNNNNN |
| grnNumber | 入库单号 | ✓ | 格式 GRN-YYYY-NNNNNN |
| invoiceNumber | 发票号 | ✓ | 财政部规范 |
| poAmount | PO 金额 | ✓ | 含税 / 不含税分开 |
| grnQty | 入库数量 | ✓ | - |
| invoiceQty | 发票数量 | ✓ | - |
| quantityVariance | 数量差异 % | ✓ | 计算字段 |
| priceVariance | 单价差异 % | ✓ | 计算字段 |
| matchStatus | 匹配状态 | ✓ | matched / quantity_exception / price_exception / amount_exception |
| - | matcher（处理人）| ➕ 新增 | 业务必填 / 责任追溯 |

## 二、业务规则

1. **数量公差**：默认 ±10% / 按物料分类可配置（大宗物料 ±5% / 小件物料 ±15%）
2. **单价容差**：默认 ±5% / 按供应商类别可配置
3. **异常分支**：
   - quantity_exception → 仓储复核 → 通过则修订 PO
   - price_exception → 财务复核 → 通过则修订发票
   - amount_exception（数量 + 单价同异）→ 联合复核 → 上级审批
4. **工作流路由**：
   - 金额 ≤ 10W：直接匹配通过
   - 10W-50W：财务主管审批
   - >50W：财务总监 + 总经办审批

## 三、性能

- 数据量：每日 50-200 单 / 每月 1500-6000 单
- 实时性：实时（采购入库录入后立即触发匹配）
- 报表：按 supplier / 按 PO type / 按 period
```

### 示例 2：合同状态机反馈（C-01 contract）

```markdown
**对接人**：李经理 / 财务部合同管理 / 13900139002

## 业务规则

合同状态机：
- 草稿 → 起草人提交 → 审批中
- 审批中 → 各级会签 → 已签订 / 退回
- 已签订 → 履约中 → 履约完成 / 履约异常
- 履约完成 → 归档

会签规则：
- 法务 1 级（必)
- 财务 1 级（金额 >5W）
- 业务部门主管 1 级
- 副总经理 1 级（金额 >100W）
```

### 示例 3：库存出入库反馈（S-08 material-issuance）

```markdown
**对接人**：王主管 / 仓储部 / 13700137003

## 字段反馈

新增字段：
- batchNumber（批次号 / 物料追溯必填）
- issuanceReason（领料原因 / 维修 / 生产 / 报废 / 其他）
- approverChain（会签人链 — 仓库主管 + 财务对账 + 部门负责人三级）

## 业务规则

领料出库高敏感会签：
- 单笔金额 > 1W 触发三级会签
- 单笔金额 < 1W 仓库主管单签
- 紧急领料绿色通道（部门负责人事后补签 24h 内）
```

## 四、反馈提交方式

cici 与 PO 约定（19q D1-3 会议确认）：

- **首选**：飞书文档 / 业务方按模板填写 / cici 接收
- **次选**：邮件 + 附件 markdown / Excel
- **紧急**：电话 + 邮件确认

**反馈 deadline**：业务方收到 endpoint mock 演示 → 1 周内反馈

**催办机制**：cici 与 PO 协议：
- 第 5 天 cici 提醒 1 次
- 第 7 天 cici 升级到 PO
- 第 10 天 PO 升级到分管副总
