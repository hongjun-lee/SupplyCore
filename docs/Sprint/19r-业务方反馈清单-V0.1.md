## 一、财务方李建颖 5 问

### Q1.1 凭证导入格式

> **问**：你们 NC 系统凭证导入的标准模板是什么？Excel？CSV？或者 NC 专属格式？

> **理想答**：拿一份样例文件 — 哪怕是空白模板或一份历史凭证。

> **为什么问**：决定 SupplyCores 凭证生成器的输出格式（Sprint 19r T-B2 用 ClosedXML 还是 MiniExcel / Sprint 19s 是否要支持二进制专属格式）

> **反馈记录**：
> - [ ] 格式：______________
> - [ ] 样例文件：______________
> - [ ] 备注：______________

### Q1.2 必填字段

> **问**：一张凭证里这些字段哪些必填？哪些可选？
>
> 候选字段：
> - 凭证日期 / 凭证号 / 制单人 / 审核人
> - 摘要 / 借方科目 / 贷方科目 / 金额
> - 部门 / 项目 / 成本中心 / 存货 / 客户 / 供应商
> - 附件号 / 附件张数 / 业务类型 / 凭证字号

> **理想答**：直接圈"必填"和"可选"，并指出我们漏掉的字段。

> **为什么问**：F-03 InterfaceReceipt schema + 凭证 Excel 模板列必须对齐财务必填项

> **反馈记录**：
> - [ ] 必填字段（圈出）：______________
> - [ ] 漏掉的字段：______________
> - [ ] 备注：______________

### Q1.3 频次

> **问**：凭证生成频次怎么定？
>
> 候选：
> - **A. 实时**：每笔业务（合同审核 / 入库确认 / 付款审核）触发即生成
> - **B. 日终批量**：每天 18:00 / 24:00 批量生成当天所有凭证
> - **C. 月结批量**：月末统一生成

> **理想答**：选一个，或不同业务类型用不同频次（如 BIZ-001 入库实时 / BIZ-002 暂估月结）

> **为什么问**：决定凭证生成器的触发器实现（事件订阅 vs Hangfire 定时 vs 手动触发按钮）

> **反馈记录**：
> - [ ] 选 A / B / C / 混合：______________
> - [ ] 不同业务的频次差异：______________

### Q1.4 凭证号生成

> **问**：凭证号谁生成？
>
> 候选：
> - **A. NC 自动生成**：SupplyCores 凭证不带凭证号，财务导入时 NC 自动编号
> - **B. SupplyCores 预生成**：凭证号在 SupplyCores 端按规则生成（如 `SC-2026-05-0001`），财务导入沿用
> - **C. 双号制**：SupplyCores 生成业务凭证号 + NC 导入后回写 NC 凭证号（双向留痕）

> **为什么问**：决定 F-03 InterfaceReceipt 字段设计（`pk_voucher` UUID + `num` 业务号双字段 vs 单字段）

> **反馈记录**：
> - [ ] 选 A / B / C：______________
> - [ ] 凭证号规则（如选 B/C）：______________

### Q1.5 失败处理

> **问**：财务导入失败（字段错 / 科目映射错 / 金额错）后怎么处理？
>
> 候选：
> - **A. 财务人工修订**：财务在 NC 端直接改，SupplyCores 不感知
> - **B. SupplyCores 重生成**：财务通知物资侧改源数据，SupplyCores 重生成凭证
> - **C. 财务批注 + 物资重审**：财务批注理由，物资侧重审业务单后 SupplyCores 自动重生成

> **为什么问**：决定 F-03 留痕的 `VoucherDownloadStatus` 状态机（Generated → Downloaded → ImportedToNc → Failed → Regenerated）

> **反馈记录**：
> - [ ] 选 A / B / C：______________
> - [ ] 失败回写需求：______________

## 二、物资方汤云龙 4 问

### Q2.1 NcVoucherNo 字段命名

> **问**：物资侧 8+ 实体（合同 / 入库 / 领料 / 退货 / 调拨 / 付款 / 退料 / 月结）当前都用 `NcVoucherNo` 字段保存 NC 凭证号。这个命名你认可吗？还是要改？

> **候选改名**：
> - `NcVoucherNumber` / `NcVoucherCode`
> - `NcAccountingDocNo` / `NcAcctDocNo`
> - `NcPostingNo` / 其他

> **为什么问**：决定 8+ 实体是否需要 rename migration

> **反馈记录**：
> - [ ] 保持 `NcVoucherNo` / 改为：______________

### Q2.2 触发时机

> **问**：物资侧业务单的凭证生成触发时机？
>
> 候选：
> - **A. 业务单确认即触发**（提交保存）
> - **B. 审核通过后触发**（一级 / 二级 / 三级审核完成）
> - **C. 月结时批量**（月末 cron job）

> **理想答**：按业务类型分（如合同三级审核 / 入库一级审核）

> **反馈记录**：
> - [ ] 触发时机选项：______________

### Q2.3 批量节奏

> **问**：单条 vs 批量生成？
>
> 候选：
> - **A. 单条**：每笔业务一份凭证文件
> - **B. 按时间批量**：每小时 / 每天合并所有同类业务到一份凭证文件
> - **C. 按金额批量**：金额阈值（如 10 万以下合并 / 以上单条）

> **为什么问**：决定凭证生成器是否需要批量合并逻辑 + 财务下载频次

> **反馈记录**：
> - [ ] 选项 + 阈值：______________

### Q2.4 重推规则

> **问**：财务导入失败 / 业务单作废后，凭证如何处置？
>
> 候选：
> - **A. SupplyCores 自动重生成**：业务单变更触发新凭证 + 旧凭证标作废
> - **B. 物资侧人工触发**：业务员在 SupplyCores 后台点"重生成凭证"按钮
> - **C. 不处置**：旧凭证留痕，财务自行处理

> **反馈记录**：
> - [ ] 选 A / B / C：______________
