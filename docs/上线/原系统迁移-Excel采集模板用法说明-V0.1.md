## 二、Excel 模板的三种用法

### 用法 1：目标 schema 参考（最常用）

你写 SQL 时不用再来回查中文清单——直接打开对应 xlsx 的「说明」sheet 和数据 sheet：

- 数据 sheet 第 1 行 = 中文列名（业务可读）
- 列顺序 + 数据类型 + 必填项（标蓝/加粗/下划线 = 必填）= 新系统目标 schema
- 第 2 行起是示例数据，可对照取值范围 / 高敏感属性等

> SQL 里你按英文 canonical 字段名做 `AS` 别名输出（见各对照清单 §五），系统侧导入时会把中文表头自动归一回英文 canonical，**两条路径殊途同归**。

### 用法 2：局部小表 SQL 写不出来时的备选

某些子表（如计量单位字典 ~30 行、供应商资质几条记录），如果在原系统 SQL 跨库 / 跨 schema 难取，可改用 Excel 一次性补给我们——填好对应 sheet 上传即可。**仅限小表**，物料/期初库存这种大表必须 SQL。

### 用法 3：连库前烟雾测试样本（⭐ 推荐）

在 read-only 账号 / VPN / 跳板机还没到位前（迁移方案 §十一 待补充 #2），你已经可以做端到端验证：

1. 在你熟悉的几条数据里**手填 10-30 行**到对应 xlsx（覆盖典型场景 + 1-2 行高敏感物资 / 黑名单供应商 / 报废货位等边界）
2. 把 xlsx 发回项目组，我们走同一条 `POST /api/supply-cores/data-import/upload-and-validate` 入 staging
3. 端到端跑一遍：列映射 / 必填校验 / 双轨编码生成 / M-18 分类映射 / 数据问题台账 / staging 数据调整 UI / 一键回退
4. 暴露的问题立刻闭环（schema 不一致 / 校验过严 / 默认值缺失等）
5. 等真实数据库连通后，切到 SQL 跑全量——管道已被这批样本验证过，全量更稳

**好处**：阻塞项（数据库连接）不再卡住所有人；你和项目组可以并行推进；问题早暴露成本最低。

## 四、烟雾测试样本建议清单

如果走用法 3，建议至少覆盖以下样本量与场景：

| 模板 | 建议行数 | 必含场景 |
|---|---|---|
| 02 物料 | 20-30 | 普通物料 × 高敏感物料（HG/HX）× 设备备件 × 已停用物料 |
| 03 供应商 | 10-15 | 制造商 × 经销商 × 已准入 × 黑名单 × 资质即将到期 |
| 04 仓储 | 10（仓）+ 20（库区/货位）| 主仓 × 临时仓 × 工地仓 × 危险品库区 × 报废货位 |
| 06 期初库存 | 30-50 | 普通 × 有批次（火工品）× 有有效期 × 报废 × 缺单价的 |

烟雾测试样本不必真实，可对原系统几条样例脱敏后填，重在覆盖 schema 边界。

## 五、关键约定（写 SQL 与填 Excel 都适用）

- **OrgCode 范围**：阜新矿业集团全部单位，前缀 `001.007.*`（SQL `WHERE org_code LIKE '001.007.%'`；Excel 同范围）
- **物料编码双轨**：你只输出 `legacy_code`（原物料编码）+ `original_category_code`（原系统旧分类码）；新 `material_code` 由 SupplyCore 按《物资编码规范》V1.8 自动生成
- **分类映射**：业务方审核锁定的《物料分类基线 V0.1》＝ 14 大类 + 95 二级分类；你按《物料分类映射指南 V0.1》做原→新映射回填
- **NC 物料编码**：原系统若已存就抽出；缺失记 NC 映射台账（不阻塞 02 上线）
