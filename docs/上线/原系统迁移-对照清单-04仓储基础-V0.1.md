## 一、本清单是什么

04 仓储基础数据的迁移对照清单 — 跟 02 / 03 同一套模式（3 张表：仓库 / 库区 / 货位 + 管理员关联）。

**完整方案 / 协作模式 / 为什么用 SQL** — 见主方案 [`原系统迁移方案-V0.1.md`](./原系统迁移方案-V0.1.md)。本清单只列 04 的差异点（**§二 仓储 3 个特殊情况请先看**）+ 3 张表的字段。

> 💡 **重要 — 你不需要管的事**（避免过度清洗）：
>
> 阜矿这次迁移是借机做一次**数据治理**，业务部门会在新系统里对仓储数据**重新规划**。所以：
>
> - ❌ 不要纠结**仓库类型分类**（主仓 / 临时仓 / 工地仓 等业务方会重新规划）
> - ❌ 不要纠结 **enable_batch / enable_serial / enable_expiry 开关**（原系统没有就填 false / 业务方按新规范打开）
> - ❌ 不要纠结**库区分类是否合理**（业务方按新业务模式重新划分）
> - ❌ 不要**清理废弃货位**（业务方在 staging 决定）
> - ❌ 不要**修正仓库管理员**（业务方会重新指派）
> - ❌ 货位实在不全也**不要现场补**（让各厂矿仓储实地盘点补）
>
> ✅ **你只需要按原系统的现状原样抽出来**，业务方在新系统的 staging 区会做调整。

## 二、仓储数据的特殊点（请先看这个）

跟 02 / 03 不一样，仓储数据有 **3 个特殊情况**：

### 2.1 货位（StorageLocation）可能要"实地补"

仓库（Warehouse）/ 库区（WarehouseZone）通常原系统都有，可以 SQL 直接抽。

但**货位**（精确到"几排几列几层"的位置）很多原系统没建到这一层 / 或建得不全。处理方式：

- ✅ 原系统**有多少抽多少**（哪怕只是部分仓库的货位）
- ✅ 缺失的部分由各厂矿仓储**实地补**（不由你负责）
- ✅ 完全没有货位概念也 OK，**整张表跳过**，由业务方实地建

### 2.2 仓库管理员是关联人员的（manager_employee_no）

每个仓库要关联一个仓库管理员的**工号**（不是姓名 / 不是 id）。

原系统如果用的是人员表 id 关联：

```sql
JOIN 人员表 e ON e.id = w.manager_id
SELECT e.工号字段 AS manager_employee_no
```

我们会用 01 组织人员清单（Nova 同步）做工号 → 人员对照。

### 2.3 启用批次 / 序列号 / 有效期 是业务规则

`enable_batch` / `enable_serial` / `enable_expiry` 是仓库级别的启用开关，决定该仓库的物料是不是要管理批次/序列号/有效期。

- 原系统如果有这些字段 → 直接抽
- 原系统**没有这些字段** → **全部填 false 即可**，由业务方在新系统里按"是否火工品仓 / 是否设备仓 / 是否化学品仓"补开关

## 四、新表结构清单（04 仓储 / 共 3 张表）

### 4.1 表一：仓库（Warehouse）⭐ 核心表

**用途**：每个仓库的基本信息 + 所属单位 + 管理员 + 业务规则开关。

| 序号 | 新表字段（英文）| 中文含义 | 类型 | 长度 | 必填 | 字段用途备注 |
|---|---|---|---|---|---|---|
| 1 | `warehouse_code` | 新系统仓库编码 | 字符串 | 32 | ✅ | 第 1 阶段 = legacy_code |
| 2 | `legacy_code` | **原系统仓库编码** | 字符串 | 32 | ✅ | 直接抽 |
| 3 | `warehouse_name` | 仓库名称 | 字符串 | 128 | ✅ | 例 `本部综合仓` `恒大主井备件库` |
| 4 | `org_code` | 所属组织编码 | 字符串 | 32 | ✅ | 厂矿级 OrgCode |
| 5 | `warehouse_type` | 仓库类型 | 枚举 | 32 | ✅ | `主仓` / `临时仓` / `工地仓` / `库外保管` / `报废仓` |
| 6 | `address` | 仓库地址 | 字符串 | 256 | ⚠️ |  |
| 7 | `manager_employee_no` | 仓库管理员工号 | 字符串 | 32 | ✅ | 工号（不是 id / 不是姓名）/ 详见 §二.2 |
| 8 | `phone` | 联系电话 | 字符串 | 16 | ⚠️ |  |
| 9 | `active_date` | 启用日期 | 日期 | - | ✅ |  |
| 10 | `is_active` | 是否启用 | 布尔 | - | ⚠️ | 默认 true |
| 11 | `enable_batch` | 是否启用批次 | 布尔 | - | ⚠️ | 详见 §二.3 / 原系统没有就填 false |
| 12 | `enable_serial` | 是否启用序列号 | 布尔 | - | ⚠️ | 同上 |
| 13 | `enable_expiry` | 是否启用有效期 | 布尔 | - | ⚠️ | 同上 |
| 14 | `remark` | 备注 | 字符串 | 256 | ❌ |  |

### 4.2 表二：库区（WarehouseZone）

**用途**：仓库内部的库区划分（如"A 区电气设备" / "C 区火工品"）。

| 序号 | 新表字段（英文）| 中文含义 | 类型 | 必填 | 字段用途备注 |
|---|---|---|---|---|---|
| 1 | `zone_code` | 库区编码 | 字符串 | ✅ | 唯一在仓库范围内 / 例 `Z-A` |
| 2 | `zone_name` | 库区名称 | 字符串 | ✅ | 例 `A 区电气设备` |
| 3 | `warehouse_code` | 所属仓库编码 | 字符串 | ✅ | 引用表一 |
| 4 | `zone_type` | 库区类型 | 枚举 | ✅ | `普通库区` / `危险品库区` / `低温库区` / `隔离区` / `待检区` / 原系统没有的话**默认填 `普通库区`** |
| 5 | `description` | 描述 | 字符串 | ❌ |  |

> **原系统没有"库区"层级**（直接仓库挂物料）→ 整张表跳过 / 我们按"一仓库一库区"自动补 default 库区。

### 4.3 表三：货位（StorageLocation）

**用途**：库区内的精确货位（几排几列几层）。详见 §二.1 关于"实地补"的说明。

| 序号 | 新表字段（英文）| 中文含义 | 类型 | 必填 | 字段用途备注 |
|---|---|---|---|---|---|
| 1 | `location_code` | 货位编码 | 字符串 | ✅ | 唯一 / 例 `L-Z-A-01-01-01` |
| 2 | `location_name` | 货位名称 | 字符串 | ⚠️ |  |
| 3 | `warehouse_code` | 所属仓库 | 字符串 | ✅ |  |
| 4 | `zone_code` | 所属库区 | 字符串 | ✅ |  |
| 5 | `row_no` | 排号 | 整数 | ⚠️ |  |
| 6 | `column_no` | 列号 | 整数 | ⚠️ |  |
| 7 | `level_no` | 层号 | 整数 | ⚠️ |  |
| 8 | `location_status` | 货位状态 | 枚举 | ✅ | `可用` / `冻结` / `维护` / `报废` / **默认填 `可用`** |
| 9 | `freeze_reason` | 冻结原因 | 字符串 | ⚠️ | location_status=冻结 时必填 |

## 五、SQL 输出规范 + 样例

### 5.1 核心约定（同 02）

每个 SELECT 字段都用 `AS` 起新表英文字段为别名。

### 5.2 SQL 样例

**表一：仓库**

```sql
SELECT
  w.原仓库编码字段       AS legacy_code,
  w.原仓库编码字段       AS warehouse_code,
  w.仓库名称字段         AS warehouse_name,
  w.组织字段            AS org_code,
  w.仓库类型字段         AS warehouse_type,
  w.地址字段            AS address,
  e.工号字段            AS manager_employee_no,  -- JOIN 人员表取工号
  w.联系电话字段         AS phone,
  w.启用日期字段         AS active_date,
  CASE WHEN w.启用状态 = '启用值' THEN 1 ELSE 0 END AS is_active,
  COALESCE(w.批次启用字段, 0)    AS enable_batch,
  COALESCE(w.序列号启用字段, 0)  AS enable_serial,
  COALESCE(w.有效期启用字段, 0)  AS enable_expiry,
  w.备注字段            AS remark
FROM 仓库表 w
LEFT JOIN 人员表 e ON e.id = w.管理员id字段
WHERE w.组织字段 LIKE '001.007.%';
```

> 如果原系统没有批次/序列号/有效期启用字段，把那几行 COALESCE 改成 `0 AS enable_batch` 即可。

**表二：库区**（如果原系统有库区概念）

```sql
SELECT
  z.库区编码字段         AS zone_code,
  z.库区名称字段         AS zone_name,
  z.所属仓库编码字段     AS warehouse_code,
  COALESCE(z.库区类型字段, '普通库区') AS zone_type,
  z.描述字段            AS description
FROM 库区表 z
WHERE z.所属仓库编码字段 IN (
  SELECT 原仓库编码字段 FROM 仓库表
  WHERE 组织字段 LIKE '001.007.%'
);
```

**表三：货位**（如果原系统有货位概念）

```sql
SELECT
  l.货位编码字段         AS location_code,
  l.货位名称字段         AS location_name,
  l.所属仓库字段         AS warehouse_code,
  l.所属库区字段         AS zone_code,
  l.排号字段            AS row_no,
  l.列号字段            AS column_no,
  l.层号字段            AS level_no,
  COALESCE(l.状态字段, '可用') AS location_status,
  l.冻结原因字段         AS freeze_reason,
  l.最大承重字段         AS max_weight_kg,
  l.最大容积字段         AS max_volume_m3
FROM 货位表 l
WHERE l.所属仓库字段 IN (
  SELECT 原仓库编码字段 FROM 仓库表
  WHERE 组织字段 LIKE '001.007.%'
);
```

## 七、新系统会做的校验（提前告知）

| 校验项 | 失败处理 |
|---|---|
| warehouse_code / legacy_code 重复 | 记台账 |
| 仓库无管理员（manager_employee_no 空）| 警告 / 业务方补 |
| 管理员工号在人员表查不到 | 警告 / 等 01 组织人员同步完成后回填 |
| 火工品仓 / 易燃易爆仓 未启 batch | 警告 / 业务方在新系统调整 |
| 库区 warehouse_code 与库区所属仓库不一致 | 错误 / 必修 |
| 货位 zone_code 与货位所属库区不一致 | 错误 / 必修 |
| 货位 location_status=冻结 但 freeze_reason 空 | 警告 / 业务方补 |
