## 四、系统侧 endpoint 接通（main 主轨 / Sprint 20l T-A1a 已落地）

**V0.2 修订（cici 复查 P2.5）**：6 类导入统一为单一 endpoint + form `templateType` 路由，
代码侧由 `IDataImportHandler` DI 按 `TemplateType` 路由分发到 6 个 handler。

| 类型 | 调用方式 | 实现 Sprint |
|---|---|---|
| **统一上传 + 校验** | POST /api/supply-cores/data-import/upload-and-validate<br>multipart: `templateType=OrgUser\|Material\|Supplier\|Warehouse\|NcMapping\|InitialStock` + `templateVersion=V0.2` + `file` | 20l T-A1a 框架已落 / handler 增量真实现 20l T-A1b~20m T-A1b |
| **业务方确认 + 导入** | POST /api/supply-cores/data-import/{batchId}/confirm-and-apply | 20l T-A1a |
| **模板下载（业务方）** | GET /api/supply-cores/data-import/templates/{templateType}?templateVersion=V0.2 | 20l T-A1a 已落 / 6 类 .xlsx on-demand 生成 |
| **批次查询** | GET /api/supply-cores/data-import-batch?templateType=&status= | 20k T-A2 |
| **批次下载原文件** | GET /api/supply-cores/data-import-batch/{id}/download（[Authorize]）| 20k Codex P1 立修配套 |
| **批次回滚** | POST /api/supply-cores/data-import-batch/{id}/rollback | 20k T-A2 |
| **共用：数据问题台账** | CRUD /api/supply-cores/data-issue-log | 20k T-A2（已实施 commit `c5893fe`）|
| **共用：数据质量报告** | GET /api/supply-cores/data-quality-report | 20m T-A2（顺延）|

**V0.1 deprecated 口径**：原文档列 6 类型化 endpoint（/data-import/org-user 等）— 实际代码做的是统一入口

## 六、试点单位推荐（V0.2 修订 / cici 反馈 #5 替换 dashboard mock 依据）

### 6.1 5 维度评分依据（替换 V0.1 dashboard HeightPct）

| 维度 | 说明 | 权重 |
|---|---|---|
| ① 数据基础好不好 | 现有物料 / 库存 / 供应商台账完整度 + 数字化程度 | 25% |
| ② 配合度高不高 | 业务方对接积极度 + IT 历史合作 | 20% |
| ③ 业务规模中等不太极端 | 月单量 + 物料种类 + 仓库数（不要最大也不要最小）| 20% |
| ④ 仓储/财务流程是否完整 | 是否有规范流程 + 财务月结配合度 | 20% |
| ⑤ 出问题影响面是否可控 | 影响半径 + 替代方案 + 回退难度 | 15% |
