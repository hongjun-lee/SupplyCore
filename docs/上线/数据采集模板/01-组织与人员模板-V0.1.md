# 01 组织与人员数据采集模板 V0.1（2026-05-17 起草）

**目的**：第 1 批基础数据采集 / 组织架构 + 人员 + 岗位 + 角色 + 权限范围
**责任部门**：物资公司（牵头）+ 各厂矿人事
**填报方式**：Excel 模板（本文档附 sheet 结构）+ 系统导入 endpoint POST /api/supply-cores/data-import/org-user
**数据源优先**：Nova 平台同步 / 本地仅扩展业务字段
**校验**：导入前 + 重复识别 + 编码冲突 + 错误反馈下载

---

## 一、组织架构（Sheet 1: Organization）

### 1.1 字段清单

| # | 字段（中文）| 字段（英文）| 类型 | 长度 | 必填 | 说明 / 校验 |
|---|---|---|---|---|---|---|
| 1 | 组织编码 | org_code | string | 32 | ✅ | 唯一 / 字母数字 / 例 `FK001` |
| 2 | 组织名称 | org_name | string | 128 | ✅ | 例 `阜矿集团本部` |
| 3 | 组织类型 | org_type | enum | 16 | ✅ | 集团 / 厂矿 / 部门 / 班组 / 仓储 |
| 4 | 上级组织编码 | parent_org_code | string | 32 | ⚠️ | 顶层组织留空 / 否则必须在本表存在 |
| 5 | 组织级别 | org_level | int | - | ⚠️ | 1=集团 / 2=厂矿 / 3=部门 / 4=班组 |
| 6 | 是否启用 | is_active | bool | - | ⚠️ | 默认 true |
| 7 | 描述 | description | string | 256 | ❌ | 备注 |

### 1.2 示例数据

| org_code | org_name | org_type | parent_org_code | org_level | is_active |
|---|---|---|---|---|---|
| FK000 | 阜矿集团 | 集团 | | 1 | true |
| FK001 | 本部 | 厂矿 | FK000 | 2 | true |
| FK002 | 艾友矿 | 厂矿 | FK000 | 2 | true |
| FK002-WL | 物流仓库 | 仓储 | FK002 | 3 | true |

### 1.3 校验规则

- `org_code` 唯一性（跨整表）
- `parent_org_code` 必须在本表存在（除顶层）
- `org_type` 必须为枚举值之一
- `org_level` 必须与 `parent_org_code` 对应级别相关（顶层 1 / 下一级 2 / ...）

## 四、导入与错误反馈

### 4.1 系统导入流程

1. cici / 物资公司管理员登录系统 → 数据导入页面
2. 选择类型："组织与人员"
3. 上传 Excel（含 3 sheet：Organization / User / Role）
4. 系统校验（< 10s 完成 / 1000 行内）
5. 显示结果：
   - 成功导入行数
   - 校验失败行数 + 错误明细
   - 重复数据警告
6. 错误明细下载（Excel / 含错误原因列）

### 4.2 错误反馈格式

| 行号 | sheet | 字段 | 错误类型 | 错误描述 | 修复建议 |
|---|---|---|---|---|---|
| 5 | User | employee_no | 重复 | FK0001 已存在 / 行 3 | 检查是否同人 / 修改 employee_no |
| 8 | User | org_code | 不存在 | FK999 不在 Organization sheet | 添加 Organization 行 或 修正 org_code |
| 12 | User | role | 枚举不合 | "operator" 不在枚举 | 改为 admin/物资/财务/仓储/审批/浏览 |

### 4.3 数据问题台账自动记录

- 重复 / 编码冲突 / 必填缺失 / 枚举不合 / 关联不存在 → 自动记录到 `data_issue_log` 表
- 责任部门：导入用户所属组织 + 物资公司
- 状态：待处理 → 处理中 → 已解决 / 已忽略
