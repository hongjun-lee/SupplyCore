# 02 物资主数据采集模板 V0.2（2026-05-18 V1.8 / 原系统迁移口径修订）

**目的**：第 1 批基础数据采集 / 物料分类 + 物料主数据 + 计量单位  
**责任部门**：物资公司 + 各厂矿物资部  
**填报方式**：Excel 模板 + 系统导入 endpoint `POST /api/supply-cores/data-import/material`  
**数据源优先**：现有物资系统迁移 / NC 物料对照

---

## 口径先说明（重要）

- **物料分类遵循集团《物资编码规范》V1.8**：14 个启用大类 + 1 个保留大类 PS；二类合计 95 个。
- **分类编码存储格式**：数据库 / API 使用无横杠格式，如 `ZH01`；页面或 Excel 可展示为 `ZH-01`。
- **新物料编码 `material_code` 由 SupplyCore 自动生成**：数据库 / API 存 `ZH01000001`，页面展示为 `ZH-01-000001`。
- **原系统迁移时不要手填新 `material_code`**：只填 `legacy_code`（原系统物料编码）+ `original_category_code`（原系统旧分类编码），系统通过 M-18 映射到 V1.8 二级分类后生成新编码。
- **【V0.2.3 增强】xlsx 已预填权威基线，无需业务方维护**：
  - 「物料分类」sheet 已预填 V1.8 标准 **15 大类 + 95 二级分类 = 110 行**，含高敏感标识（HG/HX 大类与 YZ01 燃油标黄）和保留状态（PS 大类 5 行灰底 `is_active=false`），来源 [`docs/上线/物料分类映射指南-V0.1.md`](../物料分类映射指南-V0.1.md) §二
  - 「计量单位」sheet 已预填 **25 个常用单位**（长度 4 / 重量 3 / 体积 3 / 数量 15）+ 换算系数
  - 「物料主数据」sheet 的 `unit`（F 列）和 `category_code`（G 列）已添加 **下拉数据验证**，业务方只能从上面 110 / 25 个标准值中选择；如需新增分类或单位，走《物资编码规范文档》§六新增分类提报机制

### 1.1 字段清单

| # | 字段 | 类型 | 长度 | 必填 | 说明 / 校验 |
|---|---|---|---|---|---|
| 1 | 分类编码 | `category_code` | string | 8 | ✅ | V1.8 编码；一级如 `ZH`，二级如 `ZH01` |
| 2 | 分类名称 | `category_name` | string | 64 | ✅ | 中文显示名 |
| 3 | 上级分类编码 | `parent_category_code` | string | 8 | ⚠️ | 一级留空；二级填一级编码，如 `ZH` |
| 4 | 分类级别 | `level` | int | - | ✅ | 1=一级大类 / 2=二级分类 |
| 5 | 是否高敏感 | `is_high_sensitive` | bool | - | ✅ | 火工品 / 易燃易爆等 |
| 6 | 是否启用 | `is_active` | bool | - | ⚠️ | 默认 true |

## 二、物料主数据（Sheet 2: Material）

### 2.1 字段清单

| # | 字段 | 类型 | 长度 | 必填 | 说明 / 校验 |
|---|---|---|---|---|---|
| 1 | 原系统物料编码 | `legacy_code` | string | 64 | ✅ | 迁移主键；直接取原系统物料编码 |
| 2 | 原系统分类编码 | `original_category_code` | string | 64 | ✅ | 旧分类编码；系统通过 M-18 映射到 V1.8 二级分类 |
| 3 | 原系统分类名称 | `original_category` | string | 128 | ⚠️ | 便于人工核对，可空 |
| 4 | 物料名称 | `material_name` | string | 128 | ✅ | 例：树脂锚杆 / 乳化炸药 |
| 5 | 规格型号 | `specification` | string | 256 | ✅ | 例：Φ20×2200 |
| 6 | 计量单位 | `unit` | string | 16 | ✅ | xlsx 已加下拉验证 → 必须从「计量单位」sheet 25 个标准单位中选择 |
| 7 | V1.8 分类编码 | `category_code` | string | 8 | ⚠️ | 可空；xlsx 已加下拉验证 → 若已人工确认请从「物料分类」sheet 95 个二类中选（如 `ZH01`），否则留空由 M-18 映射补 |
| 8 | NC 物料编码 | `nc_material_code` | string | 32 | ⚠️ | 原系统如果存了 NC 编码则直接抽；缺失记 NC 映射台账 |
| 9 | 是否高敏感 | `is_high_sensitive` | bool | - | ⚠️ | 默认从分类继承 |
| 10 | 是否设备备件 | `is_equipment_spare` | bool | - | ⚠️ |  |
| 11 | 是否可采购 | `is_purchasable` | bool | - | ⚠️ | 默认 true |
| 12 | 是否启用 | `is_active` | bool | - | ⚠️ | 默认 true |
| 13 | 安全库存 | `safety_stock` | decimal | - | ❌ | 没有就留空 |
| 14 | 备注 | `remark` | string | 256 | ❌ |  |

> `material_code` 不在填报字段中。系统落库时按 V1.8 自动生成，例如数据库/API 为 `ZH01000001`，页面展示为 `ZH-01-000001`。

### 2.2 示例

| legacy_code | original_category_code | original_category | material_name | specification | unit | category_code | nc_material_code | is_high_sensitive |
|---|---|---|---|---|---|---|---|---|
| 000123 | L-Z-A-01-01-01 | 支护-锚杆 | 树脂锚杆 | Φ20×2200 | 件 |  | NC-BOLT-001 | false |
| 000456 | OLD-HG-001 | 火工品-炸药 | 乳化炸药 | 32mm×200mm | 卷 | HG01 | NC-EXP-001 | true |
| 000789 | OLD-SB-003 | 设备备件-采掘 | 采煤机截齿 | U95 | 件 | SB01 |  | false |

### 2.3 校验规则

- `legacy_code` 必填且唯一。
- `legacy_code + material_name + specification` 用于辅助识别重复物料。
- `unit` 必须在 Unit sheet 中存在。
- `original_category_code` 必须能通过 M-18 映射命中 V1.8 二级分类；若直接填 `category_code`，也必须是二级分类，如 `ZH01`。
- `nc_material_code` 缺失 → 记 NC 映射台账，不阻断导入。
- `is_high_sensitive` 若与分类继承不一致 → 警告 + 记台账。
- 若旧模板误带 `material_code`，系统会忽略该值并重新生成。

## 四、导入与错误反馈

### 4.1 系统导入流程

同 `01-组织与人员` §四 / type=`material`。

### 4.2 关键校验聚焦

- **legacy_code 重复**：高优先级 / 记台账 / 业务方确认是否合并。
- **M-18 映射缺失**：原分类无法映射到 V1.8 二级分类时，该物料行跳过并记台账。
- **NC 物料编码缺失 / 不一致**：必记 NC 映射台账 / cici 与 NC 侧对照。
- **高敏感属性误标**：火工品 / 易燃易爆 / 必双重确认。
- **计量单位错误**：影响库存数量准确性 / 严格校验。

### 4.3 数据问题台账（物资专属）

| 类型 | 描述 | 责任部门 | 严重度 |
|---|---|---|---|
| `legacy_code_duplicate` | 原系统物料编码重复 | 物资公司 | 高 |
| `material_category_legacy_mapping_missing_apply` | M-18 原→新分类映射缺失 | 物资公司 | 高 |
| `material_no_nc_mapping` | NC 编码缺失 | 物资公司 + 财务 | 中 |
| `high_sensitive_mismatch` | 高敏感属性与分类不一致 | 物资公司 | 高 |
| `invalid_unit` | 计量单位错误 | 物资公司 | 高 |
