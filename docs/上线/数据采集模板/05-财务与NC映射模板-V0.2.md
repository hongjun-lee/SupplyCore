## 五、SC/NC 双号制规则（Sheet 5: VoucherNumberRule）

### 5.1 SC 业务号生成规则（已 production）

- 格式：`SC-YYYY-MM-NNNNNN`（年-月-6 位序号）
- 例：`SC-2026-05-000123`
- 生成时机：凭证生成时由 IVoucherNumberGenerator 同步生成
- 月初序号从 1 开始

### 5.2 NC 真号回写规则（G-12 B 后台 endpoint）

- 业务方手动导入 NC 凭证后 → cici 调 PUT /api/supply-cores/interface-receipts/{id}/nc-voucher-number
- 字段：InterfaceReceipt.NcVoucherNumber（与 NcVoucherNo SC- 业务号并存）
- DataGrid 同时展示两列（20b 修复 + 20e frontend dialog）

### 5.3 字段清单（仅参考 / 不需填报）

| 字段 | 含义 | 来源 |
|---|---|---|
| NcVoucherNo | SC- 业务号 | SupplyCores 自动生成 |
| NcVoucherNumber | NC 真号 | 财务回填 / cici PUT endpoint |

## 六、导入与错误反馈

### 6.1 关键校验

- `sc_org_code` 必须在 Organization sheet
- `nc_dept_code` / `nc_supplier_code` / `nc_account_code` 必须与 NC 系统对照清单一致（cici 与 NC 管理方核对）
- `interface_code` 必须为 5 枚举值之一
- `default_debit` / `default_credit` 必须在 AccountMapping sheet
- 缺失 NC 映射 / 严重度极高 / 必须记台账 + 阻止 NC 真接通启动

### 6.2 数据问题台账（NC 映射专属）

| 类型 | 描述 | 责任部门 | 严重度 |
|---|---|---|---|
| nc_dept_mapping_missing | 部门映射缺失 | 财务 + 物资 | 高 |
| nc_supplier_mapping_missing | 供应商映射缺失 | 财务 + 物资 | 高 |
| nc_account_mapping_missing | 科目映射缺失 | 财务 | **极高** |
| nc_account_direction_error | 借贷方向错 | 财务 | **极高** |
| nc_interface_code_no_default_account | InterfaceCode 默认科目缺失 | 财务 | 高 |
