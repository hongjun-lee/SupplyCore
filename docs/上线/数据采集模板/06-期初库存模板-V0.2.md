## 二、校验规则（关键 / 严格）

### 2.1 关联完整性

- `material_code` → Material sheet 必存在
- `warehouse_code` → Warehouse sheet 必存在 + is_active=true
- `zone_code` / `location_code` → WarehouseZone / StorageLocation 必存在 + 关联一致
- `supplier_code` → Supplier sheet 必存在（若填）
- `unit` 必须与 `material_code` 对应 Material.unit 一致

### 2.2 批次/序列号/有效期（按仓库设置）

- 仓库 `enable_batch=true` → `batch_no` 必填
- 仓库 `enable_serial=true` → `serial_no` 必填
- 仓库 `enable_expiry=true` → `expiry_date` 必填

### 2.3 高敏感物资（追加校验）

- 物料 `is_high_sensitive=true`（火工品 / 易燃易爆）→ 必有 batch_no + expiry_date
- 仓库 zone_type 必须是"危险品库区"

### 2.4 财务对账

- `unit_price` 与 `quantity` × `unit_price` = `total_amount` 一致性校验
- 若 `unit_price` 空 → 财务后续按 P（平均成本 / 加权 / 移动平均）计算补全

### 2.5 盘点差异

- `stock_state != 正常` 或 `adjustment_note` 非空 → 记数据问题台账（盘点差异类）/ 责任部门确认

## 三、导入与错误反馈

### 3.1 系统导入流程

1. 各厂矿仓储登录 → 数据导入页面
2. 选择类型："期初库存"
3. 上传 Excel（含 InitialStock sheet）
4. 系统校验（< 30s 完成 / 10000 行内）
5. 显示结果：
   - 成功导入条数
   - 校验失败明细 + 错误下载
   - 盘点差异警告（stock_state != 正常）
6. 财务对账报告（金额合计 / 按仓库 / 按物料分类）

### 3.2 数据问题台账（期初库存专属）

| 类型 | 描述 | 责任部门 | 严重度 |
|---|---|---|---|
| stock_material_not_found | 物料编码不存在 | 各厂矿仓储 + 物资 | 高 |
| stock_warehouse_not_found | 仓库编码不存在 | 各厂矿仓储 | 高 |
| stock_batch_missing | 批次必填缺失 | 各厂矿仓储 | 高 |
| stock_expiry_missing | 有效期必填缺失 | 各厂矿仓储 | 高 |
| stock_high_sensitive_no_batch | 高敏感物资缺批次 | 各厂矿仓储 | **极高** |
| stock_amount_mismatch | 金额与数量×单价不一致 | 财务 | 中 |
| stock_state_abnormal | 盘点状态异常 | 各厂矿仓储 | 中 |
| stock_unit_mismatch | 计量单位与物料不一致 | 物资 | 高 |
| stock_no_supplier | 供应商缺失（追溯困难）| 各厂矿仓储 | 低 |
