# 07 组织架构参考表 V0.2（2026-05-19 起草 / org_code 填报参考）

**目的**：为 02-06 五份数据采集模板提供权威 `org_code` 填报参考，避免业务方乱填或漏填。
**责任部门**：仅作参考查阅，**不需要业务方填报**（组织数据由 Nova 平台统一维护，本系统通过 NovaOrganizationSyncContributor 自动同步；本表只读用于 02-06 模板填报时反查）。
**口径声明**：本表不可作为重新填报组织数据的依据；如发现 Nova 缺失 / 错误，请通过 [01-组织与人员模板-V0.2.md](./01-组织与人员模板-V0.2.md) 的 `C_Nova异常申报` sheet 申报。

## 一、为什么需要这份参考表

### 1.1 02-06 模板的 org_code 关联点

填报 02-06 模板时，以下字段都要引用本表 `org_code`：

| 模板 | Sheet | 字段 | 业务含义 |
|---|---|---|---|
| **02 物资主数据** | 物料主数据 | （主数据 + 数据责任组织字段隐含 sub_group_id）| 物料归属二级集团范围 |
| **03 供应商档案** | 供应商基础 | （供应商归属字段隐含 sub_group_id）| 供应商归属二级集团范围 |
| **04 仓储基础数据** | 仓库 | `组织编码` (org_code) | 仓库所属厂矿 / 子公司 |
| **04 仓储基础数据** | 仓库 | `仓库管理员工号` 关联 User.org_code | 仓管员所在组织 |
| **05 财务与NC映射** | 部门编码映射 | `本系统组织编码` (sc_org_code) | SupplyCores 组织 → NC 部门映射的起点 |
| **06 期初库存** | 期初库存 | （`warehouse_code` 隐含 org_code）| 库存所属厂矿 |

### 1.2 双轨 level 视角说明（重要）

本系统组织数据由 Catio 平台同步而来，存在两套层级体系：

| 视角 | level 1 | level 2 | level 3 | level 4-7 |
|---|---|---|---|---|
| **Catio 物理 level**（Nova 同步原始）| 能源集团根 | 阜新矿业（二级集团）| 24 家厂矿/子公司 | 部门 / 班组 |
| **业务 org_level**（01 模板填报口径）| 集团 | 厂矿 / 子公司 | 部门 | 班组 |

> ⚠️ **业务方填报视角**：在 01-06 模板里，**业务 org_level 是阜矿视角的"集团"=`001.007` 阜矿本部，"厂矿"=`001.007.xxx` 24 家**。本表的"业务 org_level"列即按此口径标注，避免与 Catio 物理 level 混淆。

## 四、典型使用场景

### 4.1 04 仓库 sheet 填报时

业务方填 `仓库编码 / 仓库名称 / 组织编码 / ...`：
- 在 `组织编码` 字段，应填本表中"业务 org_level=2"的 OrgCode（厂矿/子公司层）
- 例：`001.007.002` 恒大煤矿 / `001.007.018` 阜新矿业集团物资有限公司

### 4.2 05 部门编码映射 sheet 填报时

业务方填 `本系统组织编码 / NC 部门编码 / ...`：
- 在 `本系统组织编码` 字段，应填本表中"业务 org_level=2 或 3"的 OrgCode
- 例：`001.007.001` 本部 → NC-DEPT-001（厂矿级映射）
- 例：`001.007.002.001` 恒大煤矿物资部 → NC-DEPT-002-WL（部门级映射，仅需要细粒度对账时）

### 4.3 06 期初库存 sheet 填报时

业务方填 `仓库编码 / ...`：
- 仓库编码本身隐含 OrgCode（由 04 仓库 sheet 已建立的映射查得）
- 不需要在本 sheet 直接填 OrgCode
