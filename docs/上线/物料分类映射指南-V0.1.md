## 一、本文档是什么

阜矿新系统（SupplyCore）按集团《物资编码规范》V1.8 规划了 **14 个一级大类 + 95 个二级分类**。你现在需要做的事：

> **把原系统的物料分类编码 → 映射到新系统的 14 大类 + 95 二类中**，并把结果填到 Excel 给我们。

这一步是 02 物料 SQL 抽数的**前置依赖** — 物料的 `category_code` 必须能映射到新分类，02 物料才能落库。

**我们已经在新系统建好了 M-18 `material_category_legacy_mapping` 表来持久化这份映射**（不是一次性 Excel 消费 / 长期保留追溯）。你输出的 Excel 我们会批量导入这张表。

## 三、映射 Excel 模板（请你填写）

### 3.1 Excel 字段定义（M-18 表设计）

| 列 | 字段名 | 中文 | 类型 | 必填 | 说明 |
|---|---|---|---|---|---|
| A | `legacy_category_code` | 原系统分类编码 | 字符串(32) | ✅ | 你的原系统里的分类编码 / 不能重复 |
| B | `legacy_category_name` | 原系统分类名称 | 字符串(128) | ⚠️ | 便于业务方核对 |
| C | `new_category_code` | 新系统分类编码 | 字符串(4) | ✅ | 从 §二 中选（如 `JD01`）|
| D | `new_category_name` | 新系统分类名称 | 字符串(64) | ⚠️ | 便于核对 |
| E | `mapping_type` | 映射类型 | 枚举 | ✅ | `1:1` / `N:1` / `1:N` |
| F | `remarks` | 备注 | 字符串(512) | ⚠️ | 拆分 / 合并 / 不确定 说明 |

**关键约束**：

- ✅ `legacy_category_code` **全表唯一**（每个原编码只能映射到一个新分类）
- ✅ `new_category_code` 必须在 §二 中存在
- ⚠️ N:1（多原 → 一新）是主流场景 / 业务方借迁移做规范化
- ⚠️ 1:N（一原 → 多新）请只填**最主要**的新分类 / 拆分细节写到 `remarks`

### 3.2 示例

| legacy_category_code | legacy_category_name | new_category_code | new_category_name | mapping_type | remarks |
|---|---|---|---|---|---|
| 01.05.MK | 矿用动力电缆 | JD01 | 电缆 | 1:1 |  |
| 01.05.通信 | 通信电缆 | JD01 | 电缆 | N:1 | 合并到 JD01 电缆 |
| 01.05.控制 | 控制电缆 | JD01 | 电缆 | N:1 | 合并到 JD01 电缆 |
| 02.01 | 炸药 | HG01 | 炸药 | 1:1 |  |
| 03.MK.采煤 | 采煤机配件 | BP01 | 采煤机配件 | 1:1 |  |
| 05.防水 | 防水卷材 | JZ04 | 防水材料 | 1:1 |  |
| 99.其他 | 其他物资 | BG99 | 其他办公用品 | 1:1 | **不确定 / 待业务方分流** |

### 3.3 命名建议

文件命名：`material_category_mapping-fukuang-2026-05-XX.xlsx`
Sheet 名：`Mapping`
第一行：字段英文名（A-F）
数据从第二行开始

## 四、工程师工作步骤（建议）

### Step 1：从原系统抽出全部分类清单

```sql
-- 示例（按你的原系统实际表名调整）
SELECT DISTINCT
  原分类编码字段 AS legacy_category_code,
  原分类名称字段 AS legacy_category_name,
  COUNT(*) AS 物料数量
FROM 原物料表
WHERE 状态字段 IN ('启用', '停用')  -- 含停用以保证历史物料覆盖
GROUP BY 原分类编码字段, 原分类名称字段
ORDER BY 物料数量 DESC;
```

预期输出几十到几百行 — 这就是你要做映射的所有原编码。

### Step 2：对照本指南 §二，逐条填映射 Excel

按 §三 模板填写。对每个原编码，找新系统对应的二类编码。

**对照原则**：
- 优先归到具体业务二类（如原"矿用电缆" → `JD01 电缆`）
- 找不到时归到对应大类的 `99 其他`（如原"特殊设备" → `SB99 其他设备`）
- 完全不在 14 大类内的（如"实验室仪器"），归到 `BG99 其他办公用品`并 remarks 注明

### Step 3：标记不确定项

如果某些原编码**实在拿不准归到哪**：
- 在 `remarks` 列写"不确定 / 待业务方分流"
- `new_category_code` 暂填一个最接近的（可以是 99 其他）
- 这部分我们会单独跟业务方评审

### Step 4：交付

发回给 Spencer / cici，跟 02 物料 SQL 一起或之前交付都行。**但 02 物料 SQL 启动必须在映射 Excel 完成之后**。

## 五、边界情况处理

### 5.1 N:1（多个原编码 → 一个新分类 / 主流）

业务方借迁移做规范化是好事，你直接合并即可：

| legacy | new | mapping_type |
|---|---|---|
| 01.05.MK 矿用电缆 | JD01 | N:1 |
| 01.05.通信 通信电缆 | JD01 | N:1 |
| 01.05.控制 控制电缆 | JD01 | N:1 |

三行映射 / 都指向 JD01。

### 5.2 1:N（一个原编码 → 多个新分类 / 偶尔）

如果原系统的某个分类下混了多种新类的物料（如原"03 设备及备件"既包含 SB 设备整机也包含 BP 备件），**只填最主要的目标**：

| legacy | new | mapping_type | remarks |
|---|---|---|---|
| 03 设备及备件 | BP99 | 1:N | 部分物料应归到 SB 大类下 / 02 SQL 时按物料属性单独识别 / 主代理协助处理 |

我们会在 02 物料 SQL 抽数时，根据物料的具体属性（设备型号 / 备件标识等）做二次细分。

### 5.3 找不到合适归类

如果某些原物料**完全找不到对应新二类**（如原"井下机器人控制器"，新系统 V1.8 没列出）：

- 暂归到对应大类 `99 其他`（如机器人控制器 → `SB99 其他设备`）
- `remarks` 写：建议增设新二类（如 `SB-12 矿用机器人控制系统`）
- 我们汇总 → cici + 物资公司评审 → 决定是否升 V1.9 增设新二类

详细的提报机制见 [`物料分类基线-V0.1.md`](./物料分类基线-V0.1.md) §四 工程师归类提报机制。

## 六、跟 02 物料 SQL 抽数的关系

你做的两件事：
1. **这一步**（本文档）：输出映射 Excel → 我们导入 M-18 表
2. **下一步**（[02 对照清单](./原系统迁移-对照清单-02物资主数据-V0.1.md)）：物料 SQL 抽数 → 输出 `legacy_code` + `original_category_code` + 其他字段

SupplyCore 自动做的事：
- JOIN M-18 表，把 `original_category_code` → `new_category_id`
- 按新分类规则自动生成 `material_code = XX-XX-XXXXXX`（如 `JD-01-000001`）

所以**你不需要在 02 物料 SQL 里做 JOIN**，只需要把映射 Excel 填好，SupplyCore 内部完成关联。
