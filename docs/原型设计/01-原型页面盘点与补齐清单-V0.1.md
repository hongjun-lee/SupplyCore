# 原型页面盘点与补齐清单（V0.1）

**项目：** 阜矿物资供应管理系统 / SupplyCore
**原型版本：** `prototype/README.md` 标注为 v0.9
**盘点日期：** 2026-05-10
**文档性质：** 原型沟通与补齐清单，用于管理沟通、业务确认、招标交流和下一轮原型迭代，不作为开发验收依据。

## 一、盘点结论

当前原型已不是空白线框，已形成 **39 个 HTML 页面**，覆盖工作台、审批、需求计划、采购计划编排、采购任务分解、采购订单、到货验收、质检、库存查询、库存流转、盘点、移动盘点、废旧处置、设备、设备租赁、维修工单、设备 OEE、招投标、招投标归档、合同、付款、资金计划、三单匹配、暂估闭环、主数据、基础档案、供应商履约画像、报表、预警规则、AI 助理、大屏、NC 接口、NC 接口异常详情、系统管理和信创矩阵等内容。

从概要设计的一期 15 个模块看，原型总体已覆盖主要模块，但覆盖深度不一致：

- **覆盖较完整：** 工作台、需求计划、到货验收、质检、库存查询、盘点、合同、付款、资金计划、物料主数据、审批中心、NC 接口、报表/AI、大屏。
- **已补齐关键缺口：** 库存领用/退料/调拨、供应商履约画像、招投标过程归档、预警规则与报表订阅入口、设备 OEE 看板、NC 接口异常详情、三单匹配。
- **本轮已同步：** 2026-05-10 详设同步口径已反映到原型 v0.8：采购计划编排与采购任务分解中间层（需求审批 → 采购计划 → P-05 采购任务 → 路径分流 → 标包 / 招采或采购申请）、暂估 D-90 / D-30 / D-0 / D+30 四阶段（对齐 06 V1.1a §7.6.4）、直达例外 WF-DIR-001、外委检修 WF-RPR-001 / 40% 原值上限 / 1 万 · 10 万分档 / 月度 20 日上报、履约保证金 SENS-CON-003（竞争性方式 + ≥20 万双触发）、外委超限 SENS-CON-004、后评价供应商重评估 WF-SUP-REASSESS-001、低储 / 超储新口径。
- **v0.9 P1 收口：** **合同变更与履约节点深化**（contract-detail 加 C-05 变更记录 + 履约节点表 + ALR-CON-* 履约预警 + 变更冻结控制；contract-list 加变更列；payment-request 加变更冻结提示）；**设备租赁计费深化**（equipment-rent 加租赁详情面板 + 月度费用 E-13 + 付款节点 C-04 + 三笔保证金独立核算 押金/履约保证金§8.6/租赁质保金§8.10 + 续/停/退租演示动作 + 全生命周期 timeline + ONE_TIME 计费案例）。
- **仍建议增强：** 系统管理细化、委托加工台账、直达使用单位专项页（**P0 Q-08 业务方回执后再做**）。

下一步不建议盲目新增大量页面，应围绕“已有页面是否能串成完整业务闭环”继续优化页面之间的来源单据、生成单据、审批状态、NC 触发点和审计留痕。

## 四、核心业务链路检查

### 4.1 需求计划 → 审批 → 采购

**现状：** 已有需求计划列表、需求详情、审批中心、采购计划编排、采购任务分解、采购订单页面，具备串讲基础。
**已同步：** 新增 `purchase-planning.html` 与 `purchase-task-decomposition.html`，用全流程图和 P-05 分解页展示“基层需求审批 → P-02/P-03 采购计划 → P-05 采购任务 → T-01/T-03/T-07 标包 / 招采或 S-01/S-02 采购申请/订单 → 合同 / 到货入库”，明确需求审批通过后先进入计划汇总和任务分解，不是直接招标。

**现状：** 已有采购计划编排、采购任务分解、招投标、招投标过程归档、合同列表、合同详情页面。
**已同步：** 标包来源在 `purchase-planning.html` 中前置展示，P-05 到 T-01/T-03 的分流在 `purchase-task-decomposition.html` 中展示，`tender.html` 与 `tender-archive.html` 继续承接采购文件、评标资料、结果回传和合同联动。

### 4.3 合同 → 到货 → 质检 → 入库

**现状：** 已有合同详情、采购订单、到货验收、质检、库存查询页面，主链路基本完整。

## 六、适合领导汇报展示的页面

建议汇报时优先选择以下页面，避免陷入过多字段细节：

1. `dashboard-bigscreen.html`：展示集团/物资管理驾驶舱效果。
2. `index.html`：展示日常工作台、待办和业务入口。
3. `requirement-list.html` + `requirement-detail.html`：展示需求计划入口。
4. `purchase-planning.html` + `purchase-task-decomposition.html` + `tender.html`：展示需求审批后进入采购计划、采购任务分解、标包编排和招标管控。
5. `contract-list.html` + `contract-detail.html`：展示合同履约。
6. `goods-receipt.html` + `quality-check.html` + `inventory.html`：展示到货、质检、库存闭环。
7. `nc-interface.html`：展示业财协同和接口监控。
8. `ai-assistant.html` + `ai-write-flow.html`：展示 AI 接入边界和用户确认机制。
9. `xinchuang-matrix.html`：展示信创适配和招标响应约束。

## 七、下一步建议

v0.8 已完成 P0/P1 关键缺口和详设同步补齐。下一步建议不继续追求页面数量，而是围绕以下目标深化：

- 在现有页面之间增加更明确的跳转和来源单据关系；
- 补合同变更、设备租赁计费、委托加工台账等少量高价值页面；
- 对采购计划编排、库存流转、供应商履约、招投标归档、预警规则继续细化字段和审批节点；
- 给领导汇报版单独整理一条“推荐演示路径”。

这样原型就能进一步从“业务闭环展示”升级为“业务确认与招标演示包”。
