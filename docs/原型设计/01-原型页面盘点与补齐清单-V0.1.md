# 原型页面盘点与补齐清单（V0.1）

**项目：** 阜矿物资供应管理系统 / SupplyCore  
**原型版本：** `prototype/README.md` 标注为 v0.6  
**盘点日期：** 2026-05-08  
**文档性质：** 原型沟通与补齐清单，用于管理沟通、业务确认、招标交流和下一轮原型迭代，不作为开发验收依据。

---

## 一、盘点结论

当前原型已不是空白线框，已形成 **33 个 HTML 页面**，覆盖工作台、审批、需求计划、采购订单、到货验收、质检、库存查询、库存流转、盘点、移动盘点、废旧处置、设备、设备租赁、维修工单、招投标、招投标归档、合同、付款、资金计划、主数据、基础档案、供应商履约画像、报表、预警规则、AI 助理、大屏、NC 接口、系统管理和信创矩阵等内容。

从概要设计的一期 15 个模块看，原型总体已覆盖主要模块，但覆盖深度不一致：

- **覆盖较完整：** 工作台、需求计划、到货验收、质检、库存查询、盘点、合同、付款、资金计划、物料主数据、审批中心、NC 接口、报表/AI、大屏。
- **已补齐关键缺口：** 库存领用/退料/调拨、供应商履约画像、招投标过程归档、预警规则与报表订阅入口。
- **仍建议增强：** 系统管理细化、设备租赁计费、合同变更与履约节点、接口异常处理详情、设备 OEE 指标看板。

下一步不建议盲目新增大量页面，应围绕“已有页面是否能串成完整业务闭环”继续优化页面之间的来源单据、生成单据、审批状态、NC 触发点和审计留痕。

## 四、核心业务链路检查

### 4.1 需求计划 → 审批 → 采购

**现状：** 已有需求计划列表、需求详情、审批中心、采购订单页面，具备串讲基础。  
**建议：** 在需求详情页或采购订单页增加“由需求计划生成采购任务/采购订单”的来源标识，便于业务人员理解计划如何进入执行。

### 4.2 招投标 → 定标 → 合同

**现状：** 已有招投标、合同列表、合同详情页面。  
**建议：** 强化招投标结果回写合同的连接，例如在 `tender.html` 增加“生成合同/关联合同”按钮或结果区，在合同详情中显示来源标包和成交结果。

### 4.3 合同 → 到货 → 质检 → 入库

**现状：** 已有合同详情、采购订单、到货验收、质检、库存查询页面，主链路基本完整。  
**建议：** 增加入库后库存增加、合同履约进度更新、NC 接口任务生成的链路说明。

### 4.4 库存 → 领用/退料/调拨 → 盘点/废旧

**现状：** 库存查询、库存流转、盘点、移动盘点、废旧处置已覆盖；v0.6 已将领料出库、退料入库、调拨单合并补成 `inventory-flow.html`。  
**建议：** 后续在库存流转页继续细化来源单据、审批节点、库存事务流水和 NC 触发详情。

### 4.5 合同 → 付款 → NC 接口

**现状：** 已有合同、付款申请、资金计划、NC 接口监控页面。  
**建议：** 在付款申请中明确合同、到货/验收/发票、资金计划和 NC 付款接口的关系，避免被理解为单纯线下付款审批。

### 4.6 报表 / 预警 / AI 助理

**现状：** 已有报表中心、报表详情、预警规则配置、AI 助理、AI 写操作链路、大屏，展示效果较完整。  
**建议：** 后续补预警处理详情、异常关闭审批和接口失败重推审批。

## 五、下一轮补齐优先级

### P0：已在 v0.6 补齐，直接影响核心闭环

   - 覆盖供应商准入、黑名单、履约评价、历史交付准时率、验收一次合格率。

3. **招投标过程归档增强**
   - 已新增 `tender-archive.html`。
   - 覆盖采购文件、标包、评标报告、开评标资料、中标通知书、合同生成联动。

   - 已新增 `alert-rules.html`。
   - 覆盖安全库存、合同逾期、接口失败、供应商资质、报表订阅与推送入口。

### P1：下一轮建议增强，提升闭环细节与实施说服力

1. **设备 OEE 指标看板**
   - 单设备运行率、性能率、故障频次、维修成本趋势。

3. **合同变更与履约节点深化**
   - 合同变更申请、履约节点预警、三单匹配、付款限制。

4. **设备租赁计费深化**
   - 起租、停租、退租、费用汇总、付款联动。

### P2：可作为二期或深化设计阶段考虑

1. 电子商务平台深度直连演示。
2. 复杂租赁计费与结算。

## 六、适合领导汇报展示的页面

## 七、下一步建议

v0.6 已完成 P0 关键缺口补齐。下一步建议不继续追求页面数量，而是围绕以下目标深化：

- 在现有页面之间增加更明确的跳转和来源单据关系；
- 补接口异常处理、合同变更、设备 OEE 等少量高价值页面；
- 对库存流转、供应商履约、招投标归档、预警规则继续细化字段和审批节点；
- 给领导汇报版单独整理一条“推荐演示路径”。

这样原型就能进一步从“业务闭环展示”升级为“业务确认与招标演示包”。
