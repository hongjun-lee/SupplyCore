## 一、演示前准备

### 1.1 环境

- 浏览器：Chrome / Edge / Safari 现代版本
- 屏幕：建议 1920×1080；演示前关闭多余标签页
- **演示推荐**：本地 HTTP 服务 — `python3 -m http.server 4175 --bind 127.0.0.1` 后访问 `http://127.0.0.1:4175/prototype/`（避免 file:// 跨标签同步问题）

### 1.2 重置数据（每场演示前）

打开 `prototype/_engine-test.html` → 点 **"重置 (reset)"** 按钮 → 确认 → LocalStorage 全部清空回 seed 状态。

### 1.3 角色切换

页面顶部 header 右侧"当前角色"下拉，**6 个角色 id**（roles.js capabilities）：
- `buyer` 采购员
- `planner` 计划员
- `storage_mgr` 物资主管
- `finance` 财务
- `it` IT
- `group_committee` 集团委员会

切换后页面 reload，菜单可见性 + 数据范围按角色过滤。也可在 `system-admin.html` 的"📡 档 A 引擎接入"卡片中切换并查看每个角色的数据范围 / 操作 / 高敏感约束三段说明。

### 1.4 演示场景切换器

`chrome.js` 顶部场景下拉提供 19+ 个预设演示场景（采购主线 / 出库 / 调拨 / 盘点 / 废旧 / 应急 / 直达 / 委托加工 / 外委检修 / 设备租赁 / 三对一致 / AI Tool / 数据快照 / ...），点选自动跳转该场景的步骤序列。

### 1.5 时间穿越（B1）

页面右下角浮动 widget 可设置 mock 系统时间，用于演示"6 个月后暂估超期"等多日才出现的场景。重置数据后会回到当前真实时间。

## 六、口径补充

### 6.1 演示中常被问的问题

- **"档 A 二阶段是不是招标二期？"** → **不是**。档 A 整体是本次招标承诺范围；"一阶段/二阶段"是档 A 内部开发分批节奏，与招标分期无关。
- **"数据是真的吗？"** → 全部 mock，存 LocalStorage，单机演示用，刷新页面不丢
- **"NC 真的接通了吗？"** → mock-nc.js 模拟，1-2 秒延迟 + 5% 失败率，可调
- **"角色权限是怎么实现的？"** → roles.js 6 角色 + capabilities（含 `canSee(menuId)` / `dataFilter(entity)`）；演示原型**不按角色隐藏菜单**（仅徽标 + 数据范围按角色变化），正式系统按 `canSee` 完整过滤菜单 + 按 sub_group_id 完整过滤数据
- **"为什么菜单不按角色隐藏？"** → 演示场景下若菜单频繁出现/消失会让业务方理解负担过大；roles.js capabilities 配置已就绪，正式系统接入时一行 `canSee()` 调用即可启用过滤
- **"详设里的字段都有吗？"** → 核心字段约 60% 子集（详设字段表完整版以详设 V1.x 为准）


- **"BIZ-014/015/020 真的实现了吗？"** → mock NC 推送 + linkage 自动按节点类型选择凭证；BIZ-014 预付款（节点 1）/ BIZ-013 普通节点（验收/质保）/ BIZ-015 暂估转正式（S-07 冲销）/ BIZ-020 月度消减（手动按钮触发）

### 6.2 重置数据

任何时候演示卡了 → 打开 `_engine-test.html` → "重置 (reset)" → 重新开始。

### 6.3 不在范围内的明确事项

- 真实多用户并发（单机演示）
- 真实 SSO（mock 角色）
- 真实 NC / Nova / 招采平台（全 mock）
- IoT 设备实时数据接入（性能 + 联调由实施期处理）
- 信创栈兼容性联调（招标响应阶段后启动）
