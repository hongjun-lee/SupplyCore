## 一、技术栈与平台对接类

### Q1：是否允许非 .NET 技术栈投标？

**统一口径**：允许。开发语言不做强制限定。

- 若投标方采用 **.NET 技术栈**，可直接引用集团平台模块，降低集成成本
- 若采用**其他技术栈**（如 Java、Python、Go 等），须通过标准 API 完成全部平台对接，实现相同的功能和合规要求

**出处**：正文 10.1 节。

---

### Q2：能否使用国外商业数据库（Oracle、SQL Server 等）？

**统一口径**：不允许。

本项目须采用 **PostgreSQL 兼容的国产信创数据库**，如瀚高 HGDB、人大金仓 KingbaseES 等，**独立部署**，不与集团平台共用数据库实例。

**出处**：正文 10.2 节。

---

### Q3：必须对接集团 Nova Platform 吗？不对接可以吗？

**统一口径**：必须对接。

- **SSO 单点登录**：必须接入，不另行建设独立用户认证体系
- **组织架构/人员基础数据**：从 Nova Platform 通过 API 获取，本系统仅保留本地同步副本
- **数据权限体系**：从集团平台获取，本系统内部自行实现数据范围控制，但须以二级集团为数据隔离边界
- **审批体系**：可自带审批引擎，但组织角色解析须与平台兼容

**出处**：正文 8.3 节、附件三 7 章。

---

### Q4：集团统一 API 网关具体什么时候建？对接要求是什么？

**统一口径**：

- API 网关由招标方**网信办统一建设和运维**，具体对接规范、鉴权协议、流量管控、日志与监控格式以联调阶段提供的技术文档为准
- 投标方**不参与网关选型**，只需按统一规范接入
- 投标方应在实施阶段配合完成接入注册、联调验证

**出处**：正文 16.5 节。

---

### Q5：部署环境是招标方提供还是投标方提供？

**统一口径**：**私有化部署**，部署在招标方指定的数据中心或机房。

硬件环境由招标方提供；投标方负责应用部署、配置、参数优化。

**出处**：正文 10.1 节。

## 三、NC 对接与财务接口类

### Q10：NC 是否需要改造配合？

**统一口径**：**NC 零改造**。

本系统采用 **N:1 映射**（多个物料规格 → 一个 NC 存货），物资系统**主动适配 NC**，NC 侧无需做任何代码或配置改造。

**出处**：附件六 7 章、物资管理与财务接口规范。

---

### Q11：物料和 NC 存货如何映射？一物一存货吗？

**统一口径**：**N:1 映射**，非一对一。

- 物资侧按规格细化（如"锚杆 Φ20×2400mm""锚杆 Φ22×2400mm"分别为独立物料编码）
- NC 侧按科目归集（如"锚杆"挂原材料科目，不区分规格）
- 映射关系：**物资 N → NC 1**
- 同一物料在不同组织可能映射到不同 NC 存货，通过 `material_nc_org` 表存储组织级差异

**出处**：附件六 7.1 / 7.2 节。

---

### Q12：接口是 API + JSON 还是消息中间件？

**统一口径**：

- **对外集成标准统一为 API + JSON**，这是集团硬性要求
- **内部实现**可引入异步消息机制作为实现手段，但**不影响对外集成标准**

**出处**：正文 7.1 节、05 文档 10.1 节。

---

### Q13：接口失败怎么处理？有自动补偿吗？

**统一口径**：四级处理机制：

1. **自动重试**：按指数退避策略自动重试
2. **人工重推**：超过自动重试次数后，由接口管理员人工重推
3. **异常台账**：所有失败记录进入异常台账，可查询、追踪、回溯
4. **告警推送**：失败达阈值时触发告警，推送给运维和业务相关人员

**出处**：正文 7.3(三)、10.9 节、物资管理与财务接口规范 5 章。

---

### Q14：接口幂等键怎么定义？

**统一口径**：

幂等键由"**sourceBillNo + 业务类型 + 组织 + 期间**"等字段组合构成，其中 `sourceBillNo` 是关键幂等字段之一。完整幂等键字段组合详见配套的《物资管理与财务接口规范》。

**出处**：正文 7.3(一)、物资管理与财务接口规范。

---

### Q15：暂估与正式入账差异如何处理？

**统一口径**：

- **月末暂估**：对已入库但未收到发票的单据，月末进行暂估入库
- **次月初冲销**：次月初自动冲销上月暂估
- **差异处理**：以正式发票调整为准

**出处**：正文 6.7 节 / 7.3、04 已确认 9、物资管理与财务接口规范 4 章。

---

### Q16：月结后发现错误怎么改？能直接改吗？

**统一口径**：**严格反结机制**，不允许直接修改。

- 已结账期间**不允许直接修改**
- 须通过**反结申请 → 财务审批 → 反结执行**流程处理
- 全过程留痕，支持事后追溯审计
- 反结权限属高敏感场景，默认不开放给普通业务角色

**出处**：正文 6.7、7.3(四) 节、附件三 6.3 节、04 已确认 10。

---

### Q17：跨组织调拨会触发财务处理吗？

**统一口径**：会。

跨组织调拨触发**财务内部结算接口**，按"送出方/接收方"双向推送 NC，形成内部往来凭证。

**出处**：正文 7.3(二) 节、物资管理与财务接口规范 3.5 节。

## 四、业务范围类

### Q18：招标方是否有电子招采平台，本系统是否要替代它？

**统一口径**：**不替代**，按"**协同而非替代**"原则建设。

- 招采平台由能源集团负责建设和运维
- 本系统一期与招采平台的协同方式为：**采购文件导出/上传 + 结果导入/附件归档**
- 深度直连、自动同步、流程穿透纳入**二期**深化范围

**出处**：正文 3.1(四)、6.1-1 节。

---

### Q19：合同法务功能（模板库、电子签约、条款库）在一期范围内吗？

**统一口径**：**不在**。

一期合同模块覆盖：审批会签、履行跟踪、付款条款定义、到期提醒、合同风险点管理、变更终止留痕、合同台账。

**不在一期范围的能力**：合同模板库、条款库、电子签约、复杂评分模型。不作为本次招标范围和验收依据。

**出处**：正文 3.1(九)、6.6 节。

---

### Q20：设备租赁是不是单独模块？范围是什么？

**统一口径**：**独立模块**，一期覆盖完整闭环。

一期设备租赁模块须覆盖：

- 租赁申请与审批
- 租赁合同与设备档案关联
- 起租登记、在租状态管理
- 使用单位/地点和责任人维护
- 续租停租退租与交接确认
- 租金标准与计费周期维护
- 月度租赁费用汇总
- 与资金计划管理模块的联动（付款）

**出处**：正文 3.1(八)、4.2 模块九、4.4、6.6-2 节。

---

### Q21：设备管理只管理采购吗？资产留痕怎么做？

**统一口径**：

- 设备管理一期**须覆盖完整生命周期**：台账建档、权属分级、变更审批、来源关联、资产标识留痕、状态管理、跨组织调拨、检修轻量闭环、报废处置
- 资产相关能力**不单列独立模块**，通过设备管理中的**关联资产变动留痕**实现
- 完整资产管理（折旧、减值、处置净值核算等）由 NC 财务系统承担，本系统不替代

**出处**：正文 3.1(七)、4.4 节。

---

### Q22：火工品、危险品有什么特殊要求？

**统一口径**：设置**高于普通物资**的审批强度和管控要求：

- **发起环节**须独立授权
- **审批链路**至少包含业务和安全两个维度
- **留痕**必须完整
- **库存、出库、销毁**均须形成专项台账
- **批次管理**：强制启用，录入生产日期 + 批次号，自动计算到期日，到期前 30 天预警，到期冻结
- **双人双锁**：独立台账管理
- **领用实名制**：按需领用

**出处**：附件三 6.1、附件六 6.4、9.3 节。

## 五、编码与主数据类

### Q23：物料编码能否按我们产品默认规则重编？

**统一口径**：**不能**。

投标方应**主动适配**阜矿现有编码规范（XX-XX-XXXXXX 结构、13 大类、一物一码、只增不改、不删不复用、不含业务属性）。

不得以"产品默认编码规则"为由要求重编现有物料编码。

**出处**：附件六。

---

### Q24：辽宁能源集团统一编码还没出台，我们怎么做？

**统一口径**：**预留能力，暂不启用**。

- 一期须预留与辽宁能源集团未来统一编码体系对接的字段（`group_mat_code`）、映射表（`material_group_map`）和接口扩展能力
- 在辽宁能源统一编码标准尚未正式发布前，**不要求一期直接启用**
- 不得以未来集团统一编码为由，要求对阜矿现行编码体系推倒重建

**出处**：正文 5.6、12 章、附件六 11 章。

---

### Q25：组织、人员数据由谁维护？本系统要不要维护？

**统一口径**：**不维护权威数据**。

- 组织架构、人员信息、SSO 身份以**集团统一平台（Nova Platform）**为权威来源
- 本系统通过平台 API 引用，仅保留**本地同步副本**用于业务关联查询
- 人员入职、离职、调岗须通过平台 API 同步更新，权限及时联动

**出处**：正文 5.6、8.3、附件三 7.2 节。

---

### Q26：物料主数据新增流程谁来审批？

**统一口径**：

1. 使用单位提申请（填写物料名称、规格、用途等）
2. 集团物资部审核（查重、归类、确定分类代码）
3. 系统自动生成编码（大类 + 分类 + 流水号）
4. 判断是否有对应 NC 存货：
  - 有 → 填写 `nc_inv_code`，完成映射
  - 无 → 通知 NC 管理员创建存货档案，回填 `nc_inv_code`
5. 编码生效，各单位可用于业务单据

**出处**：附件六 6.5 节。

## 六、权限与审批类

### Q27：角色能不能自定义？我们产品默认就 5 个角色够不够？

**统一口径**：**不够，须支持扩展**。

- 本项目一期至少需要 **11 类角色**（集团领导、网信办、集团物资、各矿物资、仓库、使用单位、采购、质检、财务、安全、系统管理员）
- 每类角色下可能进一步细分（审批人、复核人、接口管理员、审计查询等）
- 投标方应证明系统可承载本附件所列角色、44 项权限条目和 30+ 审批事项，并支持实施阶段进一步细化

**出处**：附件三 3~5 章。

---

### Q28：审批流能不能按金额自动升级？

**统一口径**：**必须支持**。

- 系统须自带审批引擎，支持按**金额、物资类别、组织层级**等维度自动分级升级
- 典型场景：合同会签按金额分级、盘亏处置按金额升级、付款申请超阈值升级管理层
- 审批过程和结果须完整留痕

**出处**：附件三 5 章。

---

### Q29：能否承接已有审批系统的审批流？

**统一口径**：可协调，但**本系统须有独立审批能力**。

- 物资系统自带审批引擎
- 审批人的组织角色解析与 Nova Platform 兼容（可通过平台 API 获取审批人，或按等效规则自行实现）
- 集团如有统一审批平台，投标方应说明如何对接

**出处**：附件三 7.4 节。

## 七、实施与验收类

### Q30：实施周期多长？上线后支持多久？

**统一口径**：

- **深化设计 + 开发 + 实施**：一期周期以投标方响应方案为准，招标方按进度评估
- **试运行**：新旧系统并行，关键业务数据定期比对
- **上线首月**：**现场驻场支持**
- **上线后**：至少提供 **3 个月运行保障期**，含现场 + 远程支持

**出处**：正文 13.4 节。

---

### Q31：培训要求是什么？

**统一口径**：至少两轮培训。

- 覆盖：管理员培训、业务操作培训、财务接口培训
- 包含操作手册/使用指南
- 时间点：**上线前 + 上线后**各一轮

**出处**：正文 13.3 节。

---

### Q32：验收要看什么？能不能只做功能验收？

**统一口径**：不能，须通过综合验收。

验收维度包括：

1. **功能验收**：15 模块核心功能正常、主要流程闭环
2. **接口验收**：29 项接口按附件二要求通过联调
3. **AI 验收**：8 类能力按附件五要求接入并通过 Tool 注册和场景测试
4. **数据验收**：主数据映射完整、期初数据与 NC 核对一致
5. **性能验收**：响应时间、并发能力达标
6. **安全验收**：权限、日志、审计、数据安全达标
7. **文档验收**：需求规格、设计文档、操作手册、运维手册齐全

**出处**：正文 14 章。

---

### Q33：质保期多长？运维响应时间是多少？

**统一口径**：

**响应时间分级**：


| 故障等级        | 首次响应   | 恢复/解决  |
| ----------- | ------ | ------ |
| 紧急（系统不可用）   | 30 分钟  | 4 小时   |
| 重要（核心功能异常）  | 1 小时   | 1 个工作日 |
| 一般（非核心功能异常） | 4 小时   | 2 个工作日 |
| 建议（优化类）     | 1 个工作日 | 协商确定   |


**质保期**：投标方应在响应文件中明确质保期时长（一般不少于 1 年）。

**出处**：正文 15 章。

## 九、答疑管理说明

### 答疑接收方式

1. **书面答疑**：发送至网信办招标联系邮箱（具体联系方式见招标公告）
2. **答疑会**：发标后第 **7~10 个工作日**召开集中答疑会
3. **现场咨询**：答疑会后不再受理口头咨询，如有新疑问须书面提交

### 答疑响应时限

- 收到书面疑问后 **3 个工作日内**统一回复
- 重要事项以**书面澄清文件**形式统一发给所有投标方，不单独回复
- 凡涉及变更招标技术要求的事项，以正式**补遗文件**发布，不以口头回复为准

### 答疑原则

1. **统一口径**：所有回复以本文档和配套澄清文件为准
2. **公平公正**：同一疑问的回复对所有投标方一致公开
3. **记录留痕**：所有书面疑问和回复均归档留存
