# 发标准备-下一步工作清单（V1.0）

## 一、当前判断

当前阶段不宜继续扩需求，工作重点应由“补内容”转为“收口、核对、发标准备”。

根据现有文档情况，P0/P1 关键事项已基本明确，下一步重点是保持口径一致、形成完整发标包，并提前准备供应商答疑口径。

## 二、建议工作顺序

### 第一步：封版核对（优先级最高）

目标：确保“需求梳理、招标正文、附件、报审对照表”四套材料口径一致。

重点核对：
- 一期建设边界是否一致
- 模块数量与名称是否一致
- 附件编号与引用是否一致
- NC 接口、统一平台、AI 接入、权限体系、设备租赁等重点专题表述是否一致
- 二期建设建议与一期正式要求是否冲突

输出物：
- 封版检查清单
- 口径差异修订记录

### 第二步：形成发标包最终目录

目标：形成可直接用于发标或送审的目录结构。

建议目录：
1. 招标技术要求正文
2. 附件一 功能响应清单
3. 附件二 接口清单及报文示例
4. 附件三 角色权限矩阵
5. 附件四 供应商响应模板
6. 附件五 AI接入能力要求及验收条款
7. 附件六 物料编码规范

补充建议：
- 增加《发标包附件说明》1 页，说明各附件用途
- 明确哪些文档对外发放，哪些仅内部保留

### 第三步：准备供应商答疑口径

目标：提前把最容易被问到的问题统一口径，减少后续反复解释。

建议优先准备的问题：
- 与集团统一平台的对接边界是什么
- 与 NC 的接口方式、职责分工和异常处理机制是什么
- AI 接入是必做项还是加分项
- 设备管理与设备租赁一期做到什么深度
- 物料编码是否要求一期直接执行集团统一编码
- 招投标管理与集团招采平台如何分工

输出物：
- 供应商答疑口径表
- 招标澄清问题模板

### 第四步：准备管理层汇报版材料

目标：服务领导审阅、项目立项推进和招标沟通。

建议汇报内容：
- 项目建设目标
- 一期建设范围
- 与集团统一平台的关系
- 与财务 NC 的关系
- 预期管理价值与风险控制价值
- 实施路径与阶段计划

输出物：
- 5 至 8 页汇报版 PPT 提纲或文字稿

## 三、我建议马上做的事

先做“封版核对 + 发标包最终目录”，因为这两步完成后：
- 文档体系就能稳定下来
- 后续答疑才不会反复改口
- 管理层汇报也可以直接复用

## 四、一句话判断

**现在最重要的不是再补需求，而是把现有成果收成一个可以发、可以讲、可以答疑的完整包。**
