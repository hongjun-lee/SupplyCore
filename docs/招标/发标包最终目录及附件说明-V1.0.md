## 三、各外发附件使用说明

### 1. 正文

正文负责回答以下问题：
- 项目为什么建设
- 一期范围是什么
- 供应商需要建设哪些模块
- 技术、接口、实施、验收总体要求是什么

正文是发标包的主体文件，附件均围绕正文展开，不单独替代正文。

### 2. 附件一：功能响应清单

主要用于：
- 要求供应商按功能项逐条响应
- 支撑评标时开展功能符合性比对
- 减少供应商只写“满足”的空泛响应

### 3. 附件二：接口清单及报文示例

主要用于：
- 约束供应商明确接口建设范围
- 明确 API + JSON 的集团统一接口原则
- 提前暴露集成复杂度，避免后续只口头承诺“可对接”

### 4. 附件三：角色权限矩阵

主要用于：
- 明确岗位、角色、审批、数据权限的控制边界
- 支撑供应商做权限设计和工作量评估
- 降低后续因权限理解不一致带来的返工

### 5. 附件四：供应商响应模板

主要用于：
- 统一投标方技术响应结构
- 便于横向比较不同供应商方案
- 引导供应商重点回答系统方案、接口、AI接入、设备租赁、NC协同等重点内容

### 6. 附件五：AI接入能力要求及验收条款

主要用于：
- 明确 AI 接入为正式要求，不是可选加分项
- 强调统一AI入口、统一权限、统一日志审计要求
- 作为后续AI联调和验收的专项依据

### 7. 附件六：物料编码规范

主要用于：
- 明确编码规范来源和边界
- 约束投标方在主数据设计时遵循现有编码规则
- 为后续辽宁能源统一编码扩展预留接口和字段基础

## 六、版本与发放控制建议

### 1. 版本控制

- 本轮发标建议统一采用 `V1.0`
- 正文与附件版本号应保持一致
- 如发标后有澄清修改，应通过“澄清文件/补遗文件”管理，不建议直接覆盖原发标版本

### 2. 发放控制

建议明确以下信息：
- 发放日期
- 发放对象
- 发放渠道
- 是否盖章
- 是否以 PDF 为准
- 是否允许供应商就附件二、附件五提出专项答疑

### 3. 内外口径控制

外发材料应以正文和 6 个附件为准；内部底稿和讨论稿不得与外发版本长期并行且口径不一致。

## 七、一句话结论

**当前发标包建议收敛为“1个正文 + 6个附件”，其余材料作为内部底稿和发标准备支撑文件留存。**
