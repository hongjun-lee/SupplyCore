## 第 1 页 项目背景与建设目标

### 1.1 项目背景

阜矿集团作为辽宁能源集团下属二级集团，下辖多个煤矿、洗煤厂及辅助单位，物资品类涵盖火工品、支护材料、备品备件、机电材料、油脂燃料等十余个大类。当前物资供应业务存在以下问题：

- **系统分散**：物资相关管理分布在若干独立工具中，主数据口径不一，业务数据难以串联
- **线下流转多**：审批与业务流转线上化程度不足，效率有待提升
- **与财务协同不紧**：与 NC 财务系统缺乏稳定、实时、可追溯的接口
- **集团统筹不足**：信创适配、统一集成、AI 赋能等集团要求难以落地

### 1.2 建设目标

结合集团管理要求和《集团业务系统统一建设原则（V2.0）》，本次建设的核心诉求为：

1. **无纸化审批**：全面线上审批、审批留痕、催办提醒
2. **多端协同**：PC + 手机 APP/小程序 + PAD 审批
3. **流程优化**：需求→计划→招投标→合同→入库→资金计划→财务全链路贯通
4. **数据赋能**：库存、采购、消耗、盘点、呆滞、设备状态、租赁执行、资产关联留痕等核心视图
5. **基层延伸**：使用单位、仓库、区队均能直接在系统中操作
6. **集团合规**：统一权限、统一数据、统一接口、统一 AI 入口、统一运维

### 1.3 项目定位

> **以物资流转为主线、强化设备管理并保留资产关联能力、以财务接口为落点、以权限审批和报表预警为支撑的业务平台。**

## 第 2 页 一期建设范围与业务主链路

### 2.1 一期 15 大模块

一期建设按"先跑通核心闭环、强化设备管理、设备租赁独立纳入并保留资产关联能力"原则，覆盖如下 15 大模块：


| 序号    | 模块        | 核心内容                               |
| ----- | --------- | ---------------------------------- |
| 一     | 基础档案与组织仓库 | 组织、仓库、单位、供应商、成本中心等基础数据             |
| 二     | 物料主数据与编码  | 编码结构、一物一码、NC 映射、辽宁能源扩展预留           |
| 三     | 需求提报与采购计划 | 需求汇总→计划审批→任务下达→执行跟踪                |
| **四** | **招投标管理** | **采购方式分流、招采平台协同、结果归档和回写**（一期核心补充项） |
| 五     | 采购执行与到货   | 采购订单、到货验收、质检、入库                    |
| 六     | 库存领用与退料调拨 | 领料、退料、调拨、成本归集                      |
| 七     | 设备管理      | 台账、权属分级、跨组织调拨、检修、报废、资产留痕           |
| 八     | 设备租赁管理    | 申请、起租、在租、续租停租退租、费用汇总、付款联动          |
| 九     | 合同管理      | 审批会签、付款条款、风险点、变更、台账                |
| 十     | 资金计划管理    | 付款计划、月度预支付、付款申请、支付进度、结果回写          |
| 十一    | 供应商管理     | 准入、分类、考评、黑名单                       |
| 十二    | 盘点与处置     | 盘盈盘亏、废旧认定、处置流程                     |
| 十三    | 财务与 NC 接口 | 凭证推送、对账、暂估、月结反结                    |
| 十四    | 报表预警与查询分析 | 多维度报表、预警识别、AI 问答                   |
| 十五    | 系统管理与基础支撑 | 权限、审批流、日志、参数、通知、安全、导出              |


### 2.2 一期业务主链路

```
需求提报 → 采购计划 → 招投标/询比价 → 合同/采购执行 → 到货入库
    → 设备建档与投用/设备租赁执行 → 资金计划管理 → 生命周期管理 → 财务联动
```

### 2.3 边界约束

**本系统不替代**：

- 用友 NC 财务总账（本系统仅作为业务源头，NC 负责价值核算）
- 能源集团招采平台（一期仅以导出上传/结果导入协同为准，深度直连属二期）
- 集团 BI 平台（本系统提供数据，集团级汇总分析由 BI 承担）

**一期不纳入但预留扩展**：

- 集团统一编码体系（预留 `group_mat_code` 字段和映射表）
- 高级 AI 场景（一期以 8 类基础能力为主）

**一期接入但规范中标后提供**：

- 集团统一 API 网关（由网信办统筹建设，联调阶段按网信办提供的技术文档完成接入注册与验证）
- 集团统一 AI 入口（由网信办统筹建设，联调阶段按网信办提供的技术文档完成 Tool 注册与场景测试）

### 3.2 信创与国产化合规

- **数据库**：须采用 PostgreSQL 兼容的国产信创数据库（瀚高 HGDB、人大金仓 KingbaseES 等），独立部署，不与集团平台共用实例
- **服务器**：国产服务器优先
- **部署方式**：私有化部署，部署在招标方指定的数据中心
- **开发语言**：不做强制限定；.NET 可直接复用集团平台模块；其他技术栈须通过标准 API 完成全部对接

### 3.3 权威数据源与扩展预留（V2.0 新增对齐）


| 数据对象            | 权威源                   | 本系统定位                             |
| --------------- | --------------------- | --------------------------------- |
| 组织架构、人员、SSO 身份  | 集团统一平台（Nova Platform） | 通过平台 API 引用，不重复维护                 |
| 会计科目、会计期间       | NC                    | 通过接口获取，不在本系统自建科目                  |
| 物料编码            | 本系统                   | 作为权威源，预留对齐集团未来统一编码的字段、映射关系和接口扩展能力 |

### 3.4 与 NC 的协同边界（N:1 映射）


| 维度   | 物资系统（物料）  | 用友 NC（存货） |
| ---- | --------- | --------- |
| 管理对象 | 实物，按规格区分  | 资金，按科目归集  |
| 粒度   | 细（一规格一编码） | 粗（同类合并核算） |

| AI 入口建设进度       | 网信办 AI 入口尚未完全就绪 | 附件五明确联调节点和验收条款，允许分阶段接入        |

| 辽宁能源统一编码未发布     | 编码规范扩展预留可能需后续调整 | 已预留字段和映射表能力，对接时按需启用           |
| 供应商对 N:1 映射理解偏差 | 可能影响 NC 对接工作量评估 | 附件六第 7 章详细说明映射逻辑和 SQL 示意      |

**结论建议：**

1. **发标包完成封版核对，质量可控、合规到位、边界清晰，建议签批放行发标**
2. **建议按本汇报所列时间表推进，从签批到中标通知总周期约 35~40 个工作日**
3. **签批事项请集团分管领导在本汇报基础上做二次确认，网信办将据此形成正式发标包**
