## 一、附件说明

本附件明确物资供应管理系统一期的角色定义、权限矩阵、审批矩阵、高敏感场景控制要求，以及与集团统一平台（Nova Platform）的身份、组织和审批对接要求，作为投标方进行权限模型设计、流程引擎选型、工作量评估和上线后配置的统一依据。

本附件所列矩阵为一期建设的最小要求，实施阶段可根据管理需要进一步细化；投标方须证明系统具备承载以下角色、权限和审批结构的能力，不得以"本产品仅支持固定角色模型"为由偏离本附件要求。

## 二、权限设计基本原则

一期权限与审批设计须遵循以下原则：

1. **职责分离原则**：发起、审核、执行、复核、财务处理等关键环节不得由同一人包办，防止内控风险。
2. **分级授权原则**：集团、矿厂、仓库、使用单位按照组织层级和职责范围授予不同权限，不允许"一把钥匙开所有门"。
3. **数据归属原则**：各角色默认只能查看其职责范围内的数据；集团管理角色可按授权查看汇总数据。
4. **特殊业务加强控制原则**：火工品、危险品、盘亏、废旧处置、跨组织调拨、月结反结等场景应设置更严格的审批门槛。
5. **留痕审计原则**：所有关键操作、审批、驳回、作废、重推、反结等动作均须保留完整日志，支持事后追溯。

| 11  | 系统管理员       | 账号、角色、参数、流程、日志等系统级管理                     | 用户与角色管理；参数配置；流程配置；日志与状态监控；不直接代替业务人员处理业务事实         |

## 六、高敏感场景控制要求

### 6.1 火工品、危险品

须设置高于普通物资的审批强度：

- 发起环节须独立授权，非授权用户不得发起
- 审批链路至少包含业务和安全两个维度
- 必须完整留痕
- 库存、出库、销毁均须形成专项台账，支持独立查询和审计

### 6.2 盘亏、报废、销毁

按金额和物资类别分级审批：

- 小额普通物资可由单位内审批
- 金额较大或特殊物资应升级至集团或相关分管领导
- 财务须参与盘亏与处置类关键审批

### 6.3 月结、反结、补录

须作为高敏感权限场景控制：

- 不得默认开放给普通业务角色
- 必须具备申请、审批和完整日志
- 操作后须保留前后状态和原因说明，支持审计回放

## 七、与集团统一平台（Nova Platform）的对接要求

本系统权限与审批设计须与集团统一平台对接，具体要求如下：

### 7.1 统一登录与身份认证

- 须对接集团统一平台 SSO（单点登录）服务，不另行建设独立用户认证体系
- 用户基础身份信息通过平台 API 获取，本系统可保留本地同步副本，权威源在平台侧
- 人员入职、离职、调岗须通过平台 API 同步更新，系统权限及时联动
- 离岗、离职、停用人员须具备及时禁用和权限收回机制

### 7.2 组织与人员数据

- 组织架构和人员信息通过集团统一平台 API 获取并本地同步
- 物资系统拥有独立数据库，组织和人员数据以本地副本形式存储，用于业务关联查询
- 权威来源为集团统一平台，物资系统不作为组织和人员的权威维护方

### 7.3 数据权限

- 物资系统内部自行实现数据范围控制（按组织、仓库、角色划分）
- 数据隔离粒度须与集团平台一致，以二级集团为数据隔离边界
- 物资系统业务角色和业务权限由本系统自行维护，但组织层级结构须与平台保持同步

### 7.4 审批流程

- 物资系统须自带审批引擎，能够承载本附件第五章的审批矩阵
- 审批人的组织角色解析须与集团平台体系兼容：可通过平台 API 获取审批人信息，或按等效规则（角色 + 组织过滤 + 层级递进）自行实现
- 审批过程和结果须完整留痕
