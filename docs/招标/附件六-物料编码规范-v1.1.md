## 一、附件说明

本附件明确物资供应管理系统一期的物料编码规则、分类体系、条码/二维码规范、与用友 NC 的对接映射要求，以及与辽宁能源集团未来统一编码体系对接的扩展预留要求，作为投标方进行主数据设计、NC 对接实施和后续编码扩展对齐的统一依据。

本附件所列编码体系为阜矿集团一期建设的既定规范；投标方不得以"产品默认编码规则"为由要求重编现有物料编码，应主动适配本附件所列编码结构、映射关系和扩展字段。

## 二、编码目标

- **唯一性**：一物一码，全集团唯一，不因组织不同而重复编码
- **可读性**：大类采用 2 位拼音首字母，简洁且基层人员易识别
- **可扩展**：预留充足号段，不因业务增长而需要重构
- **可对接**：支持与用友 NC 存货编码 N:1 映射，物资系统主动适配 NC，NC 侧零改造

## 三、编码结构

### 3.1 编码格式

```
XX - XX - XXXXXX
 ↑    ↑     ↑
大类  分类   流水号
(2位) (2位)  (6位)

示例：ZH-01-000001（支护材料-锚杆-第1号）
```

### 3.2 各段说明


| 段    | 位数           | 说明                  |
| ---- | ------------ | ------------------- |
| 大类代码 | 2 位拼音首字母     | 物料一级分类，见第四章         |
| 分类代码 | 2 位数字（01~99） | 物料二级分类，见第五章         |
| 流水号  | 6 位数字        | 按分类独立自增，从 000001 开始 |


**总长度：** 10 位有效字符，含分隔符"-"共 12 位。

### 3.3 设计原则

- **编码是给系统用的，名称才是给人看的**——大类拼音仅辅助粗识别，业务查找以名称/规格/属性为主
- **编码不含组织信息**——库存归属通过"组织/仓库"字段区分
- **两级分类足够**——更细的区分（规格、材质等）通过物料属性字段实现

### 5.16 归类裁决规则

现行大类体系中存在若干跨类物料（如支护用钢、含化学添加剂的液压介质、防爆成套电气设备等），为避免同一物料在不同单位、不同时间被归入不同大类，本节规定统一的归类裁决优先级。基层申报、物资部审核、系统自动推荐均须遵循本规则。

**优先级顺序（自上而下，命中即止）：**

**① 专业管控优先**

属于下列专业管控范围的物料，一律进入对应大类，不按物料形态拆分：

- 火工品管控范围（炸药、雷管、起爆器材及配套）→ HG
- 劳动防护装备管控范围（安全帽、矿灯、自救器、防护服装、防护手套、安全靴、呼吸防护等）→ LB
- 通风防灭火专业管控的材料（风筒、抽采管路、密闭材料、防灭火材料）→ TF，不因其金属/非金属形态转入 GC/HX
- 排水专业管控的材料（启用 PS 后）→ PS，不因其金属形态转入 GC

**② 终端用途优先于物料形态**

物料采购时已有明确终端用途的，按用途归类，不按原材料形态归入 GC/HX：

- 锚杆用螺纹钢、锚杆托盘、锚索、钢带/托梁 → ZH（支护），不进 GC
- 井下型钢支架、坑木 → ZH，不进 GC 或木材类
- 建筑用钢筋、箍筋、预埋件 → JZ（建材），不进 GC
- 明确用于某台整机配套的专用件 → BP（备件），不进 GC/JD

**③ 主要功能优先于附加属性**

液态、化学品类物料，按"主要功能"而非"成分"归类：

- 液压介质（乳化液、液压油，不论是否含化学添加剂）→ YZ
- 润滑介质（润滑油、润滑脂、导热油）→ YZ
- 以化学反应或化学作用为主要用途的产品（浮选药剂、絮凝剂、注浆材料、阻燃剂、阻化剂）→ HX
- 清洗剂、除锈剂：以物理清洁为主的 → BG/GJ；含化学反应的 → HX

**④ 整机优先于部件**

同一台设备可能既对应 SB（整机）也对应 BP（配件），按采购形态裁决：

- 采购时为"成套设备"、有独立铭牌、按台计量 → SB（含防爆开关柜、变压器、成套水泵机组等）
- 采购时为"备件/元器件"、需装配进既有设备 → BP 或 JD
- 采购时为通用机电元器件（接触器、继电器、按钮、指示灯）→ JD

**⑤ 物料形态兜底**

前四条均不适用时，按物料形态归类：

- 呈现为钢材初级形态（板/管/线/棒/型），无明确终端用途 → GC
- 呈现为液态流体，以流体功能为主 → YZ 或 HX（按主要功能）
- 可重复使用的工具、量具、便携检测仪 → GJ
- 一次性消耗的辅助材料（标签、包装袋等）→ BZ

**裁决争议处理：**

- 规则无法覆盖的新物料，由集团物资部组织归类评审，评审结论纳入本附件案例库
- 同一物料在不同组织、不同时间必须保持同一大类，不得因申报单位不同而分歧
- 历史误归类物料的纠正须通过"编码废弃 + 新编码"流程，不得直接改动已生效编码的大类字段（遵循 §6.1 "只增不改"原则）

**投标方须在系统中提供：**

- 新增物料时按本节规则自动推荐大类
- 跨类申报时弹出规则提示（如用户选了 GC 但名称含"锚杆"，提示按规则应归 ZH）
- 归类评审流程和案例库维护入口


## 六、编码规则

### 6.1 基本原则

1. **一物一码**：同一物料不论采购来源、所属矿厂、存放仓库，全集团唯一编码
2. **只增不改**：编码一经生效不得修改，停用物料标记"停用"状态
3. **不删不复用**：已用编码永久保留，停用号不再分配给新物料
4. **编码不含业务属性**：规格、供应商、组织等信息存入对应字段，不嵌入编码

### 6.2 组织与库存管理

物料编码不区分组织，库存归属通过记录字段区分。示例：

```
物料编码：ZH-01-000001（左旋螺纹钢锚杆）

一号矿主仓库：库存 5,000 根
二号矿井下仓：库存 3,000 根
洗煤厂仓库：  库存 0
```

### 6.3 供应商物料号处理

供应商各自的货号存入**"供应商物料号"**字段，独立于本系统编码，不参与主编码生成。

### 6.4 火工品特殊管控

火工品（HG）大类须执行：

- 独立台账管理，双人双锁
- 领用实名制，按需领用
- 库存变动实时上报
- 财务科目单独核算

### 6.5 新增物料流程

```
使用单位提申请（填写物料名称、规格、用途等）
        ↓
集团物资部审核（查重、归类、确定分类代码）
        ↓
系统自动生成编码（大类+分类+流水号）
        ↓
判断是否有对应 NC 存货？
    ├─ 有 → 填写 nc_inv_code，完成映射
    └─ 无 → 通知 NC 管理员创建存货档案，回填 nc_inv_code
        ↓
编码生效，各单位可用于业务单据
```

## 七、与用友 NC 对接规则

### 7.1 核心设计理念

> 物料是存货的明细，存货是物料的财务轨迹。


| 维度   | 物资系统（物料）                    | 用友 NC（存货）    |
| ---- | --------------------------- | ------------ |
| 管理对象 | 实物，按规格区分                    | 资金，按科目归集     |
| 粒度   | 细（一规格一编码）                   | 粗（同类合并核算）    |
| 典型例子 | 锚杆 Φ20×2400mm、锚杆 Φ22×2400mm | 锚杆（统一挂原材料科目） |
| 关系   | 存货的明细                       | 物料的汇总财务轨迹    |


**映射关系：物资 N → NC 1**。多个物料规格共享同一 NC 存货编码，推送财务时按 NC 存货汇总。

### 7.2 数据结构要求

投标方须为物料主表建立 NC 存货映射字段，并为组织级差异映射建立独立映射表。以下为字段示意，实际开发时须根据最终选定的国产信创数据库（如达梦、人大金仓等）调整语法：

```sql
-- 物料主表新增字段
ALTER TABLE material ADD COLUMN nc_inv_code VARCHAR(50) COMMENT 'NC存货编码（N:1）';
ALTER TABLE material ADD COLUMN nc_inv_name VARCHAR(100) COMMENT 'NC存货名称（冗余，便于核对）';

-- 多组织补充表（当同一物料在不同矿/组织的 NC 存货编码不同时使用）
CREATE TABLE material_nc_org (
  id          BIGINT PRIMARY KEY,
  mat_code    VARCHAR(20)  NOT NULL COMMENT '物资物料编码',
  nc_org_code VARCHAR(20)  NOT NULL COMMENT 'NC核算组织编码',
  nc_inv_code VARCHAR(50)  NOT NULL COMMENT '该组织下的NC存货编码',
  status      TINYINT      DEFAULT 1 COMMENT '0=停用 1=有效',
  remark      VARCHAR(200),
  UNIQUE KEY uk_mat_org (mat_code, nc_org_code)
);
```

**查找优先级：**

1. 先查 `material_nc_org`（组织级精确匹配）
2. 无记录则取物料表的 `nc_inv_code`（默认映射）
3. 仍为空 → 拦截推送，触发告警

### 7.3 推送 NC 逻辑

```
物资出入库明细
    ↓
按 nc_inv_code 分组汇总（数量、金额）
    ↓
生成 NC 入库单/出库单（存货粒度）
    ↓
NC 记账，挂对应会计科目
```

NC 侧仅感知存货维度的汇总数据，不感知物资侧的规格细分。

### 7.4 历史数据初始化

1. 导出 NC 现有存货档案（invcode + invname）
2. 人工或脚本按名称匹配，为物资物料表批量填写 `nc_inv_code`
3. 无法匹配的物料单独列清单，业务人员确认后补录
4. **上线前必做校验**：检查是否存在 NC 存货比物资物料更细的反例，局部用 `material_nc_org` 补充处理

### 7.5 映射监控


| 频率  | 内容                                              |
| --- | ----------------------------------------------- |
| 每日  | 统计 `nc_inv_code` 为空的有效物料数，异常告警                  |
| 每周  | 核对当周新增物料是否已完成映射                                 |
| 每月  | 全量对账：物资出入库记录 vs NC 凭证，按 `nc_inv_code` 核对数量/金额差异 |


## 八、编码查重与校验

1. 新增物料时系统须自动查重：编码唯一 + 名称规格组合唯一
2. 推送 NC 前须校验存货编码是否已存在
3. 每月运行全量对账，比对两系统编码差异并告警

### 9.2 特殊属性字段


| 字段名             | 适用大类            | 说明                                                |
| --------------- | --------------- | ------------------------------------------------- |
| shelf_life_days | HG、HX           | 保质期（天），0 表示无期限                                    |
| batch_required  | HG、HX           | 是否启用批次管理（Y/N）                                     |
| hazard_level    | HG              | 危险品等级（1/2/3 级）                                    |

| loss_rate       | YZ、HG、HX        | 合理损耗定额（%），0 表示无损耗，入库时系统据此判断差异是否在定额内               |
| is_anbiao       | SB、BP、JD、HG、LB、TF | 是否属于煤矿安全标志（MA）管控物资（Y/N）；Y 时在采购审批、供应商资质、到货验收、质检存档环节触发专项校验 |

### 9.3 批次管理规则（HG / HX）

- 启用批次管理的物料，入库时必须录入**生产日期**和**批次号**
- 系统自动计算**到期日**（生产日期 + 保质期）
- 到期前 **30 天**触发预警通知，到期当日自动冻结库存，禁止领用
- 领用时遵循 **先进先出（FIFO）**原则，系统自动推荐最早批次

### 9.4 安标（MA）管控规则

安标物资指依据国家煤矿安全标志管理制度，须取得煤矿安全标志（MA）方可在煤矿井下或相关工况使用的物资。系统须按以下规则执行专项管控：

- **识别**：`is_anbiao=Y` 为安标物资标识，物料主数据新增/修改时由物资部审核确认，审核通过方可落库
- **采购审批**：
  - 招标路径：采购申请审批链中**生产技术部与安监部分别审批**（独立节点）
  - 非招标路径：采购申请审批链中**生产技术部与安监部一同审批**（并行节点合并结论）
  - 涉安标物资不得规避集团分管领导签字要求
- **供应商资质**：供应商须提交对应物料 MA 证书复印件，证书编号、有效期登记入供应商档案，过期供应商不得中标
- **到货验收**：《设备直达到货验收单》和《设备安装调试完好联合验收单》须记录 MA 证书编号、实物铭牌照片；安监部参与联合验收
- **证书失效**：`ma_cert_expire` 到期前 60 天触发预警，到期后阻断该物料新增采购申请，已在途订单由安监部判定是否继续执行
- **追溯**：领用、调拨、报废环节均须保留 MA 证书编号与物料编码的关联记录，供监管检查

## 十、条码/二维码规范

### 10.1 默认方案

- 物料编码（如 `ZH-01-000001`）直接作为条码内容，生成 **Code 128** 条形码或 **QR Code**
- 打印标签时同时打印：物料编码、物料名称、规格型号、计量单位

### 10.2 批次物料扫码规范

启用批次管理的物料（HG/HX），标签内容扩展为：

```
物料编码 + "-" + 批次号
示例：HG-01-000001-20260315A
```

扫码时系统自动拆分，识别物料编码和批次号。

### 10.3 手持终端要求

- 须支持 Code 128、QR Code 扫码
- 出入库、盘点操作均应通过扫码完成，禁止手工录入编码
- 手持终端选型应在业务规则稳定后统一定型，并确保条码格式兼容

## 十一、扩展预留：辽宁能源统一编码与多维分类

### 11.1 背景与原则

阜矿集团为辽宁能源二级集团。辽宁能源尚未形成统一物料编码标准，但未来存在集团层面统一编码的可能。投标方须在一期建设中满足以下扩展预留要求，确保阜矿编码体系在大集团标准落地时可平滑对接，**无需重新编码**。

**核心原则：**

- 本系统一期建设中，必须预留与辽宁能源集团未来统一编码体系对接的字段、映射关系和接口扩展能力
- 在辽宁能源统一编码标准尚未正式发布前，不要求一期直接启用集团统一编码
- 不得以未来集团统一编码为由，要求对阜矿现行物料编码体系整体推倒重建

**多级编码对应关系：**


| 层级     | 编码主体                  | 对接方式                                |
| ------ | --------------------- | ----------------------------------- |
| 阜矿物资系统 | 本系统物料编码（XX-XX-XXXXXX） | 主体，永久不变                             |
| 用友 NC  | 存货编码                  | N:1 映射（`nc_inv_code` 字段）            |
| 辽宁能源集团 | 集团统一物料码（待定）           | 待定映射（`group_mat_code` 字段，预留 M:N 能力） |


### 11.2 数据结构预留

为满足一期招标和实施约束，数据模型至少须支持以下扩展预留：

```sql
-- 物料主表预留字段（暂时为空，等大集团标准落地后回填）
ALTER TABLE material ADD COLUMN group_mat_code VARCHAR(50) DEFAULT NULL COMMENT '辽宁能源集团物料编码（待对接）';
ALTER TABLE material ADD COLUMN group_mat_name VARCHAR(100) DEFAULT NULL COMMENT '集团物料名称（冗余，便于核对）';

-- 跨集团映射表（预建，暂时不用）
CREATE TABLE material_group_map (
  id              BIGINT PRIMARY KEY,
  mat_code        VARCHAR(20)  NOT NULL COMMENT '阜矿物料编码',
  group_mat_code  VARCHAR(50)  NOT NULL COMMENT '辽宁能源集团物料编码',
  group_org_code  VARCHAR(20)  DEFAULT NULL COMMENT '集团核算组织（如有差异）',
  map_ratio       DECIMAL(10,4) DEFAULT 1.0000 COMMENT '换算比例（单位不同时使用）',
  status          TINYINT      DEFAULT 1 COMMENT '0=停用 1=有效',
  remark          VARCHAR(200),
  created_at      DATETIME,
  UNIQUE KEY uk_mat_group (mat_code, group_mat_code)
);
```

### 11.3 对接时的工作内容

当辽宁能源发布统一编码标准后，本系统须具备完成以下工作的能力（与 NC 初始化对接类似）：

1. 获取辽宁能源物料/存货档案
2. 按名称、规格进行人工或脚本匹配，填写 `group_mat_code`
3. 无法匹配的物料，向大集团申请新增或确认归类
4. 上线后定期与大集团平台数据对账

### 11.4 编码体系兼容性说明

阜矿当前编码设计已考虑兼容性：

- 大类代码采用行业通用拼音缩写，与煤炭行业惯用分类一致，和大集团标准冲突概率低
- 编码不含组织信息，集团统一时无需剥离组织前缀
- 唯一性范围当前为阜矿集团内；若大集团要求全集团唯一，阜矿编码天然满足（集团内唯一 ⊆ 全集团唯一，只需确认流水号不与其他二级集团重复）

### 11.5 多维分类能力预留

本附件确立的"大类 + 分类 + 流水号"是物料的**唯一主身份**，不可变更。但在与用友 NC、辽宁能源集团、煤炭行业标准（如 MT/T 1090《煤矿物资分类与编码》）以及内部管理统计对接时，需要同时支持**多个分类视角**（财务科目、专业系统、行业标准、自定义管理维度等）。

投标方须在不改动主编码结构的前提下，为物料主数据预留"多维分类标签"能力，支持一物同时归属多个分类维度，且标签体系可在运行态扩展。

**11.5.1 设计原则**

- **主副分离**：主编码唯一、不变；标签是辅助视角，可增删改
- **维度正交**：不同 tag_type（财务 / 专业 / 行业 / 自定义）彼此独立，互不干扰
- **运行态扩展**：新增一个分类维度无需改表结构、无需发版
- **与现有映射不重复**：NC 存货映射（§7）、辽宁能源集团码映射（§11.2）已有专用字段，不再重复进 material_tag；material_tag 仅承载上述两者**之外**的分类维度

**11.5.2 数据结构预留**

以下为字段示意，实际开发时须根据最终选定的国产信创数据库调整语法：

```sql
-- 分类维度定义表
CREATE TABLE material_tag_type (
  tag_type    VARCHAR(20)  PRIMARY KEY COMMENT '维度代码，如 professional / industry / custom_xx',
  type_name   VARCHAR(100) NOT NULL COMMENT '维度名称',
  is_tree     TINYINT      DEFAULT 0 COMMENT '0=扁平 1=树形',
  status      TINYINT      DEFAULT 1 COMMENT '0=停用 1=启用',
  remark      VARCHAR(500)
);

-- 物料分类标签表（一物多标签）
CREATE TABLE material_tag (
  id          BIGINT PRIMARY KEY,
  mat_code    VARCHAR(20)  NOT NULL COMMENT '物资物料编码',
  tag_type    VARCHAR(20)  NOT NULL COMMENT '维度代码，引用 material_tag_type.tag_type',
  tag_code    VARCHAR(50)  NOT NULL COMMENT '标签代码',
  tag_name    VARCHAR(100) COMMENT '标签名称（冗余便于查询）',
  parent_code VARCHAR(50)  DEFAULT NULL COMMENT '父级标签代码，支持树形维度',
  status      TINYINT      DEFAULT 1 COMMENT '0=停用 1=有效',
  created_at  DATETIME,
  UNIQUE KEY uk_mat_tag (mat_code, tag_type, tag_code)
);
```

**11.5.3 一期建议启用的分类维度**

一期可先启用以下维度，其他维度随业务需要扩展：

| 维度代码 | 维度名称 | 结构 | 用途 |
| --- | --- | --- | --- |
| professional | 煤矿专业系统 | 扁平 | 采煤 / 掘进 / 运输 / 提升 / 通防 / 机电 / 洗选 / 辅助 |
| management | 管理属性 | 扁平 | 战略物资 / 常规物资 / 大宗物资 / 危险品 / 低值易耗 |
| industry | 行业标准编码 | 树形 | 预留 MT/T 1090 等行业标准对接（一期可不填充） |

**11.5.4 典型应用场景**

1. 按"煤矿专业系统"维度聚合统计（一号矿采煤系统本月消耗了哪些物资）——跨越 SB/BP/JD/ZH 等多个大类
2. 按"管理属性"维度筛选（危险品台账、战略物资储备清单）
3. 未来辽宁能源或行业标准发布后，新增 tag_type 挂接即可，主编码不动
4. 与 NC 财务统计联合时，可按 management 维度生成领导层报表

**11.5.5 投标响应要求**

投标方须说明：

- material_tag 结构的实现方式及运行态扩展机制
- 多维分类查询和聚合统计的实现能力（至少支持按任一维度分组汇总）
- 分类标签的批量导入 / 导出 / 清洗能力
- 多维标签与大类主编码的一致性校验机制（如同一物料在专业维度下的标签与大类有无逻辑冲突，提供校验工具）
