# 采购方式流程

> **来源：** `docs/流程调研/调研原文档/2.采购方式流程图（新表序调整）.docx`
> **范围：** 采购方式选择决策树（100 万阈值集体决策 → 集团目录 → 招标条件 → 招标/非招标三种）+ 三种非招标方式的执行细节
> **下游：** 供应商库 + 采购价格库（含历史价格）

---

## 总流程

```mermaid
flowchart TD
    Start[采购需求] --> Q1{单份项目<br/>是否超 100 万?}

    %% 100 万阈值
    Q1 -- 是 --> Council{集体决策}
    Council -- 否 --> R1[退回重新调研]
    Council -- 通过 --> Q2

    Q1 -- 否 --> Q2

    %% 集团目录内
    Q2{能源集团目录内?<br/>非建设项目} --> Q3{是否符合<br/>招标条件?}

    %% 招标 vs 非招标
    Q3 -- 符合 --> Tender[招标申请<br/>表/]
    Tender --> Q4{招标审批}
    Q4 -- 通过 --> InTender[进入招标程序]
    InTender --> WinTender[中标通知书]
    Q4 -- 未通过 --> NonTender

    Q3 -- 不符合 --> NonTender[执行非招标<br/>填报《非招标采购申请》<br/>表单]

    %% 三种非招标方式
    NonTender --> Way{非招标方式}
    Way --> M1[询价/竞价采购<br/>节 1]
    Way --> M2[竞争性谈判<br/>节 2]
    Way --> M3[直接采购<br/>节 3]

    %% 汇合到下游库
    WinTender --> DB[供应商库 +<br/>采购价格库<br/>含历史价格]
    M1 --> DB
    M2 --> DB
    M3 --> DB

    classDef council fill:#e6f4ff,stroke:#1677ff;
    classDef reject fill:#fff1f0,stroke:#cf1322;
    classDef branch fill:#fff7e6,stroke:#d48806;
    class Council council;
    class R1 reject;
    class M1,M2,M3 branch;
```

## 决策维度（自顶向下）

| 维度 | 阈值 / 判定 | 走向 |
|---|---|---|
| **金额阈值** | 单份项目 > 100 万 | 触发**集体决策**（不通过 → 退回重新调研） |
| **集团目录** | 能源集团目录内 / 非建设项目 | 走集团统一渠道 |
| **招标条件** | 符合 → 招标；不符合 → 非招标 | 招标审批未通过 → 降级走非招标 |
| **非招标方式** | 询价/竞价 / 竞争性谈判 / 直接采购 | 三选一 |

---

## 1. 询价/竞价采购

```mermaid
flowchart LR
    A[采购文件 表单] --> B[询价/竞价通知 表单]
    B --> C[3 家以上供应商报价<br/>表单]
    C --> D[采购评审 表单]
    D --> E[成交通知书 表单]
```

**关键约束：3 家以上供应商**报价。

## 2. 竞争性谈判

```mermaid
flowchart LR
    A[编制谈判文件] --> B[发出谈判文件]
    B --> C[3 家以上供应商响应]
    C --> D[谈判小组集中<br/>与单一供应商谈判]
    D --> E[确定成交候选人]
    E --> F[确定成交供应商<br/>成交通知]
```

**关键约束：3 家以上响应** + **谈判小组集中** + **与单一供应商谈判**（一对一谈判轮替）

## 3. 直接采购

```mermaid
flowchart LR
    A[采购邀请书 表单] --> B[直接采购报价单<br/>表单]
    B --> C[商定记录 表单]
    C --> D[成交通知 表单]
```

直接采购**不需要 3 家比价**，但要有**商定记录**作为留痕。

## 4. 招标采购（招标程序入口）

> 招标程序在本图为概略节点（招标申请 → 进入招标程序 → 中标通知书），具体招标内部流程由独立招标管理系统承载。
