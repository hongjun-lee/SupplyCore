# 采购入总库流程

> **来源：** `docs/流程调研/调研原文档/5.采购入总库流程图（按新表序调整）.docx`
> **范围：** 业务员打印收货通知 → 保管员收货验收（三类） → 合格登记台账 → 暂估入库 → 收到发票冲减暂估 → 物资采购入库单 / 保管卡片
> **核心：** **暂估入库 ↔ 发票冲减** 双轨闭环（最长不超 12 个月）

---

## 总流程

```mermaid
flowchart TD
    PO[采购合同] -->|打印依据| A[业务员打印"收货通知单"<br/>表单]
    Vendor[供货商 送货/物流<br/>随货清单] --> B
    A --> B[保管员依据《物资采购交库清单》<br/>+ 随货清单 收货]

    B --> Check[三类验收<br/>品种 + 数量 + 质量]
    Check --> Q1{是否合格?}

    %% 不合格分支
    Q1 -- 否 --> Reject[存放待验区]
    Reject --> Report[物资验收不符报告单]
    Report --> Return[办理退货或索赔]

    %% 合格分支
    Q1 -- 是 --> Log[登记到货/实验台账<br/>表单㊶㊷]
    Log --> Confirm[确认收货 表单]
    Log --> Inspect[到货验收 表单]

    %% 暂估入库 → 冲减暂估
    Confirm --> Tmp[暂估入库 表单㊵<br/>系统中填唯一编号]
    Tmp --> Inv[业务员接收供应商发票]
    Inv --> Reverse[按暂估入库料单<br/>唯一编号 抓对冲减暂估]
    Reverse --> Final[物资采购入库单<br/>表单㊸]

    %% 库存登记
    Inspect --> Card[保管员登记<br/>材料保管卡片<br/>表单㊹]

    Final --> End([完成])
    Card --> End

    classDef reject fill:#fff1f0,stroke:#cf1322;
    classDef temp fill:#fff7e6,stroke:#d48806;
    class Reject,Report,Return reject;
    class Tmp,Reverse temp;
```

> **暂估期限：最长不超 12 个月**（暂估入库到冲减暂估的时间上限）
> **审核依据：** 采购合同 / 执行订单

## 1. 入口与触发

| 输入 | 来源 | 用途 |
|---|---|---|
| **采购合同** | 详设 04 合同模块 | 业务员打印"收货通知单"的依据 |
| **随货清单 / 物流单** | 供货商送货 | 保管员实物收货比对 |
| **物资采购交库清单** | 上游计划 / 合同执行 | 保管员收货依据 |

## 2. 收货与三类验收

- **执行方：** 保管员
- **依据：** 物资采购交库清单 + 随货清单
- **验收三类（图示并列）：**

| 验收类型 | 关注点 |
|---|---|
| **品种验收** | 物料编码 / 规格型号是否一致 |
| **数量验收** | 实物数量 vs 单据数量 |
| **质量验收** | 物料质量是否合格 |

## 3. 验收结果分流

| 结果 | 后续 |
|---|---|
| **合格** | 登记到货/实验台账（表单㊶㊷） → 进入暂估入库 + 保管卡片登记 |
| **不合格** | 存放待验区 → 开具《物资验收不符报告单》 → 办理退货或索赔 |

## 4. 暂估入库 ↔ 发票冲减（核心闭环）

> 业务场景：实物到货早于发票到达，先按合同价"暂估"入账，待发票到达后按发票金额冲减暂估、出正式入库单。

| 顺序 | 动作 | 表单 |
|---|---|---|
| 1 | 暂估入库（系统填**唯一编号**） | 表单㊵ |
| 2 | 业务员接收供应商发票 | — |
| 3 | 按暂估入库料单**唯一编号**抓对冲减暂估 | — |
| 4 | 出具物资采购入库单 | 表单㊸ |

> **时限：最长不超 12 个月**（从暂估入库到收到发票冲减的窗口）

## 5. 库存登记（与暂估并行）

- **执行方：** 保管员
- **动作：** 登记材料保管卡片（表单㊹）
- **审核依据：** 采购合同 / 执行订单

## 与详设的对应关系（初步）

| 流程节点 | 详设落点 |
|---|---|
| 收货通知单（打印） | 详设 02 采购管理 — 收货环节 |
| 三类验收（品种/数量/质量） | 详设 06 入库管理 — 验收子模块；质检类对接详设 10 §3.3 QC 模块（4 项权限） |
| 物资验收不符报告单 + 退货索赔 | 详设 06 入库异常处理 — INV:RETURN 3 项权限对应 |
| 暂估入库（表单㊵） | 详设 03 主数据 — 暂估单据；表单㊵ 编号体系 |
| 暂估冲减（按唯一编号抓对） | 详设 06 暂估闭环逻辑 + 详设 11 财务对账接力 |
| 物资采购入库单（表单㊸） | 详设 06 正式入库单据 |
| 材料保管卡片（表单㊹） | 详设 06 库存台账 / 详设 07 库存计价基础 |
| 12 个月时限 | 详设 11 时限 + 详设 09 报表（暂估超期预警） |

## 待业务方核对要点

| # | 疑点 | 影响 |
|---|---|---|
| 1 | 三类验收（品种/数量/质量）是**并行**还是**串行**（任一不合格即终止）？ | 影响详设 06 验收状态机 |
| 2 | "最长不超 12 个月"约束的对象是**暂估入库时长**还是**审核依据有效期**？ | 影响详设 11 时限 |
| 3 | 暂估入库料单"唯一编号"的编码规则是什么？跨年度延续还是按月/年滚动？ | 影响详设 03 编码规则 |
| 4 | 不合格物资是否产生"暂估入库"？还是只有合格才暂估？ | 影响详设 06 暂估范围 |
| 5 | "确认收货"与"到货验收"两个表单是同一表的不同视角，还是独立单据？ | 影响详设 03 表单设计 |
| 6 | 退货/索赔后续流程（与供应商对账、扣款）是否在本图范围？ | 影响详设 06 退货闭环 |
