## 冲突 1：Q-MERGE-1 厂矿销售/移拨财税口径

### 业务方答复

> ✅ **(x) 否，应按外部销售（原因：税法：内部移拨视同销售）**
> ❌ ( ) 确认（按内部移拨实质开票）

### 关键解读

- 税法上"集团内部交易"在母子公司 / 兄弟公司之间，**仍按视同销售开票计税**（增值税口径）
- "实质内部移拨" 是会计 / 管理报表层面的合并抵消，**不影响开票口径**
- 因此**详设要按外部销售开票**（标准增值税专票），合并报表抵消由集团财务侧处理（不在 SupplyCore 范围）

### 详设修订动作

| 详设位置 | 当前状态 | 动作 |
|---|---|---|
| 详设 05 §销售开票 | 未明确口径 | **新增**：厂矿销售统一按外部销售口径开具增值税专用发票（与外部供应商口径一致） |
| 详设 08 NC 接口 §销售凭证科目 | 未明确 | **新增**：销售凭证按"主营业务收入 / 应收账款"标准科目；不区分内部 vs 外部 |
| 详设 05 §合同管理 — 内部企业合同 | 已有合同分类（流程 04） | **加注**：内部企业合同的销售开票仍按外部销售实质，**不走移拨凭证** |
| 详设 06 §出总库 / §直达 | 未明确 | **加注**：出库给厂矿走"销售出库"凭证，不是"内部移拨"凭证 |

### 影响范围

- ✅ 简化系统：不需要"内部移拨 vs 外部销售"双轨开票模板
- ⚠ 需要详设 05 显式说明"集团内部销售按外部口径"，避免后续质疑
- ⚠ 详设 08 NC 接口的销售凭证科目映射要与集团财务对齐（一期不动 NC 接口的话，**手工对账时按外部销售**）

## 冲突 2：Q-13-1 集体决议范围

### 业务方答复

> ✅ **(x) 现状：所有付款都集体决议**
> ❌ ( ) 同意限额分流（≤50 万财务自审 / 50-500 万业务分管 / >500 万集体决议）

### 关键解读

- 业务方明确**保留现状**：管理层集体决议作为**所有付款的前置审批节点**
- 与详设 10 V1.1 现行 WF-PAY-001 单链审批（发起→初审→财务复核→终审）的"金额阈值终审"方向**不一致**
- 必须在 WF-PAY-001 中**显式加入"集体决议"节点**，覆盖所有付款

### 详设修订动作

| 详设位置 | 当前状态 | 动作 |
|---|---|---|
| 详设 10 §6.2 WF-PAY-001 | 4 节点：发起 → 初审 → 财务复核 → 终审 | **改为 5 节点**：发起 → 初审 → 财务复核 → **集体决议（月度）** → 终审 |
| 详设 10 §九 配置项 | 付款申请终审阈值（CONDITION 节点） | **保留**作为"终审"节点的金额阈值；**集体决议节点**无金额阈值（强制全部经过） |
| 详设 10 §7 高敏感清单 | SENS-PAY-001 付款申请终审 | **加注**：付款申请须经集体决议，集体决议会议留痕入 A-14 审批日志 |
| 详设 05 §C-09 月度付款汇总 | 已定义 NC 月结推送 | **加注**：集体决议输出"付款对象/金额（细化到具体供应商）"作为 C-09 输入；C-09 由集体决议名单生成（非业务部门自由汇总） |
| 详设 05 §资金计划 | 未明确决议机制 | **新增 §C-09A 资金计划框架**：业务部按月拟订 → 集体决议拍板 → C-09 锁定名单制（无灵活额度，依 Q-13-2） |

### 影响范围

- ⚠ 详设 10 V1.1 模板需要小改 → V1.2
- ⚠ 集体决议节点的实施层（Catio Workflow）需要支持"会议节点"语义；可用 V2 DSL `approvalConfig.signMode = COUNTERSIGN`（多决议人会签）+ `approvalConfig.name = "集体决议"` 标识
- ⚠ "月度"集体决议如何触发：手工排期 / 系统提醒？需详设 11 SLA 配置

## 冲突 3：Q-02-6 3 家供应商约束

### 业务方答复

> ✅ **(x) 软提示**
> ❌ ( ) 强约束 + 特批例外
> ❌ ( ) 强约束无例外

### 关键解读

- 业务方倾向**保持灵活性**：不足 3 家供应商时**警告但允许提交**
- 与我们推断"强约束 + 特批"不同 — 业务方不要走特批流程，直接软提示让用户自行判断
- 适合现实复杂场景（如紧急采购 / 独家供应 / 集团目录内有限供应商）

### 详设修订动作

| 详设位置 | 当前状态 | 动作 |
|---|---|---|
| 详设 04 §询价/竞价 | 未明确校验级别 | **新增 §校验规则**：供应商响应数 < 3 时显示**警告提示**（"建议补充供应商至 3 家以上"），但**允许提交**；提交后该字段进入 R 域报表"少于 3 家供应商比例"指标 |
| 详设 04 §竞争性谈判 | 同上 | 同上规则 |
| 详设 04 §直接采购 | 已说明无 3 家约束 | **加注**：直接采购本身不要求 3 家，不在此规则范围 |
| 详设 09 §六预警规则 | 未涉及 | **新增 ALR-PUR-001 询价供应商不足预警**：当询价/谈判**少于 3 家时**触发"重要"级提醒，进 RPT-PUR-INSUFFICIENT 月度报表 |
| 详设 11 §六合规与审计 | 未涉及 | **新增**：少于 3 家询价 / 谈判须在备注栏说明原因（系统不强制，但作为后续审计依据） |

### 影响范围

- ✅ 简化用户操作：不需要"特批"路径，减少阻断
- ⚠ 风险：可能滋生"为达成而忽略 3 家底线"的合规问题；详设 11 应加"备注必填"软引导 + 详设 09 月度报表暴露指标
