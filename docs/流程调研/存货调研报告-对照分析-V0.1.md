## 二、PDF 提出的 3 大问题

### 问题 1 · 存货资金占用规模偏大，资金使用效率偏低

> 截至 3 月末，集团存货账面占用资金合计 50,784 万元，**剔除库存煤炭 33,089 万元，其余物资存货占用资金 17,695 万元**。大额存货沉淀流动资金，直接降低存货周转效率，加重资金周转压力，增加集团整体经营风险，制约企业现金流良性循环。

### 问题 2 · 存货结构失衡，无效库存占比偏高

> 在 17,695 万元物资存货中，**非正常储备存货高达 7,208 万元，占比 40.73%**。受去产能、停产等因素影响，部分物资长期闲置积压，**生产急需物资储备不足、闲置物资占用资金的结构性矛盾突出**。物资存放周期过长，部分配件、材料淘汰及性能衰减，库存商品无法形成销售，进一步加大资产贬值。

### 问题 3 · 暂估材料管理不规范，账务核销滞后

> 部分单位暂估材料挂账时间较长，**存在超 3 个月未取得发票、未及时冲调账务现象**，造成账面存货与实际不符，间接形成隐性积压，不利于存货真实管控及资金精准核算。

## 四、系统能力对照（3 大问题 × 7 项措施）

### 4.1 问题 1：存货资金占用偏大 — ✅ 可解决

| 我们的能力 | 来源 |
|---|---|
| 库存余额表 + 周转率分析 | 详设 09 §五 + 招标 §6.7 |
| 呆滞库存清单（默认 180 天，可调）| 详设 09 ALR-INV-005 + 招标 §9 |
| **库龄分析报表** + 系统自动库龄分布 | 详设 09 + 流程 10 |
| **内部调剂三层级**（厂矿内部 → 集团内部 → 对外销售） | 业务方答 Q-10-4（详设 06 调剂模块） |
| 严控新增采购 — **库存优先使用原则** | 政策 04 + 详设 04 采购计划 |
| 化整为零反规避检测（30 天累计 + 应招标阈值） | A14 已落地（split-detection.html） |

### 4.2 问题 2：存货结构失衡 — ✅ 可解决

| 我们的能力 | 来源 |
|---|---|
| **超储双维度判定**（金额超储 + 周转天数超期）| Q-10-3 业务方决议 |
| 5 类自动预警 ALR-INV-001~005（低库存 / 超储 / 临期 / 过期 / 呆滞）| 详设 09 §六 |
| **报废核销审批分档**（< 5 万业务主管 → 5-50 万分管 → > 50 万企业负责人 → > 500 万集团）| Q-10-5 业务方决议（详设 10 §6.2 WF-SHT-001 + WF-SCP-001） |
| **设备残体处置**（高敏感 + 资产盘活计划）| 详设 02 业务模块 §5.8 + Q-00-1 联动 |
| 物料分类 + 状态管理（启用 / 停用 / 报废 / 锁定）| 详设 03 物料主数据 |
| **三对一致约束**（财务账 = 系统账 = 保管卡片 = 实物） | 流程 10 + 详设 06 + 详设 11 对账定时任务 |

### 4.3 问题 3：暂估管理不规范 — ✅ 可解决（**但需调整窗口默认值**）

| 我们的能力 | 来源 |
|---|---|
| **暂估闭环 6 个月窗口**（可配置 90-365 天） | 详设 06 §7.6 + 招标 V1.2 §6.7 |
| **D-90 / D-30 / D-0 / D+30 四阶段升级**（一般 / 重要 / 紧急 / 月度专项） | 业务方答 Q-05-2 + 二期 A8 已落地（tentative-estimate.html） |
| **D-0 阻断该供应商新采购订单**（强制对账完成才解除）| 详设 09 ALR-INV-007 + 招标 V1.2 |
| **唯一编号一对一抓对冲减**（ZG-{YYYYMM}-{8 位流水}） | 业务方答 Q-05-3 |
| BIZ-002 暂估凭证 + BIZ-003 红字冲销 | 详设 08 接口清单 |
| 暂估超期月报（详设 09 §六 暂估超期物资）| 详设 09 |

> ⚠ **关键差异点：财务部要求"超 3 个月"vs 当前业务方答复"6 个月"**
>
> | 来源 | 时限 |
> |---|---|
> | PDF 财务部报告（2026-04-29）| **3 个月** ❗ |
> | 业务方 Q-05-2 答复（2026-05-09）| 6 个月 |
> | 系统默认 `INVENTORY_TENTATIVE_WINDOW_DAYS` | 180 天（6 月）|
>
> **冲突解读：** PDF 是财务侧调研结论（关注"账实一致"），业务方答复是业务侧体感（关注"操作弹性"）。两者**不矛盾，建议合并**：
>
> - **D-90（3 个月触发）→ 财务部要求的"催收发票"硬约束**（一般级预警 + 强制催票）
> - **D-30（5 个月）→ 重要级 + 财务采购联合处理**
> - **D-0（6 个月）→ 紧急级 + 阻断新订单**
> - **D+30（7 个月）→ 月度专项对账**
>
> 三段预警刚好把"3 个月业务催收 + 6 个月硬阻断"两个口径融合，**不需修改配置**，只需在文档中明确："**财务部 3 个月红线 = D-90 阶段触发**"。

## 六、最少上线模块建议

### 6.1 P0 核心 7 模块（**解决全部 3 个问题的最小必要集**）

| 序 | 模块 | 直接解决 |
|---|---|---|
| 1 | **基础档案与组织仓库管理** | 数据底座，所有业务前置 |
| 2 | **物料主数据与编码管理** | 一物一码 + 库龄属性 + 状态管理（问题 2 基础）|
| 3 | **采购到货与入库管理**（**含暂估闭环**） | **问题 3 直接解决** + 部分问题 2 |
| 4 | **库存领用与退料调拨管理** | **问题 1 内部调剂** + 周转可视化 |
| 5 | **盘点与废旧处置管理** | **问题 2 残体处置** + 减值识别 |
| 6 | **财务与 NC 接口管理** | 减值传递 + 暂估凭证 + 联通财务部 |
| 7 | **报表预警与查询分析** | **问题 1/2/3 全部预警可视化** |

### 6.2 P1 强烈推荐（4 个）

| 模块 | 增益 |
|---|---|
| **需求提报与采购计划** | 严控新增源头（整改措施 3） |
| **招投标管理** | 化整为零检测 + 价格控制（整改措施 3） |
| **统一 AI 能力开放** | 自然语言查询"哪些物资呆滞？哪个矿哪个物料？"（整改措施 7 财务预警） |
| **供应商管理** | 5 等级评价联动（避免低质物资形成无效库存） |

### 6.3 P2 可选（4 个）

| 模块 | 备注 |
|---|---|
| 系统管理与基础支撑 | 权限 / 审批流 / 日志（合规底座） |
| 合同管理 | 与本次"存货专项"关联弱 |
| 资金计划管理 | 与本次"存货专项"关联弱 |
| 设备租赁管理 | 与本次"存货专项"关联弱 |

### 6.4 最小演示版（3 模块也能见效）

> 如果资源紧张，**只上 3 个模块**也能初步解决：

1. 物料主数据（底座 + 库龄属性）
2. 库存领用与退料调拨（台账 + 调剂）
3. 报表预警（库龄 / 呆滞 / 暂估超期 3 张关键报表 + 5 类预警）

> **但要完整闭环（尤其问题 3 暂估），"采购入库 + NC 接口"两个模块必不可少**，所以 P0 7 模块是**底线**。

## 八、风险点提示

| # | 风险 | 缓解 |
|---|---|---|
| 1 | 系统覆盖能力强，但实际上线节奏受制于资源 + 数据迁移 + NC 联调进度 | 按 P0 7 模块顺序上线，可在 12-16 周内完成核心闭环 |
| 2 | 财务部要求"3 个月催收发票"为硬约束，业务方实操可能不达标 | 通过 D-90 预警 + 月度财务核查 + 阻断后续采购 形成约束闭环 |
| 3 | 设备残体 730 万元转入固定资产清理 | 需 NC 接口对账定时任务支持账务调整（详设 08 §九）+ 业务方对账操作规范 |
| 4 | 滞销产成品 1,710 万元（金石膨润土制品）属于特殊业务 | 不在物资公司核心业务（采购+库存）范围，建议**财务部主导处置**，本系统仅提供库存数据导出 |
| 5 | 各单位"账外物资 / 手头料"清理工作量大 | 系统能力到位但需业务侧动员；建议设"账外物资清理专项"作为上线后第一个月运营动作 |
