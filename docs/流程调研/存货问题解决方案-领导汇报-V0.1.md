# 阜矿集团存货问题解决方案 — 领导汇报材料 V0.2

> **汇报对象：** 项目领导小组 / 集团领导
> **背景：** 阜矿集团财务部 2026-04-29《存货情况调研报告》提出 3 大问题 + 7 项整改措施
> **覆盖范围：** **集团 17 家子单位**（PDF 调研为 11 家，本方案为系统上线全集团覆盖范围）
> **核心建议：** **分两步走** —— 立即启动"存货摸底 + 数据底座建立",并行推动物资管理系统**最小模块**上线
> **预期收益：** **数据先行 + 系统稳步上线**，1-2 个月见首批成效，**8-12 个月**完成全集团核心闭环
> **时间安排原则：** 鉴于 17 家单位差异大 + 现系统使用深度不足需结合实物统计 + 与现运行系统建立映射，**周期保守留富裕**，避免后期返工

---

## 一、问题摘要（一页看懂）

| 维度 | 数据 |
|---|---|
| 集团存货账面 | **50,784 万元**，已计提减值 3,271 万 |
| 物资存货（剔除煤炭） | **17,695 万元** |
| **非正常储备**（积压 / 损毁 / 滞销 / 残体） | **7,208 万元，占比 40.73%** ⚠ |
| 预计新增计提减值 | **1,340 万元** |

**财务部点出的 3 大根因：**

1. **存货资金占用偏大** → 资金沉淀，周转效率低
2. **结构失衡，无效库存占比偏高** → 急需物资不足 vs 闲置积压并存
3. **暂估材料管理不规范** → 超 3 个月未取得发票，账实不符

> **结论：** 现状靠人工管控已不可持续，**必须依托物资管理系统建立标准化、数字化、可追溯的存货管理体系**。

---

## 二、解决方案核心 — **两步走 · 并行推进**

### 🔵 第一步:立即启动「存货摸底 + 数据底座建立」(**0-10 周,不依赖系统**)

> **这一步无需等待系统上线即可立即启动**,由业务部门牵头,IT 配合,**为系统稳健上线铺平道路**。
>
> **关键前提说明:**
> - **覆盖 17 家子单位**(PDF 摸底范围为 11 家,本方案扩展至全集团)
> - **不能只靠现有系统数据导出** — 因现系统使用深度不够、口径不统一,**必须结合现场实物统计**
> - **基础档案需与现运行系统建立映射** — 避免新旧系统脱节,保留历史业务追溯能力
> - **17 家单位差异大** — 物料编码 / 仓库口径 / 计量单位等需逐家比对去重

| # | 工作内容 | 牵头 / 配合 | 周期(17 家口径) | 与 PDF 整改对应 |
|---|---|---|---|---|
| **1.1** | **存货全面摸底** — 17 家单位**现场实物盘点 + 库龄登记** + 状态分类(正常 / 超储 / 损毁 / 残体 / 滞销);**不依赖现系统数据,以实物核实兜底** | 经发部 + 物资公司 + 各单位 | **4-6 周** | 措施 1、4、5 |
| **1.2** | **基础档案标准化 + 现系统映射** — 组织、仓库、库区货位、计量单位、成本中心、使用单位统一编码;**与现运行系统建立 ID 映射表**(确保历史数据可追溯) | 物资公司 + 财务部 + IT | **3-4 周** | 措施 1 数据底座 |
| **1.3** | **物料主数据治理** — 一物一码 + NC 存货映射 + 物料分类(按 13 项分类标准)+ 安全 / 火工 / 防爆等特殊属性;**17 家差异大,需逐家比对去重 + 现系统映射** | 物资公司主导 | **6-8 周** | 措施 1、4 |
| **1.4** | **暂估专项清查** — 17 家单位**超 3 个月暂估材料**逐笔登记 + 限期催票 / 冲销 | 各单位 + 财务部 | **4-6 周** | 措施 6 |
| **1.5** | **账外物资清理** — 领用未消耗物资全面盘点 + 统一入库管控;**实物 + 台账双轨核对** | 各单位 | **4-6 周** | 措施 4 |
| **1.6** | **现系统数据导出 + 映射对账** — 17 家现系统的存货数据导出 → 与摸底实物对账 → 差异清单 → 治理方案 | IT + 物资公司 + 各单位 | **3-4 周** | 措施 5 |

> **第一步并行节奏:** 1.1(实物盘点)+ 1.4(暂估清查)+ 1.5(账外清理)可**全集团 17 家并行**启动;1.2 + 1.3 + 1.6 需有专人统筹(建议物资公司牵头 + IT 配合)。**总周期 8-10 周(保守估算,留 1-2 周富裕应对差异处理)**。

**第一步价值:**

- ✅ **真实存货数据沉淀** —— **不依赖现系统**,实物统计兜底,给系统提供**可信的期初数据**
- ✅ **基础数据资产化** —— 数据底座是系统稳健上线的**关键前置**,省去未来 8-12 周的数据治理返工
- ✅ **业务流程梳理** —— 摸底过程同步梳理操作规范,**17 家人员同步培养**
- ✅ **快速见效** —— 暂估清查 / 账外清理 / 残体处置等动作**首月即可在部分单位出成果**,不必等系统全集团统一上线
- ✅ **新旧系统映射** —— 1.2 + 1.6 建立的映射表是后续**历史数据迁移**与**业务连续性**的关键保障

---

### 🟢 第二步:物资管理系统「最小模块」分批上线(**第 4 周与第一步并行启动 → 第 32 周完成核心闭环**)

> 第一步推进的同时,**第 4 周启动系统建设**,按"先骨架、再业务、后增强"分批上线。**17 家单位差异大,建议先在 2-3 家试点 → 再分批推广**,确保稳健。

#### 2.1 必上模块(P0 共 7 个,**底线**,全部解决 PDF 3 大问题)

| 上线批次 | 模块 | 周期 | 试点 → 推广节奏 | 直接解决 |
|---|---|---|---|---|
| **第 1 批**<br/>(第 4-12 周) | **① 基础档案与组织仓库**<br/>**② 物料主数据与编码** | **8 周** | 第 4-8 周试点 2-3 家 → 第 9-12 周扩展到 17 家 | 数据底座(与第一步无缝衔接) |
| **第 2 批**<br/>(第 13-22 周) | **③ 采购到货与入库**(含**暂估闭环**)<br/>**④ 库存领用与退料调拨**<br/>**⑤ 盘点与废旧处置** | **10 周** | 第 13-16 周试点 → 第 17-22 周分批推广 | **问题 1 调剂周转**<br/>**问题 2 残体处置**<br/>**问题 3 暂估闭环** |
| **第 3 批**<br/>(第 23-32 周) | **⑥ 财务与 NC 接口**<br/>**⑦ 报表预警与查询分析** | **10 周** | 第 23-26 周 NC 联调 + 试点 → 第 27-32 周全集团切换 | 减值传递 + 全部预警可视化 |

> **节奏说明:** 17 家单位差异大,**每批均采用"试点 → 推广"两阶段**,避免一次性切换风险;NC 接口联调受 NC 厂商节奏制约,留 4 周缓冲。

#### 2.2 增强模块(P1 推荐 4 个,**第 33-40 周**继续推进)

| 模块 | 增益 |
|---|---|
| 需求提报与采购计划 | 严控新增源头(措施 3) |
| 招投标管理 | 化整为零检测(措施 3) |
| 统一 AI 能力开放 | 自然语言查询库龄 / 呆滞 / 异常 |
| 供应商管理 | 5 等级评价联动 |

| **2** | **是否同步启动 P0 7 模块系统建设?** | ✅ **第 4 周启动**(与第一步并行) | **8 个月完成核心闭环**(留富裕) |

## 五、为什么"两步走 + 并行"是最优解

### 5.1 与"等系统上线再动手"对比

| 维度 | 等系统先 | **两步走（推荐）** |
|---|---|---|
| 见效时间 | 16-20 周后 | **3 周即出第一波成果**（摸底 / 暂估清查） |
| 数据质量 | 系统上线时还要补 4-6 周数据治理 | **第一步已完成**，系统直接跑 |
| 业务培训 | 上线后才培训 | **摸底过程已同步培训**，上线即用 |
| 财务部诉求 | 滞后 4-5 个月 | **即时回应**（暂估清查 / 残体处置当月出数） |
| 风险 | 系统延期 = 整体延期 | **数据底座独立可用** |

### 5.2 与"只做业务摸底不上系统"对比

| 维度 | 只摸底不上系统 | **两步走（推荐）** |
|---|---|---|
| 一次性成果 | ✅ 短期数据清晰 | ✅ 短期 + 长期 |
| 可持续性 | ❌ 半年后再次失控（无系统支撑） | ✅ **系统自动管控** |
| 暂估闭环 | ❌ 人工催收易疏漏 | ✅ D-90/D-30/D-0/D+30 自动预警 + 阻断 |
| 库龄分析 | ❌ Excel 难以维护 | ✅ 系统自动库龄分布 + AI 查询 |
| 减值预警 | ❌ 滞后发现 | ✅ 实时预警 |

## 六、风险与应对

| # | 风险 | 应对 |
|---|---|---|
| 1 | **17 家单位差异大,口径不统一**(物料编码 / 仓库 / 计量单位) | 第一步 1.3 物料主数据治理 6-8 周用于逐家比对去重 + 现系统映射;先做 2-3 家试点验证再全量推广 |
| 2 | **现系统使用深度不足,数据可信度低** | 第一步坚持**实物统计兜底**,不只信导出数据;1.6 现系统对账作差异分析 |
| 3 | 各单位摸底进度不一致 | 经发部统一节奏 + 周例会 + 红黄绿三色预警;**17 家骨干分头跟进** |
| 4 | 系统建设资源不足导致 P0 延期 | 试点 → 推广两阶段降低风险;先保第 1+2 批 5 个模块上线(22 周内),第 3 批可灵活调整 |
| 5 | NC 联调进度受 NC 厂商节奏制约 | 已在招标 V1.2 中预留过渡口径(手工录入 → NC 自动);留 4 周缓冲 |
| 6 | 业务部门动员配合度 | 报告本身已点名经发部牵头 + 7 项整改责任部门明确,**建议领导小组每月督导 + 季度考核纳入 KPI** |
| 7 | **数据治理工作量大,17 家差异处理耗时** | 第一步周期保守留 1-2 周富裕;允许有差异的物料"挂账先用,后续治理" |
| 8 | **现系统与新系统并行期** | 建立映射表 + 关键业务双系统对账(每月 1 次),并行期 2-3 个月后视稳定情况停用现系统 |

## 七、一句话结论

**立即启动「存货摸底 + 数据底座建立」(第一步,0-10 周,覆盖 17 家)**;**第 4 周同步推动物资管理系统 P0 最小模块(7 个)分 3 批"试点 + 推广"上线**;**32 周内闭环 PDF 全部问题**;**周期保守留富裕**应对 17 家差异 + 现系统映射 + 实物统计;数据先行 + 并行推进 + 试点稳健 = 风险最小、可持续。
