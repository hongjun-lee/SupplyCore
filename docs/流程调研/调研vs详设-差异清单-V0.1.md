# 调研流程 vs 现详设 — 差异清单 V0.1

> **目的：** 把 13 张流程调研 md 中观察到的"调研流程 vs 现详设方案"差异统一登记，待逐项核对详设后给出最终建议。
> **原则：** 调研 = 现状（手工 + 既有规约）；详设 = 我们正在做的新系统方案。本项目包含**业务再造 + 流程优化**目标，不应默认"对齐调研"。
> **状态：** **V0.1 简略版** — 列出潜在差异点 + 我的初步倾向，**未逐项验证详设的实际描述**；待用户挑选深挖项后，再去详设找具体证据回填。

---

## 分类说明

| 标签 | 含义 |
|---|---|
| **🔄 流程优化建议** | 调研流程能跑但有改进空间，建议**详设偏离调研** |
| **➕ 详设需补充** | 调研有，但**详设可能缺失或未明确** |
| **✅ 调研合理** | 调研做法合理，需要**确认详设有覆盖** |
| **❓ 待澄清** | 调研描述本身不清楚，无法判断 |

## A. 流程优化建议（建议详设偏离调研）

### A1. 🔄 暂估入库期限：12 个月 → **建议 6 个月**

- **调研**：[05-采购入总库流程.md](05-采购入总库流程.md) "最长不超 12 个月"
- **依据**：行业标杆 3-6 个月。12 个月窗口太长，应付账款不实在，掩盖发票拖延
- **倾向**：缩到 **6 个月**，超期触发预警 + 强制供应商对账
- **影响详设**：详设 06 暂估闭环 + 详设 11 时限 + 详设 09 报表（暂估超期）

### A2. 🔄 库存计价：基层"月末一次"+ 总库"移动加权"混用 → **统一移动加权**

- **调研**：[12-采购供应入财务账流程.md](12-采购供应入财务账流程.md) "移动加权平均（与基层一致）；基层不适用月末一次加权平均"（描述本身有矛盾）
- **依据**：混用导致月末对账复杂度翻倍；移动加权实时性更好
- **倾向**：**全口径统一移动加权**
- **影响详设**：详设 07 库存计价 — 删除"月末一次"分支

### A3. 🔄 付款流程：月度集体决议**全前置** → **限额分流**

- **调研**：[13-支付采购款流程.md](13-支付采购款流程.md) "管理层集体决议"前置所有付款
- **依据**：集体决议每月一次会成为瓶颈，小额常规付款不应阻塞
- **倾向**：**限额内（如 ≤50 万）由财务审核即可，>50 万或异常上集体决议**
- **影响详设**：详设 05 资金计划 + 详设 10 阈值审批节点

### A4. 🔄 大额合同：~10 级会签链全串行 → **集团三部门并行**

- **调研**：[04-合同审批流程.md](04-合同审批流程.md) 大额路径（>20 万）：物资公司内部 5 级 + 集团 4-5 级，**全串行**
- **依据**：详设 10 V1.1 审批模板支持 signMode（COUNTERSIGN/OR_SIGN），集团"财务部 / 法控部 / 经发部"完全可以并行
- **倾向**：**集团三部门改并行会签**，预计审批链时长缩短 30-50%
- **影响详设**：详设 10 §6 审批模板 — 大额合同模板的 signMode 配置

### A5. 🔄 纸质审批双轨 → **全电子化**

- **调研**：[01-采购计划流程.md](01-采购计划流程.md)、[03-供应商管理流程.md](03-供应商管理流程.md) 等多处提到"电子版 + 纸质审批版双轨"
- **依据**：电子签章 + PDF 归档已合规，纸质双轨增加错配风险
- **倾向**：**取消纸质审批版**，仅保留电子审批 + 必要时打印归档
- **影响详设**：详设 02-08 单据流转 + 详设 11 归档规约

### A6. 🔄 出库单"过月做废 + 不能重复打印" → **状态机 + 一次性 token**

- **调研**：[06-出总库流程.md](06-出总库流程.md) 当前的反作弊措施
- **依据**：电子签收 + 状态机锁定 + 一次性 token 替代后，"过月失效"可以保留作为兜底，但不必硬作废
- **倾向**：**状态机 + token**为主，"过月失效"作为兜底定时任务
- **影响详设**：详设 06 出库单状态机 + 详设 11 定时任务

### A7. 🔄 直达物资"车号+货物+人员同框影像" → **详设 11 落具体规格**

- **调研**：[08-直达使用单位流程.md](08-直达使用单位流程.md) 强制要求影像附件
- **依据**：是好的防伪手段，但缺规格定义（清晰度/角度/时间戳/GPS）
- **倾向**：**保留**，但详设 11 应落具体规格 + 自动校验（EXIF 时间戳、上传时间窗）
- **影响详设**：详设 11 附件强校验

### A8. 🔄 "出租设备"业务类型 → **建议一期不实施**

- **调研**：[00-内控总体框架.md](00-内控总体框架.md) 框架图唯一提到，13 张子流程图都没覆盖
- **依据**：调研材料本身没把"出租设备"做成独立流程，可能是非主流业务
- **倾向**：**一期不实施**，纳入二期或挂账
- **影响详设**：详设 02 业务类型枚举（去掉 RENT 或标 [一期不实施]）

## B. 详设需补充（调研有，详设可能缺）

### B1. ➕ 供应商 5 级分类库

- **调研**：[03-供应商管理流程.md](03-供应商管理流程.md) **优质 / 重点 / 一般 / 高风险 / 不合格**
- **影响详设**：详设 03 供应商主数据 — `category` 字段值域应明确这 5 类
- **联动**：与 [11-后评价流程.md](11-后评价流程.md) 反馈结果挂钩

### B2. ➕ 供应商 4 种角色标签（中标/投标/潜在/成交）

- **调研**：[03-供应商管理流程.md](03-供应商管理流程.md)
- **影响详设**：详设 03 供应商主数据 — 多值角色标签字段
- **倾向**：保留，作为评价输入维度

### B3. ➕ 采购方式枚举：招标 / 询价 / 谈判 / 直接

- **调研**：[02-采购方式流程.md](02-采购方式流程.md) 详细描述四种方式
- **影响详设**：详设 04 — `pur_method` 枚举（TENDER / INQUIRY / NEGOTIATION / DIRECT）

### B4. ➕ 6 类财务账业务枚举

- **调研**：[12-采购供应入财务账流程.md](12-采购供应入财务账流程.md) 直达材料 / 直达设备 / 外购入库 / 代储 / 外委检修 / 委托加工
- **影响详设**：详设 08 凭证路由 — 6 类业务的凭证类型映射表

### B5. ➕ 履约保证金触发规则

- **调研**：[04-合同审批流程.md](04-合同审批流程.md) 仅说"符合缴纳条件的"，未定义条件
- **影响详设**：详设 05 合同 — `bond_required` 触发规则（金额阈值 + 合同类别）
- **倾向**：详设应**显式定义触发规则**（如金额 > X 万 + 类别 ∈ {工程/服务} → 必交保证金）

### B6. ➕ 委托加工损耗率字段

- **调研**：[07-委托加工流程.md](07-委托加工流程.md) "按约定损耗率"
- **影响详设**：详设 05 委托加工合同 — `loss_rate decimal(5,4)` 字段；详设 06 入库校验

### B7. ➕ 防爆/煤安按月反馈强制时限

- **调研**：[11-后评价流程.md](11-后评价流程.md) 高危类强制按月
- **影响详设**：详设 11 时限 + 详设 09 报表（合规反馈到期预警）+ 详设 03 物资分类（与 M-05 `is_safety_special` 字段联动）

### B8. ➕ "无计划不采购"原则的系统性约束

- **调研**：[01-采购计划流程.md](01-采购计划流程.md) 全程贯穿原则
- **影响详设**：详设 04 采购订单创建时**强校验**关联计划单号（无计划单号不允许创建）；详设 02 异常路径

### B9. ➕ "3 家以上供应商比价"硬约束

- **调研**：[02-采购方式流程.md](02-采购方式流程.md) 询价/竞价 + 谈判都要求
- **影响详设**：详设 04 询价/谈判模块 — 最低供应商数校验（系统强约束）

### B10. ➕ 销售开票口径区分（厂矿 = 内部销售 vs 移拨）

- **调研**：[00-内控总体框架.md](00-内控总体框架.md)、[06-出总库流程.md](06-出总库流程.md)、[08-直达使用单位流程.md](08-直达使用单位流程.md) 都提到"对厂矿开销售发票"
- **疑点**：厂矿是集团内部用户单位，"销售发票" vs "移拨" 财税口径需明确
- **影响详设**：详设 05 销售开票 + 详设 08 NC 接口（销售/移拨科目区分）

## C. 调研合理（需确认详设覆盖）

### C1. ✅ 三对一致约束（财务账=系统账=保管卡片=实物）

- **调研**：[10-存货管理流程.md](10-存货管理流程.md)
- **影响详设**：详设 06 库存台账 + 详设 11 对账定时任务 + 详设 09 报表（差异预警）

### C2. ✅ 银行结算硬约束（不允许现金/票据）

- **调研**：[13-支付采购款流程.md](13-支付采购款流程.md)
- **影响详设**：详设 05 付款方式枚举只保留银行结算

### C3. ✅ 价审前置（合同/发票均经价审）

- **调研**：[12-采购供应入财务账流程.md](12-采购供应入财务账流程.md) 共性原则
- **影响详设**：详设 05 合同/发票状态机 — `price_audited` 状态前置

### C4. ✅ 反馈影响供应商分类（11 → 03/04 联动）

- **调研**：[11-后评价流程.md](11-后评价流程.md) 处理结果应回流到供应商分类
- **影响详设**：详设 03 + 04 跨模块联动 — 反馈结果触发供应商 category 重评估

### C5. ✅ 三类验收（品种/数量/质量）

- **调研**：[05-采购入总库流程.md](05-采购入总库流程.md)、[08-直达使用单位流程.md](08-直达使用单位流程.md)
- **影响详设**：详设 06 验收子模块 + 详设 10 §3.3 QC 模块（4 项权限已含）

## D. 待澄清（调研描述本身不清晰）

### D1. ❓ "100 万阈值" vs "1000 万阈值" 是否独立？

- **调研**：[02-采购方式流程.md](02-采购方式流程.md) 100 万触发集体决策；[04-合同审批流程.md](04-合同审批流程.md) 1000 万触发总经理会签
- **疑点**：两个阈值是否串联（先过 100 万再过 1000 万）？还是独立维度？

### D2. ❓ "能源集团目录内（非建设项目）" 判断的具体规则

- **调研**：[02-采购方式流程.md](02-采购方式流程.md)、[07-委托加工流程.md](07-委托加工流程.md) 都用此判断
- **疑点**：目录由谁维护？非建设项目如何识别？是否已经有维护机制？

### D3. ❓ "直达"两个目录是同一目录还是不同目录？

- **调研**：[08-直达使用单位流程.md](08-直达使用单位流程.md) "可直达物资目录"
- **疑点**：和 [00-内控总体框架.md](00-内控总体框架.md) "直达·大型设备 / 直达·材料配件" 二分是否对应？目录是分大型设备 + 材料配件两份还是一份？

### D4. ❓ "外委·设备检修" 没有独立流程图

- **调研**：[12-采购供应入财务账流程.md](12-采购供应入财务账流程.md) 节 5 提到该业务，但 docx 列表中无独立流程图
- **疑点**：是否漏掉了？还是合并到 12 中视为子分支？

### D5. ❓ "直达材料 vs 直达设备" 财务凭证差异

- **调研**：[12-采购供应入财务账流程.md](12-采购供应入财务账流程.md) 节 1/2 — 材料"挂应付账款"，设备"开销售发票"
- **疑点**：两者业务实质差异？为何同样直达却凭证不同？
