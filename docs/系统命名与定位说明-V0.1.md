## 二、为什么改名

### 2.1 旧名（"物资供应管理系统"）的局限

招标合同口径 "物资供应管理系统" — 当时立项时偏传统物资管理范围：物料 / 采购 / 库存 / 出入库。

### 2.2 实际建设范围已经超出旧名

系统现状（截至 2026-05-21）：

| 业务链条 | 已实现模块 |
|---|---|
| **需求端** | 需求计划 / 采购计划 / 直达申请 / 物料申请 |
| **采购端** | 采购协同 / 招投标 / 合同管理 / 合同付款节点 / 供应商管理 |
| **执行端** | 到货质检 / 库存流转（入库 / 出库 / 调拨 / 退料 / 盘点）/ 设备管理 / 设备租赁 / 折旧 |
| **财务端** | NC 财务接口（13 类 BIZ）/ 凭证生成 / 三单匹配 / 付款申请 / 暂估闭环 |
| **管理端** | 报表预警（21 报表 + 14 类预警）/ 大屏（6 类看板）/ AI 助理（8 Tools）|
| **平台层** | 统一权限（51 角色） / 审批工作流（12 类模板） / 审计 / 后评价自动联动 |

业务链条贯通：**需求 → 采购 → 库存 → 财务 → 管理监督**，形成完整物资供应链闭环。

### 2.3 新名"阜矿物资供应协同管理平台"的优势

| 关键词 | 含义 |
|---|---|
| **阜矿** | 单位归属 / 品牌专属（非通用产品名）/ 易传达 |
| **物资供应** | 保留行业属性 / 与原招标语境衔接 |
| **协同** | 反映核心创新：跨角色 / 跨部门 / 跨系统协同（与 NC / Nova 等打通）|
| **管理** | 不只是流程执行 / 还含管理监督 / 预警 / 审批 / 审计 |
| **平台** | 反映扩展能力：30+ 单位推广 / AI Wave 1-3 / 集团化部署 / 平台化输出 |

## 三、双口径并存（避免合同变更）

### 3.1 口径分层

| 层 | 口径 | 用名 |
|---|---|---|
| **招标 / 合同** | 合同标的物 / 不动 | **阜矿物资供应管理系统**（11 个招标文档保持原称） |
| **详细设计** | 项目内技术文档 / 升版时机过渡 | 新版次（V1.2/V1.3 时机）切换为新名 |
| **上线 / Sprint** | 新文档用新名 / 老文档不动 | **阜矿物资供应协同管理平台**（本说明发布后新文档生效）|
| **对外汇报 / 演示 / 大屏** | 立即生效 ⭐ | **阜矿物资供应协同管理平台** / 简称 **供应协同平台** |
| **代码 namespace** | 已固化 / 不动 | `Nova.SupplyCores` |

### 3.2 为什么不一次性批量替换

- 招标技术要求 v1.1 是发标合同文件 / 改名等于改合同标的物 / 需要走变更流程
- 160 个文档批量替换工作量大 / 风险高 / 不符合"项目内先口径锁定"风格
- 业务方签字 + 集团认可后再启动"系统更名通知"流程

### 3.3 后续过渡时间窗（建议）

| 时间点 | 动作 |
|---|---|
| 2026-05-21 | 本说明发布 / 项目内口径锁定 |
| 2026-Q3 | 业务方 / 集团认可（与 NC 真联调里程碑同步）|
| 2026-Q4 | 启动"系统更名通知"集团变更流程 / 招标层文档历史归档 |
| 2027-Q1 | 全集团统一新名（与全量上线节奏一致）|
