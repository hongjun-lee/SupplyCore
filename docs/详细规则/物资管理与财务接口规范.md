# 物资管理系统与财务系统接口规范

**版本：** V1.5  
**日期：** 2026-03-27  
**适用范围：** 集团型煤炭企业物资供应管理系统与用友NC财务系统业务对接

> **文档定位：** 本文档为《05-财务与NC接口需求说明》的配套详细规则文档，主要用于细化凭证规则、会计分录、接口报文、对账与联调实施要求。需求边界、招标口径和管理层接口要求，以《05-财务与NC接口需求说明》及项目主文档为主。

## 一、概述

### 1.1 两系统定位与分工

| 维度 | 物资管理系统 | 用友NC财务系统 |
|------|------------|--------------|
| 核心职责 | 实物流转管理 | 资金价值核算 |
| 管理对象 | 物料、库存、出入库单据 | 凭证、科目、成本 |
| 数据粒度 | 细（按规格、批次、仓库） | 粗（按存货、科目、组织） |
| 驱动逻辑 | 业务驱动（有单据才有操作） | 凭证驱动（有凭证才记账） |
| 关系 | 业务发生方，数据源头 | 价值记录方，财务结果 |

> **核心原则：物资系统是业务的主人，财务系统是价值的镜子。**
> 物资每一笔实物流转，都应在财务留下对应的价值记录。

### 1.2 接口总体架构

```
物资管理系统
    │
    ├─ 主数据同步 ──────────→ 存货档案（N:1映射）
    │
    ├─ 合同/订单 ───────────→ 预付款登记
    ├─ 采购入库单 ──────────→ 暂估/正式入库凭证（三单匹配）
    ├─ 采购退货单 ──────────→ 红字入库冲销凭证
    ├─ 领料出库单 ──────────→ 费用/成本凭证
    ├─ 退料入库单 ──────────→ 红字费用冲销凭证
    ├─ 调拨单 ──────────────→ 内部往来/库存调整凭证
    ├─ 盘点单（盘盈）────────→ 盘盈凭证
    ├─ 盘点单（盘亏）────────→ 盘亏凭证
    ├─ 废旧处置单 ──────────→ 报废/变价收入凭证
    ├─ 质检不合格单 ─────────→ 退货或让步入账凭证
    └─ 月末结账 ────────────→ 存货计价、摊销、结转凭证
```

## 二、主数据对接

### 2.1 物料 ↔ 存货编码映射

采用N:1映射策略，详见《物资编码规范文档》第七章。

| 物资系统 | NC财务系统 | 说明 |
|---------|-----------|------|
| 物料编码（mat_code） | 存货编码（invcode） | N:1，多规格对应一存货 |
| 物料名称 | 存货名称 | 参考同步，NC为准 |
| 计量单位 | 计量单位 | 必须统一，见2.2 |
| 大类/分类 | 存货分类 | 按大类建立对应关系 |

### 2.2 计量单位统一

物资系统与NC须维护统一单位字典，禁止出现同一物料两系统单位不一致情况。

| 单位类型 | 常见单位 | 注意事项 |
|---------|---------|---------|
| 重量 | kg、t | 统一用kg或t，不混用 |
| 长度 | m、km | 电缆、风筒等 |
| 数量 | 根、个、套、台、顶 | 按物料特性定义 |
| 体积 | 桶、L | 油脂类 |
| 面积 | m² | 网片、防水材料 |

> 若换算不可避免（如采购单位与领用单位不同），在物料表维护**换算比例**字段，接口推送时自动转换。

### 2.4 主数据权威来源与同步边界

为避免后续接口联调时出现“谁是权威源”争议，建议在财务接口范围内明确以下边界：

| 数据类别 | 权威来源 | 使用原则 |
|---------|---------|---------|
| 集团组织架构、人员信息 | 人资系统 | 物资系统与 NC 原则上引用或同步使用，不应各自长期维护主档 |
| 财务核算组织、会计科目、成本中心 | 财务系统 / NC | 物资系统侧负责引用、映射和使用 |
| 物料编码 | 供应系统 | NC 侧维护映射关系，不反向主导物料编码 |
| 计量单位字典 | 集团统一字典；如暂无独立权威系统，则由物资与财务联合确认维护归口 | 两系统不得双向自由扩展 |

> **特别说明：** “集团组织架构”与“财务核算组织口径”不是同一概念。前者原则上以人资系统为准，后者以财务系统/NC 为准，两者之间通过映射关系衔接。

## 三、采购业务 → 财务凭证

### 3.1 预付款/订金处理

大宗物资采购签订合同后，可能需预付定金或首付款。

**预付款凭证（合同签订/付款时）：**

| 借方 | 贷方 |
|------|------|
| 1123 预付账款 | 1002 银行存款 |

**核销（发票到达、货物入库后）：**

| 借方 | 贷方 |
|------|------|
| 2202 应付账款 | 1123 预付账款（核销预付） |
| 1403 原材料 | 2202 应付账款 |

> 物资系统须记录合同号，推送NC时携带合同信息，财务据此核销预付款。

### 3.2 三单匹配（采购订单 + 入库单 + 发票）

三单匹配是采购付款的核心控制点，三单须一致才能付款。

| 匹配状态 | 处理方式 |
|---------|---------|
| 三单一致 | 正常付款，制正式入库凭证 |
| 入库 > 订单数量 | 超出部分暂存，通知采购确认是否补单 |
| 发票金额 > 订单金额 | 价差需审批，财务记录价差科目 |
| 发票先到、货未到 | 挂暂收票据，待货物入库后匹配 |
| 货到票未到（月末）| 暂估入账，见3.3 |

### 3.3 运费/装卸费归集

采购运费是存货成本的组成部分，处理方式：

**方式A（直接计入）：** 运费金额小，直接计入对应物料成本

| 借方 | 贷方 |
|------|------|
| 1403 原材料（含运费分摊） | 2202 应付账款（货款+运费合并） |

**方式B（单独核算）：** 运费金额大或多批物料共享，单独记录后分摊

| 借方 | 贷方 |
|------|------|
| 1403 原材料-待分摊运费 | 2202 应付账款 |

> 分摊基准：按重量或金额比例分摊至各物料，调整其移动平均成本。
> **已明确：** 专属运费直接计入对应物料成本；金额较大或多批共享的运费单独核算后统一分摊。

### 3.4 暂估入库凭证

**触发时机：** 月末，物资已入库但发票未到

| 借方 | 贷方 |
|------|------|
| 1403 原材料（暂估价） | 2181 应付暂估款 |

- 暂估价取**采购订单单价**，无订单取**上期移动平均价**
- 次月发票到达后，先红字冲销暂估，再正式入账

### 3.5 正式入库凭证

**触发时机：** 发票审核通过，与入库单匹配

| 借方 | 贷方 |
|------|------|
| 1403 原材料（发票含税价） | 2202 应付账款 |
| 2221 应交税费-进项税 | |

> 发票价与暂估价差异处理：
> - 差异 ≤5%：计入当期损益（管理费用-存货价差）
> - 差异 >5%：调整存货成本，财务审批后处理

### 3.6 供应商月末对账

每月末与供应商核对应付账款余额：
- 物资系统导出：本月采购入库汇总（含暂估）
- NC导出：应付账款明细（按供应商）
- 差异原因：发票时间差、退货未处理、价差未确认
- 对账完成后双方签字，作为付款依据

### 3.7 采购退货凭证

**触发时机：** 物资系统生成采购退货单（质量问题/数量溢出等）

| 借方 | 贷方 |
|------|------|
| 2202 应付账款（红字） | 1403 原材料（红字） |

## 四、领料出库 → 财务凭证

### 4.1 业务流程

```
使用单位提领料申请
    ↓
仓库审核，出库
    ↓
物资系统生成领料出库单（含成本中心/工程项目信息）
    ↓
推送NC → 生成费用/成本凭证
```

### 4.2 领料出库凭证

**触发时机：** 领料单审核通过，实物出库

| 借方 | 贷方 |
|------|------|
| 6602 生产成本-材料费（按成本中心） | 1403 原材料 |

- 出库单价采用**加权移动平均法**（月加权或永续加权，与NC一致）
- 领料单须携带**成本中心编码**，财务凭证据此归集费用
- 同一领料单涉及多成本中心时，凭证按成本中心分行

### 4.3 成本中心对照表

物资系统的"使用单位/用途"需与NC成本中心建立对应关系：

| 物资系统使用单位 | NC成本中心 | 说明 |
|----------------|-----------|------|
| 综采队 | CC-XX-001 | 采煤直接成本 |
| 掘进队 | CC-XX-002 | 掘进直接成本 |
| 机电科 | CC-XX-010 | 间接成本 |
| 通风区 | CC-XX-011 | 间接成本 |
| 行政部门 | CC-00-001 | 管理费用 |
| 安全部门（安全专项） | CC-SA-001 | 安全费专项 |

> 完整成本中心对照表由财务牵头、各矿厂联合制定，并在上线前完成固化。

### 4.4 煤矿特有成本归集

**掘进成本 vs 采煤成本区分：**

| 领料用途 | 成本归属 | 说明 |
|---------|---------|------|
| 采煤工作面材料 | 采煤直接成本 | 锚杆、液压件等 |
| 巷道掘进材料 | 掘进直接成本 | 支护材料为主 |
| 巷道维护材料 | 维护费用 or 资本化 | 见下方说明 |
| 设备维修配件 | 设备维修费 | 归入对应设备 |

**巷道维护材料资本化判断：**
- 日常维护、小修：费用化（计入当期成本）
- 大修、改造、延伸（增加了巷道服务年限或功能）：资本化
- 资本化标准金额由集团财务统一后在系统中按政策执行

### 4.5 退料入库凭证

**触发时机：** 使用单位退回未用完物料

| 借方 | 贷方 |
|------|------|
| 1403 原材料 | 6602 生产成本-材料费（红字冲销） |

- 退料价格取**原出库单价**，不重新计算
- 退料单须关联原领料单，财务冲销对应费用

## 五、库存调拨 → 财务凭证

### 5.1 同组织内跨仓调拨

同一核算组织内不同仓库之间的调拨，**不产生财务凭证**，仅物资系统内部库位变更。

### 5.2 跨组织调拨（矿与矿之间）

| 场景 | 凭证处理 |
|------|---------|
| 无偿调拨 | 调出方：贷1403；调入方：借1403；内部往来科目对冲 |
| 有偿调拨（内部结算） | 调出方走内部销售，调入方走内部采购，月末抵消 |

> 推荐使用无偿调拨，减少内部结算复杂度。跨组织调拨单须双方仓库签收确认。

## 六、质量管理 → 财务凭证

### 6.1 质检流程与财务节点

```
采购货物到货
    ↓
物资系统：入待检区（不入正式库存，不制凭证）
    ↓
质检结果
    ├─ 合格 → 正式入库 → 制入库凭证（见第三章）
    ├─ 部分合格 → 合格部分入库 + 不合格部分退货
    ├─ 让步接收 → 降价入库，差价冲抵应付账款
    └─ 全部不合格 → 退货，制退货凭证
```

### 6.2 让步接收降价入账

质量略有瑕疵但仍可使用，供应商同意降价：

| 借方 | 贷方 |
|------|------|
| 1403 原材料（降价后金额） | 2202 应付账款（降价后金额） |
| 2221 应交税费-进项税（按降价后） | |

> 降价协议须书面确认，附质检报告，财务留存备查。

### 6.3 质检不合格退货凭证

同采购退货处理，见3.7，附质检不合格报告。

### 6.4 合理损耗定额管理

部分物料（如炸药、油脂）在运输、储存过程中存在合理损耗：

- 物资系统维护各物料**损耗定额**（如炸药0.3%）
- 入库时系统自动计算理论损耗，在定额内的差异自动消化
- 超出定额的损耗单独列示，走盘亏流程处理（见第七章）

## 七、盘点业务 → 财务凭证

### 7.1 盘点流程

```
物资系统发起盘点任务
    ↓
仓库人员实盘，录入实盘数量
    ↓
系统计算盘盈/盘亏数量及金额（按移动平均成本）
    ↓
盘点审批（盘亏需附说明）
    ↓
推送NC → 生成盘盈/盘亏凭证
    ↓
财务审核，查明原因，月末结账前完成处理
```

### 7.2 盘盈凭证

| 借方 | 贷方 |
|------|------|
| 1403 原材料 | 6301 营业外收入-盘盈 |

> 火工品盘盈须上报，由安全部门配合核查原因，不得自行消化。

### 7.3 盘亏凭证

**正常损耗（自然损耗，在定额内）：**

| 借方 | 贷方 |
|------|------|
| 6601 管理费用-存货损失 | 1403 原材料 |

**非正常损耗（丢失/事故/超定额）：**

| 借方 | 贷方 |
|------|------|
| 6301 营业外支出-非常损失 | 1403 原材料 |

> 责任人赔偿部分：借1221其他应收款，贷6301营业外支出（冲回）。

### 7.4 呆滞库存减值

对长期未动库存（建议标准：超过12个月未出库）进行减值处理：

**识别：** 物资系统定期输出呆滞库存报表（按最后出库日期）

**财务处理：**

| 借方 | 贷方 |
|------|------|
| 6181 资产减值损失-存货跌价 | 1471 存货跌价准备 |

> 减值后物资系统库存数量不变，但账面价值降低。后续若实际出售价格高于账面，冲回减值准备。

## 八、废旧物资处置 → 财务凭证

### 8.1 废旧物资来源

- 设备维修拆下的旧配件
- 到期不可继续使用的物料（如过期炸药、老化电缆）
- 报废设备拆解后的零部件
- 生产过程中产生的废料（废钢、废旧包装）

### 8.2 物资系统处理

1. 发起废旧物资鉴定申请
2. 技术/安全部门鉴定（可修复/报废/降级使用）
3. 审批通过后转入废旧品仓
4. 处置方式：变卖/以旧换新/内部再利用/销毁

### 8.3 变卖废旧物资凭证

**出库（废旧物资出库）：**

| 借方 | 贷方 |
|------|------|
| 6301 营业外支出（账面净值） | 1403 原材料 |

**收入（变卖收款）：**

| 借方 | 贷方 |
|------|------|
| 1002 银行存款 | 6301 营业外收入（变卖价款） |

> 净额为正：营业外收入；净额为负：营业外支出。

### 8.4 以旧换新处理

供应商以旧换新（如旧截齿换新截齿）：
- 物资系统记录：旧件出库 + 新件入库（差价采购）
- 财务凭证：按差价金额处理采购入库，旧件出库按残值计入成本冲减

### 8.5 危险品销毁

过期炸药、雷管等危险品不可变卖，须按规程销毁：

## 十三、存货计价与月末结账

### 13.1 计价方法

| 方法 | 说明 | 适用场景 |
|------|------|---------|
| 加权移动平均法 | 每次入库后重新计算平均成本 | 推荐，物资与NC同步计算 |
| 先进先出法（FIFO） | 批次管理物料（HG/HX） | 须启用批次管理 |

> **两系统计价方法必须一致，否则月末对账必然出现差异。**

### 13.2 月末结账流程

```
25日：物资系统停止当月补录，数据封存
    ↓
26~28日：物资系统导出月度汇总，与NC核对
    ↓
差异处理（见第十四章）
    ↓
低值易耗品摊销确认
    ↓
暂估冲销与滚动暂估
    ↓
月末最后一天：NC完成存货科目结账
    ↓
次月1日：物资系统开放次月操作
```

### 13.3 暂估冲销（月末必做）

1. 红字冲销上月全部暂估凭证
2. 对尚未收票的入库单重新暂估（滚动暂估）
3. 已收票的在发票匹配时已正式入账，不再暂估

### 13.4 存货跌价准备（年末）

年末财务部门对库存物料进行减值测试：
- 物资系统提供各物料库存数量及上次采购单价
- 财务与市场价格对比，差异>20%的计提跌价准备
- 跌价准备在NC单独处理，不影响物资系统库存数量
- 呆滞物资（超12个月未动）重点关注

## 十四、对账与差异处理

### 14.1 日常对账

| 频率 | 对账内容 | 执行方 |
|------|---------|-------|
| 每日 | 当日出入库笔数及金额 vs NC当日凭证 | 系统自动比对 |
| 每周 | 本周物资库存余额 vs NC存货科目余额 | 物资+财务 |
| 每月 | 全量库存数量/金额对账，出具对账报告 | 物资+财务 |
| 每年 | 年末存货盘点配合财务审计 | 物资+财务+审计 |

### 14.2 年末审计配合

年末外部审计时，物资系统须配合提供：
- 年末库存清单（全部物料，按仓库）
- 年度出入库明细（可导出）
- 大额收发明细（审计抽查依据）
- 盘点记录及差异处理结果
- 废旧物资处置清单

审计人员可能到仓库实地抽盘，物资系统须支持按审计指定物料导出库存记录。

### 14.3 常见差异原因及处理

| 差异原因 | 处理方式 |
|---------|---------|
| 物资已入库，NC未推送 | 检查接口推送状态，补推 |
| 单价不一致（计价方法差异） | 核实两系统计价参数，统一后重算 |
| 单位换算错误 | 核查换算比例，修正后补差额凭证 |
| 暂估未冲销 | 月末执行暂估冲销流程 |
| 手工凭证未同步 | 财务手工凭证须通知物资系统备注 |
| 跨月调整 | 需跨月补单，双方签字确认 |
| 寄售/甲供混入正式库存 | 核查仓别，调整至正确仓库 |

### 14.4 跨期单据处理

业务已发生但当月未录入（跨期）的处理原则：
- 原则上不允许跨期补录，须在当月完成
- 确需跨期的，须填写跨期申请，财务审批
- 财务NC侧需对应开放已封账期间（谨慎操作）
- 跨期单据在系统中标注，便于审计追溯

### 14.5 日对账与封账SOP（骨架）

建议至少明确以下日常与月末操作责任：

| 环节 | 最低要求 | 责任方建议 |
|------|----------|-----------|

| 每日差异处理 | 对主数据缺失、金额差异、状态不一致逐条处理 | 物资 + 财务 |
| 月末对账前检查 | 未映射、未推送、失败重推、暂估清单、盘盈盘亏、NC手工调整台账全部核对 | 物资 + 财务 |
| 月结确认 | 对账完成后由双方确认是否封账 | 物资负责人 + 财务负责人 |

## 十六（补）：单价精度与尾差处理

> **背景：** 移动平均计价在除法运算中必然产生精度损失，两系统若小数位规则不一致，月末对账时尾差会累积到数百甚至数千元。这是业财对账差异的第一大来源。

### 16A.1 小数位统一规则

| 数据项 | 建议精度 | 说明 |
|--------|---------|------|
| 移动平均单价 | **6位小数** | 中间计算保留6位，避免过早截断 |
| 入库/出库金额 | **2位小数** | 最终金额按2位小数四舍五入 |
| 数量 | **4位小数** | 适用于按重量（kg/t）计量的物料 |
| 税率 | 百分比整数（如13%） | 税额 = 不含税金额 × 税率，四舍五入到分 |

> **关键约束：** 物资系统与NC的移动平均单价精度、金额舍入规则必须完全一致。建议在实施阶段双方开发商联合确认并出具精度对照表。

### 16A.2 尾差产生场景

**场景一：出库尾差**

```
库存100根，移动平均单价 13.273333元
出库53根：53 × 13.273333 = 703.486649 → 四舍五入 703.49元
剩余47根：账面应为 100×13.273333 - 703.49 = 623.843300
实际 47×13.273333 = 623.846651
尾差 = 0.003351元（单次微小，但月累积可观）
```

**场景二：多次入库不同价格后出库**

```
第1次入库：50根 × 12.00 = 600.00
第2次入库：50根 × 14.00 = 700.00
移动平均单价 = 1300.00 / 100 = 13.00
出库100根：100 × 13.00 = 1300.00（恰好无尾差）

但如果第2次入库是30根 × 14.00：
移动平均 = (600+420) / 80 = 12.75
出库37根：37 × 12.75 = 471.75
剩余43根账面 = 1020 - 471.75 = 548.25
43 × 12.75 = 548.25（恰好无差）
但换其他数字组合就会出现尾差
```

### 16A.3 尾差处理规则

| 规则 | 说明 |
|------|------|
| **尾差容忍阈值** | 单笔出库尾差 ≤ 0.01元自动消化，不单独处理 |
| **月度累计尾差阈值** | 月累计尾差 ≤ 50元，自动计入管理费用-存货尾差科目 |
| **超阈值处理** | 月累计尾差 > 50元，须查明原因（精度不一致 or 计算错误） |
| **尾差科目** | 借/贷：6601 管理费用-存货价差（尾差调整） |
| **最后一笔出库规则** | 某物料库存清零时，最后一笔出库金额 = 账面剩余金额（倒挤），不按单价×数量计算，避免残余金额 |

### 16A.4 尾差对账要求

- 月末对账时，物资系统与NC按nc_inv_code汇总对比金额
- 差异 ≤ 50元且可归因于精度尾差的，双方确认后自动消化
- 差异 > 50元的须逐笔排查
- 对账报告中须单独列示"尾差调整"金额

> **已明确：** 尾差容忍阈值按 50 元执行，尾差调整按财务统一科目编码处理。

## 十六（补）：移动平均价异常场景处理

### 16B.1 零库存入库

当某物料库存数量和金额均为零时，新入库直接以入库单价作为新的移动平均价。

```
入库前：数量=0，金额=0，均价=0
入库：100根 × 15.00
入库后：数量=100，金额=1500，均价=15.00
```

无特殊处理，正常逻辑。

### 16B.2 零库存退料入库

使用单位退回物料时，当前库存恰好为零：

```
入库前：数量=0，金额=0，均价=0
退料入库：20根，原出库单价 13.27
入库后：数量=20，金额=265.40，均价=13.27
```

**规则：** 退料入库按原出库单价计算，移动平均以退料价重建。

### 16B.3 是否允许负库存

| 策略 | 说明 | 建议 |
|------|------|------|
| **禁止负库存（推荐）** | 出库时检查可用库存，不足则拒绝 | 一期推荐，避免移动平均价异常 |
| 允许负库存 | 出库可透支，后续入库补回 | 负库存下移动平均无意义，会导致严重对账问题 |

> **建议：** 一期系统层面禁止负库存。如因业务时序问题（先出库后补入库），通过在途或预留机制处理，不直接允许负库存出库。

### 16B.4 零成本入库（赠品/样品）

供应商赠送的物料，入库金额为0：

```
入库前：数量=50，金额=1000，均价=20.00
赠品入库：10根 × 0.00
入库后：数量=60，金额=1000，均价=16.666667
```

**问题：** 赠品会拉低移动平均，影响后续出库成本准确性。

**建议处理方式：**

- 赠品/样品单独建"赠品入库"单据类型
- 入库金额按市场参考价或最近一次采购价录入（不能为0）
- 如确实无法定价，单独入"赠品仓"管理，不参与正式库存移动平均计算
- 领用赠品时按实际录入价出库

> **已明确：** 赠品入库按市场参考价或最近一次采购价录入；确实无法定价时单独入赠品仓管理。

### 16B.5 移动平均价保护机制

为防止异常入库导致均价剧烈波动，建议设置：

| 机制 | 说明 |
|------|------|
| **价格偏离告警** | 入库单价偏离当前移动平均 ≥30% 时，系统弹出告警，需人工确认 |
| **价格上限保护** | 入库单价超过当前移动平均 200% 时，系统拦截，须审批后放行 |
| **零价拦截** | 入库单价为0时，系统拦截，须选择"赠品入库"类型 |

> 以上阈值（30%/200%）为建议值，由财务和物资部门确认。

## 十六（补）：部分到票与发票匹配细化规则

### 16C.1 实际中的发票匹配复杂场景

| 场景 | 说明 | 发生频率 |
|------|------|---------|
| 一票一入库 | 一张发票对应一张入库单 | 理想情况，约30% |
| 一票多入库 | 一张发票覆盖多次入库 | 常见，约40% |
| 多票一入库 | 一次入库分多张发票 | 偶发，约10% |
| 一票多订单 | 一张发票跨多个采购订单 | 偶发，约10% |
| 部分开票 | 供应商只开了部分金额 | 偶发，约10% |

### 16C.2 匹配策略

**推荐方案：按供应商+物料+时间范围模糊匹配**

```
匹配优先级：
1. 精确匹配：发票行 与 入库单行 按订单号+物料编码 一对一匹配
2. 汇总匹配：同一供应商、同一NC存货编码、同一月份的入库合并与发票合并后匹配
3. 手工匹配：以上均无法自动匹配时，人工指定发票与入库单的对应关系
```

### 16C.3 匹配差异容忍度

| 差异类型 | 容忍度 | 处理方式 |
|---------|--------|---------|
| 数量差异 | 0（必须精确） | 不允许发票数量与入库数量不一致 |
| 单价差异 ≤ 5% | 自动通过 | 差异计入当期损益 |
| 单价差异 > 5% | 需审批 | 触发价差审批流程 |
| 金额差异 ≤ 0.1元 | 自动消化 | 精度尾差 |
| 税额差异 ≤ 0.01元 | 自动消化 | 税额计算精度差异 |

### 16C.4 部分到票处理

一次入库1000根，供应商先开了600根的发票：

```
步骤1：600根按发票正式入账
  → 推送正式入库凭证 BIZ-001（600根）
  
步骤2：剩余400根月末未到票
  → 推送暂估凭证 BIZ-002（400根）
  
步骤3：次月剩余400根发票到达
  → 冲销400根暂估 BIZ-003
  → 推送正式入库 BIZ-001（400根）
```

**入库单状态流转：**

```
已入库(全量) → 部分到票(600/1000) → 部分暂估(400) → 全量到票 → 完结
```

## 十六（补）：价格调整与补差处理

### 16D.1 常见价格调整场景

| 场景 | 说明 | 凭证影响 |
|------|------|---------|
| 合同调价 | 合同约定按季/年调整单价，已入库物资需追溯补差 | 补差凭证 |
| 发票换票 | 供应商原发票有误，开红字发票后重开 | 红字冲销 + 新蓝字入账 |
| 年终返利/折让 | 供应商按年度采购量给予返利 | 冲减采购成本或营业外收入 |
| 供应商罚款扣款 | 质量/交期罚款从应付中扣除 | 冲减应付，差额计入营业外收入 |

### 16D.2 合同调价补差凭证

**场景：** 本季度锚杆单价从28元调至30元，本季已入库5000根需补差。

补差金额 = 5000 × (30-28) = 10,000元

| 借方 | 贷方 |
|------|------|
| 1403 原材料-支护材料 10,000.00 | 2202 应付账款 10,000.00 |

**系统处理：**

- 物资系统生成"价格调整单"，关联原入库单
- 补差后调整受影响物料的移动平均价
- 推送NC补差凭证

**是否调整历史移动平均：**

| 策略 | 说明 | 建议 |
|------|------|------|
| 追溯调整 | 回溯重算所有受影响的入库和出库 | 计算量大，一般不推荐 |
| **仅调整当前均价（推荐）** | 补差金额计入当前库存成本，重新计算当前移动平均 | 简单可行 |

### 16D.3 发票换票处理

```
步骤1：供应商开红字发票（冲原蓝字）
  → 物资系统登记红字发票
  → 推送NC红字冲销凭证
  
步骤2：供应商开新蓝字发票
  → 物资系统登记新发票并与入库单重新匹配
  → 推送NC正式入库凭证（按新价格）
  
步骤3：如跨月
  → 红字发票冲销在当月处理
  → 新蓝字发票按到达月份入账
  → 期间差异按暂估逻辑补齐
```

### 16D.4 年终返利/折让

| 返利方式 | 凭证处理 |
|---------|---------|
| 供应商开红字发票 | 冲减应付账款和原材料成本 |
| 供应商直接付款返利 | 借：银行存款；贷：营业外收入/冲减采购成本 |
| 抵扣下年采购款 | 挂预付账款或其他应收款 |

> **已明确：** 返利按财务统一政策优先冲减采购成本或存货成本，并在系统中统一留痕。

### 16D.5 供应商扣款

从应付款中扣除质量罚款或延期罚款：

| 借方 | 贷方 |
|------|------|
| 2202 应付账款（扣款金额） | 6301 营业外收入-供应商罚款 |

物资系统须支持在付款申请中标注扣款金额和原因，推送NC时体现在应付核销凭证中。

## 十六（补）：NC侧反向操作处理

### 16E.1 NC侧可能发生的操作

| 操作 | 场景 | 对物资系统的影响 |
|------|------|----------------|
| NC手工调整凭证 | 财务发现科目挂错，手工调账 | 物资侧金额与NC不一致 |
| NC删除/作废凭证 | 财务作废错误凭证 | 物资侧已推送成功但NC侧不存在 |
| NC反结账 | 财务需要修正已结账期间 | 物资侧已锁定期间 |
| NC年末调整 | 年末审计调整、汇率调整 | 影响存货余额但物资侧无对应记录 |

### 16E.2 处理原则

1. **NC侧不得在不通知物资系统的情况下修改物资推送的凭证**
2. 如确需修改，须通过以下流程：
   - NC侧提出调整需求
   - 物资系统确认是否需要同步调整业务单据
   - 如需调整：物资侧修改 → 重新推送 → NC侧处理
   - 如不需调整：NC侧自行调整 → 记入"手工凭证台账" → 月末对账时标注
3. 建议在NC侧设置：**物资系统推送的凭证默认锁定，NC操作员不可直接修改**

### 16E.3 手工凭证台账

对NC侧所有非物资系统推送的手工凭证（含调整、更正），须维护台账：

| 字段 | 说明 |
|------|------|
| 调整日期 | NC侧操作日期 |
| 原凭证号 | 被调整的原凭证 |
| 调整原因 | 科目错误/金额错误/审计要求等 |
| 调整金额 | 借方/贷方调整额 |
| 操作人 | NC侧操作人 |
| 是否通知物资系统 | 是/否 |
| 物资系统处理结果 | 已同步/无需同步/待处理 |

月末对账时，手工凭证台账须与物资系统导出数据联合比对，排除手工调整导致的"假差异"。

## 十六（补）：增值税税率差异处理

### 16H.1 税率变化场景

| 场景 | 说明 |
|------|------|
| 签订合同时税率13%，发票到达时税率调整为12% | 政策调整 |
| 暂估含税 vs 不含税 | 暂估基准选择 |
| 不同物料适用不同税率 | 部分物料免税或低税率 |

### 16H.2 暂估含税规则

**建议方案：暂估按不含税金额入账**

理由：
- 暂估时发票未到，无法确认最终税率和税额
- 不含税暂估避免了税率变化导致的暂估冲销差异
- 与NC存货科目口径一致（存货成本不含税）

```
暂估入账：
  借：1403 原材料（不含税金额）
  贷：2181 应付暂估款（不含税金额）
  
正式入账：
  借：1403 原材料（不含税金额）
  借：2221 进项税（按实际税率）
  贷：2202 应付账款（含税总额）
```

### 16H.3 税率变化差异处理

如合同约定含税总价150,000元（税率13%）：
- 签订时：不含税 132,743.36，税额 17,256.64
- 发票到达时税率调为12%：不含税 133,928.57，税额 16,071.43

差异：
- 不含税成本增加 1,185.21（入库成本增加）
- 税额减少 1,185.21

**处理原则：** 按实际发票金额入账，差异体现在存货成本和进项税中。暂估冲销时按原暂估金额冲销，正式入账按实际发票金额，差异自然消化。

> **已明确：** 合同价格应同时记录含税价和不含税价，财务处理以不含税金额为主口径。

## 十六（补）：上线切换方案细化

### 16I.1 切换策略选择

| 策略 | 说明 | 风险 | 建议 |
|------|------|------|------|
| **月初切换（推荐）** | 选定某月1日作为新系统启用日，上月末在老系统完成结账 | 风险最低，边界清晰 | ✅ 推荐 |

| 双轨运行 | 新旧系统同时运行1~2个月 | 工作量翻倍，但安全 | 可选（首月） |

### 16I.2 月初切换详细流程

```
T-30天：数据准备阶段
  ├─ 完成物料主数据清洗与导入
  ├─ 完成NC映射建立与校验
  ├─ 完成成本中心/组织/仓库/供应商档案同步
  └─ 完成系统配置（审批流、权限、参数）

  ├─ 老系统完成上月月结
  ├─ 导出老系统期末库存余额（按物料+仓库+批次）
  ├─ 导出NC存货科目期末余额
  └─ 核对老系统库存 vs NC存货余额（必须一致）

T-1天：切换前夜
  ├─ 老系统最后一次月结确认
  ├─ 冻结老系统，禁止新增业务单据
  ├─ 导出最终期初库存数据
  └─ 准备新系统期初导入文件

T日（月初第1天）：正式切换
  ├─ 新系统导入期初库存
  ├─ 期初库存校验：新系统库存 = 老系统期末 = NC余额
  ├─ 新系统开放业务操作
  ├─ 老系统设为只读（保留查询历史）
  └─ 通知所有用户切换完成

T+1至T+30天：首月护航
  ├─ 每日检查接口推送状态
  ├─ 每周与NC对账
  ├─ 首月末执行完整月结流程
  ├─ 首月末对账结果确认
  └─ 首月运行报告输出
```

### 16I.3 期初数据来源

| 数据项 | 来源 | 校验方式 |
|--------|------|---------|
| 物料主数据 | 新系统自有（已治理导入） | 与老系统物料数对比 |
| 期初库存数量 | **老系统期末库存** | 与实物盘点核对 |
| 期初库存金额 | **NC存货科目期末余额** | 以NC为准（财务权威） |
| 移动平均单价 | NC余额 / 老系统数量 | 计算得出，两边校验 |
| 暂估挂账 | 老系统暂估清单 | 与NC暂估余额核对 |
| 应付余额 | NC应付科目 | 不导入物资系统，仅核对 |

> **关键原则：** 期初库存金额以NC为权威来源。如老系统库存金额与NC存在差异，以NC为准，差异在切换前处理完毕。

### 16I.4 上线月暂估处理

**场景：** 老系统中有一批暂估，切换到新系统后如何处理？

```
方案A（推荐）：
  1. 老系统月末正常执行暂估冲销（冲掉老暂估）
  2. 新系统导入期初时，对仍未到票的入库单标记"待暂估"
  3. 新系统首月月末对这批单据重新暂估
  → 暂估链在新系统重新开始，干净清晰

方案B（备选）：
  1. 老系统暂估不冲销，直接带入新系统
  2. 新系统首月末冲销老暂估 + 重新暂估
  → 需要迁移暂估明细，增加复杂度
```

### 16I.5 首月双轨运行建议

| 操作 | 老系统 | 新系统 |
|------|--------|--------|
| 新增业务 | 禁止 | ✅ 在新系统操作 |
| 查询历史 | ✅ 只读可查 | 仅可查期初后数据 |
| NC推送 | 停止推送 | ✅ 新系统推送 |
| 月末对账 | 不参与 | ✅ 新系统与NC对账 |
| 报表 | 可查历史 | 可查首月数据 |

### 16I.6 切换风险与应对

| 风险 | 应对措施 |
|------|---------|
| 期初数据导入错误 | 导入后全量校验，发现问题回滚重导 |
| 首月接口大面积失败 | 保留手工补推能力，驻场团队现场处理 |
| 用户不熟悉新系统 | 首月安排现场培训 + 驻场支持 |
| 首月月结不顺 | 提前在测试环境完整演练月结流程 |
| NC余额与老系统不一致 | **切换前必须解决**，不带差异上线 |

### F.4 异常场景补充

**场景A：发票一直不来怎么办？**

每个月末重复执行"冲销+重新暂估"（滚动暂估），直到发票到达：

```
4月1日: 冲销3月暂估
4月30日: 发票仍未到 → 重新暂估（暂估批次=2026-04）
5月1日: 冲销4月暂估
5月15日: 发票终于到了 → 正式入账
```

**场景B：发票价差 > 5% 怎么办？**

```
订单价 28元，发票价 32元，差异率 14.3% > 5%
  → 不自动处理，触发价差审批流程
  → 审批通过后按实际发票价调整存货成本
  → 差额 (32-28)×500 = 2000元 调整存货
  → 推送NC调整凭证
```

**场景C：到货数量与订单不一致怎么办？**

```
订单500根，实到520根
  → 超出20根暂存待处理
  → 500根正常入库暂估
  → 20根通知采购：补单 or 退回
  → 补单后20根追加入库
  → 退回则不影响
```
