# 给 NC 团队的对接需求清单 V0.2

**文档目的：** SupplyCores（阜矿物资管理系统，辽宁能源集团）需要与用友 NC（集团财务总账系统）做**主数据、统一凭证接入、回执/对账**三类能力对接。本清单不是要求 NC 团队逐项实现物资侧 20 个“制证功能”，而是要求 NC 提供可承接物资侧凭证数据的**统一凭证接入能力**（或 NC 现有接口平台的等价能力），并确认凭证模板、报文字段、回执、错误码和联调边界。物资侧内部仍可按详设 [`08-财务与NC接口详细设计-V1.1.md`](08-财务与NC接口详细设计-V1.1.md) §五管理 BIZ-001~BIZ-020 业务场景；对 NC 发函时按“能力 + 场景范围”表达，避免把内部业务拆分误写成 NC 必须交付的 20 个独立 endpoint。回函后联动详设 08 升 V1.2 + F-14 `interface_definition` 预置数据落库。

**回复 SLA 期望：** T+10 工作日。NC 项目尚在不同就绪阶段（详设 08 §4.3 三段式），允许分阶段回函；最迟首笔正式凭证上线前 30 天前完成对照（即详设 08 §4.3 "NC 正式启用"前 30 天）。

**优先级口径：** P0 = 联调启动前必需（包括首批 MD-001 / MD-003 + **统一凭证接入能力** + 采购入库 / 领料出库 / 对厂矿销售出库等样例模板）；P1 = UAT 前补齐主要凭证模板和回执/对账；P2 = 正式上线前补齐低频模板；P3 = 上线后可补。本清单按章节标记。

## 零、对接定位与沟通口径

本次与 NC 团队沟通的定位是：**以物资管理业务为切入点，确定 NC 需要开放 / 配合的接口功能边界**。

换句话说，物资系统负责梳理采购、入库、出库、调拨、暂估、付款等物资业务在什么节点需要形成财务凭证、同步主数据、获取回执和开展对账；NC 团队负责确认其接口平台能否承接这些数据、需要哪些字段、采用什么凭证模板、如何返回凭证号 / 记账状态 / 错误码。

对外沟通时应避免以下表述：

- ❌ “请 NC 团队实现 20 个物资制证功能”
- ❌ “请 NC 团队理解并落地全部物资业务流程”
- ❌ “请 NC 按物资侧 BIZ-001~BIZ-020 拆 20 个独立接口”

推荐表达为：

- ✅ “请 NC 团队确认统一凭证接入能力”
- ✅ “请 NC 团队确认主数据、凭证接入、回执状态、对账查询等接口功能”
- ✅ “物资侧 BIZ 场景仅作为凭证模板覆盖范围和联调优先级依据”

因此，本清单的核心不是列物资系统内部所有制证模块，而是与 NC 确认以下接口功能：

1. **主数据接口功能**：物料、计量单位、成本中心、核算组织等如何同步 / 查询；
2. **统一凭证接入功能**：物资侧生成的凭证请求如何推送到 NC，NC 如何按模板 / 交易类型制证；
3. **凭证回执与状态查询功能**：NC 如何返回凭证号、记账状态、失败原因，是否支持异步回调或轮询；
4. **对账与期间控制功能**：日 / 周 / 月对账、期间状态、补推、反结等接口边界；
5. **鉴权、幂等、错误码和联调机制**：如何保证接口可用、可重推、可追溯、可验收。

## 三、NC 必须确认 / 提供的能力与接口清单（核心，P0~P2）

### 3.1 统一凭证接入能力（P0~P2）

> **写法原则：** 对 NC 团队不写“请实现 20 个制证接口 / 20 个制证功能”。物资系统内部可以按 BIZ-001~BIZ-020 管理不同业务场景，但对 NC 的正式需求应表述为：**请提供统一凭证接入能力，能够接收物资侧按约定模板生成的凭证请求，并返回凭证号、记账状态、错误码和回执。**

#### 3.1.1 NC 需要确认 / 提供的能力

| 能力项 | 需求描述 | 优先级 |
|--------|----------|--------|
| 统一凭证接收 | NC 是否支持一个统一凭证接入 endpoint（如 `POST /nc/vouchers`），由 `voucher_template_code` / `business_scene_code` / `transaction_type` 区分采购入库、领料出库、销售出库等场景；如 NC 只能按模板拆 endpoint，也请说明拆分规则 | **P0** |
| 凭证模板 / 交易类型 | 提供首批凭证模板编码、名称、适用场景、借贷科目规则、辅助核算维度要求 | **P0** |
| 报文字段 | 明确统一凭证请求字段：来源系统、来源单号、业务场景编码、核算组织、期间、币种、金额、税额、摘要、行明细、辅助核算、附件 / 追溯字段等 | **P0** |
| 回执机制 | 返回或回调 `nc_voucher_no`、`task_id`、`finance_state`、`accounted_time`、失败原因；支持同步返回或异步回执 / 轮询 | **P0** |
| 红冲 / 冲销 | 明确红冲走同一接口 + `reverse_flag`，还是单独红冲能力；明确原凭证号引用规则 | P1 |
| 幂等与重推 | 支持按物资侧 `idempotent_key` / 来源单号去重；明确重复推送、失败重推、已记账后重复推送的处理规则 | **P0** |
| 错误码 | 返回业务校验错误（科目不存在 / 期间关闭 / 辅助核算缺失）、系统错误、权限错误、重复请求错误等错误码 | **P0** |
| 批量能力 | 是否支持一批多张凭证 / 一张凭证多行明细；单次报文大小、行数、QPS、并发上限 | P1 |
| 附件与追溯 | 是否支持来源单据 URL / 附件 ID / 业务摘要 / 物资明细追溯字段写入 NC 凭证附件或摘要 | P2 |

### 3.2 主数据类（MD-001 ~ MD-005，共 5 个，P0~P1）

| 接口编码 | 方向 | NC 必须实现 | 期望路径 | 优先级 |
|---------|------|----------|---------|--------|
| MD-001 | 物资 → NC | **接收端**：物料/存货档案推送 + 启停状态 | `POST /nc/masterdata/material` | **P0** |
| MD-002 | 物资 → NC | **接收端**：物料停用通知 + NC 端阻塞规则（如有未完结业务单据时 NC 端返回阻塞错误码） | `POST /nc/masterdata/material/disable` | P1 |
| MD-003 | 双向 | **读 + 写**：计量单位字典；明确以谁为源 | `GET/POST /nc/masterdata/unit` | **P0** |
| MD-004 | NC → 物资 | **NC 提供查询 endpoint**：成本中心全量 + 增量（`?since=yyyy-MM-ddTHH:mm:ss`）；支持订阅推送更佳 | `GET /nc/masterdata/cost-center` | P1 |
| MD-005 | NC → 物资 | **NC 提供查询 endpoint**：核算组织树 + 与集团二级组织（阜矿子树）映射 | `GET /nc/masterdata/accounting-org` | **P0** |

**MD 类必须回函的字段：**
- 各档案的 NC 主键字段（`material_id` / `cost_center_id` / `accounting_org_id`）+ 编码字段 + 名称字段 + 状态字段

- NC 主数据的码表（如物料类别、计量单位类型等）

### 3.3 回执 / 对账类（F-03 回执 + CHK-001 ~ CHK-004，共 5 个，P0~P2）

NC 必须实现以下接口的**任一种**（NC 端能力决定选择）：

| 接口 | 选项 A（推荐） | 选项 B（兜底） |
|------|-------------|-------------|
| F-03 凭证回执 | NC 主动回调物资侧 webhook：`POST {supplycore-base}/nc-callback/voucher-receipt` | NC 提供查询接口让物资轮询：`GET /nc/voucher/{task_id}/receipt` |

| CHK-003 月末全量对账 | — | NC 提供查询：`GET /nc/reconcile/month?period=yyyy-MM&org=___` |
| CHK-004 状态查询 | — | NC 提供查询：`GET /nc/voucher/query?task_id=___` 或 `?nc_voucher_no=___` 或 `?source_bill_no=___` |

**回函必须明确：**
- F-03 回执选 A 还是 B；选 A 时物资侧暴露的 webhook 路径会另发；选 B 时物资侧的轮询频率（建议 30 秒一次）

### 3.4 期间 / 反结（F-10 + F-11，共 2 个，P1）

|------|----------|---------|--------|

## 六、错误码与异常处理（P0）

### 6.1 错误码分层

NC 必须按以下三类返回错误码（详设 08 §F-08 `exception_record` 分类对齐）：

| 类型 | 含义 | 物资侧处理 | NC 必须给出码表 |
|------|------|----------|--------------|
| 业务校验失败 | 科目不存在、组织未启用、期间已封账等 | 写 F-08 异常台账，不自动重推；待人工处理 | ___ |
| 系统暂时不可用 | NC 实例宕机、网关超时、限流 | 物资按 §4.3 退避自动重推 | ___ |
| 数据冲突 | 幂等键已存在但报文不同 | 阻断（详设 08 §十一 `NC_IDEMPOTENT_CONFLICT_BLOCK=true`） | ___ |

### 6.2 退回 / 冲销

| 场景 | NC 端流程 | 物资侧需要的信息 |
|------|---------|---------------|
| NC 端凭证退回 | NC 通过 F-03 回执通知 `finance_state=已退回` | 退回原因（自由文本 + 错误码）+ 期望物资侧动作（红冲 / 修正 / 关闭） |
| NC 端冲销凭证 | NC 自行冲销后，物资侧如何感知？ | 是否通过 F-03 推送冲销事件？还是只能物资侧拉 CHK-001/CHK-004？ |

### 6.3 重复推送

- 物资侧因网络抖动可能对同一业务单据推送多次。NC 端**必须**按 `idempotent_key`（详设 08 §7.2 规则）去重，重复推送返回首次的凭证号 + 标记 `idempotent=true`。
