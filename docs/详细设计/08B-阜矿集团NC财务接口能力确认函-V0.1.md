# 阜矿集团 NC 财务接口能力确认函 V0.1

**事项：** 阜矿集团业务系统与 NC 财务系统接口能力确认  
**发文对象：** 用友 NC 项目组 / NC 财务系统接口集成团队  
**文档性质：** 对外发文单 / 接口功能确认函  
**日期：** 2026-05-13  
**编制依据：** 阜矿集团各业务系统（含物资系统、人资系统、销售系统等）与 NC 财务系统对接需求梳理成果

---

## 一、发文目的

阜矿集团拟推进各业务系统（含物资系统、人资系统、销售系统等）与用友 NC 财务系统进行接口对接，并希望尽量以**统一、通用的接口能力**承接各业务系统的财务交互需求。本函代表阜矿集团向贵方协商确认 NC 财务侧可开放、可承接或需配合建设的通用接口能力边界，为各业务系统联调、测试和上线排期提供共同依据。

本函是阜矿集团对 NC 财务接口能力的通用性确认要求，**不针对任何单一业务系统**。集团下属物资、人资、销售等业务系统将以"统一报文 + 凭证模板"的方式接入 NC，由 NC 完成制证与账务处理。请贵方重点确认 NC 侧是否具备以下接口能力、需要哪些报文字段、采用什么凭证模板、如何返回凭证号 / 状态 / 错误信息。

---

## 二、请贵方确认的接口功能

### 2.1 主数据接口能力

请确认 NC 是否支持以下主数据查询 / 接收能力：

1. 物料 / 存货档案接收或同步；
2. 计量单位字典查询或同步；
3. 成本中心查询；
4. 核算组织 / 账套组织查询；
5. 主数据是否支持增量查询、启停状态同步和变更时间戳。

请提供可用接口路径、字段清单、主键 / 编码字段、增量条件和测试环境说明。

### 2.2 统一凭证接入能力

请确认 NC 是否支持统一凭证接入能力，即阜矿集团相关业务系统按约定报文推送凭证数据，NC 按凭证模板、交易类型或业务场景编码完成制证。

需贵方确认：

1. 是否支持统一凭证接收接口；如不支持，需按哪些凭证模板或交易类型拆分；
2. 凭证模板编码、名称、适用场景、借贷科目规则；
3. 请求报文字段：来源系统、来源单号、核算组织、期间、摘要、金额、行明细、辅助核算等；
4. 红冲 / 冲销如何处理；
5. 是否支持幂等键，重复推送如何返回；
6. 是否支持批量凭证或一张凭证多行分录。

典型场景仅作模板覆盖范围举例，**跨业务系统**包括：

- **物资类**：采购入库、领料出库、对厂矿销售出库、火工品出入库、采购暂估 / 红冲、预付款 / 付款执行；
- **人资类**：薪资发放、社保 / 公积金 / 个税计提、福利费用、人工成本归集；
- **销售类**：销售开票、销售回款、坏账计提、退货红冲、客户预收款。

上述场景不要求 NC 按各业务系统内部编码逐项拆分独立接口，**重点是确认是否可通过统一凭证接入能力按凭证模板 / 交易类型完成制证**。

### 2.3 凭证回执与状态查询能力

请确认 NC 对凭证处理结果的返回方式：

1. 同步返回凭证号，还是异步回调 / 查询；
2. 是否返回 NC 凭证号、任务号、记账状态、记账时间；
3. 失败时是否返回错误码、错误原因和处理建议；
4. 是否支持按来源单号、任务号、NC 凭证号查询处理状态。

### 2.4 应收 / 应付等账务明细只读查询能力

阜矿集团相关业务系统需要进行业务对账、付款 / 收款状态核验和凭证穿透查询。请确认 NC 是否提供正式只读查询接口，而非数据库直连。

需确认的查询能力包括：

1. 应付明细账查询：供应商 / 员工 / 内部单位等往来对象的应付发生、付款核销、余额；
2. 应收明细账查询：客户 / 厂矿 / 内部单位等往来对象的应收发生、收款核销、余额；
3. 凭证明细 / 分录查询：按 NC 凭证号、来源单号或期间查询分录；
4. 往来余额查询：按供应商 / 客户 / 员工 / 核算组织 / 期间查询余额；
5. 核销 / 结算状态查询：某笔应付 / 应收是否已付款、已收款、已核销及剩余余额。

请提供查询条件、返回字段、分页方式、权限范围、最大查询时间跨度和接口限流要求。

### 2.5 对账与期间控制能力

请确认 NC 是否支持：

1. 日 / 月对账数据查询；
2. 凭证状态批量查询；
3. 当前会计期间状态查询；
4. 已结账期间是否允许补推；
5. 反结是否支持接口申请，或只能在 NC 系统内人工处理。

### 2.6 接口基础条件

请一并确认以下基础条件：

1. UAT / 生产环境 baseURL；
2. 鉴权方式：OAuth2、API Key + 签名、mTLS 或其他；
3. IP 白名单、网络通道和证书要求；
4. 请求超时、QPS、并发、报文大小限制；
5. 错误码分类和码表；
6. 接口日志保留周期；
7. 联调负责人、联系方式和应急响应机制；
8. 接口变更提前公告周期。

## 三、建议回函材料

请贵方尽量按以下材料回函：

1. 当前 NC 项目阶段说明：账套、组织、科目、接口平台是否已具备联调条件；
2. 可开放接口清单：接口名称、路径、方法、用途、优先级；
3. 凭证模板清单：模板编码、适用场景、科目规则、辅助核算要求；
4. 主数据接口字段说明；
5. 凭证接入请求 / 响应报文样例；
6. 回执、状态查询、账务明细查询、对账接口说明；
7. 鉴权、错误码、幂等、限流、日志和联调安排；
8. Swagger / OpenAPI / Postman 文档（如已有）。

---

## 四、回复时限建议

建议贵方在收到本函后 **10 个工作日内** 完成首轮回函；如部分接口能力尚未确定，可先反馈当前能力现状、预计提供方式和计划时间。
