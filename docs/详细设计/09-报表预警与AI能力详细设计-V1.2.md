本文档承接 `06-报表预警与AI能力概要设计-v0.1.md` 中的能力边界，以及 `01-数据库逻辑模型-V1.0.md` 中 R-01~R-11 骨架，把报表定义、统计指标、预警规则、AI Tool 规格、导出审计固化到可开发、可联调、可验收的详设层。

本文档重点回答：

- R-01~R-11 各实体的完整字段、约束和索引建议

### 5.2 核心报表字段口径

#### RPT-INV-001 库存余额表

主要展示字段：组织、仓库、物料编码、物料名称、规格型号、计量单位、账面数量、可用数量、冻结数量、批次数、单位成本、库存金额、安全库存、储备资金预算、预算占用率、周转天数、安全周转周期、是否低库存、是否超储、最后出入库日期。

统计口径：以 S-13 inventory 为唯一来源，按 (org_id, warehouse_id, material_id) 汇总当前快照值；可用数量 = 账面数量 - 冻结数量。

权限控制：数据范围以 A-06 数据权限过滤 org_id 和 warehouse_id。

---

#### RPT-INV-002 收发存报表

主要展示字段：组织、仓库、物料、期初数量/金额、期间入库（采购/退料/盘盈/调拨调入）、期间出库（领用/退货/盘亏/调拨调出/废旧处置）、期末数量/金额。

统计口径：以 S-21 inventory_transaction 按 transaction_type 分组汇总；期初 = 统计周期开始时 S-13 快照（或追溯累计）；统计周期默认月度，支持自定义时间区间。

---

#### RPT-PUR-002 计划完成率统计表

主要展示字段：组织、物料分类、采购计划（P-02）、计划数量/金额、已下达采购申请数量/金额、已到货数量/金额、已入库数量/金额、完成率（%）、未完成数量。

指标公式（R-03.metric_code = MTR-PUR-CPL）：
- 计划完成率 = 已入库数量 / 计划数量 × 100%
- 分子：累计入库数量（S-05 关联 P-02 同一计划期间）
- 分母：P-02 计划总数量
- 统计周期：月/季/年（默认月）
- 异常值处理：计划数量为 0 时标识"无计划"，不计入分母

---

#### RPT-PUR-003 采购价格变动趋势表

主要展示字段：物料、供应商、采购订单日期、单价、数量、金额、与上期价格差额（%）、与年初基准价差额（%）。

统计口径：以 S-02 purchase_order 的 unit_price 按 (material_id, supplier_id, period) 分析；基准价以物料档案 M-05 的 `standard_price` 为主（Spencer 2026-05-05 确认）。

---

#### RPT-FIN-001 接口推送状态表

主要展示字段：接口编码（F-14）、接口名称、来源单据类型、来源单据号、任务状态（F-01.task_state）、推送时间、回执时间、财务状态（F-01.finance_state）、NC 凭证号、失败原因、重推次数。

统计口径：以 F-01 interface_task 关联 F-14 interface_definition；按 org_id、时间区间、task_state 过滤；支持按推送失败、待重推、映射缺失等异常状态筛选。

---

#### RPT-SUP-001 物资周转率统计表

主要展示字段：组织、仓库、物料分类、物料、统计期间、期初/期末库存金额、出库金额、库存周转率、周转天数。

指标公式（R-03.metric_code = MTR-INV-TRN）：
- 库存周转率 = 期间出库金额 / ((期初库存金额 + 期末库存金额) / 2) × (365 / 统计天数)
- 周转天数 = 365 / 库存周转率
- 统计周期：季/年（默认年）

---

#### RPT-SUP-002 供应商履约率统计表

主要展示字段：供应商、统计期间、采购订单行数、按时到货行数、逾期到货行数、未到货行数、履约率（%）、平均逾期天数。

指标公式（R-03.metric_code = MTR-SUP-FUL）：
- 履约率 = 按时到货行数 / 有效采购订单行数 × 100%
- 按时到货：S-03 到货日期 ≤ S-02.delivery_date
- 统计周期：季/年
- 排除已关闭/已作废订单行

#### RPT-CMP-001 防爆/煤安按月反馈合规报表（业务方 Q-11-2 答复 2026-05-09）

主要展示字段：报告期月份、组织、物料分类、`is_safety_special=true` 物料清单、应反馈条数、已反馈条数、未反馈条数、反馈完整率（%）、反馈处理状态分布、超期未反馈条数、关联反馈单号、关联使用单位。

统计口径与生成方式：

- **物资范围**：M-05 material 中 `is_safety_special=true` 的物料（与详设 03 V1.1 字段一致），覆盖防爆电气、安全专项物资、煤矿安全准用产品（MA 标志）等高危类别。
- **应反馈条数**：上月该组织所有 `is_safety_special=true` 物料的领料出库（S-09）记录条数（按使用单位 + 物料 + 月份去重）；每条物资在使用单位投入使用后须由该单位反馈。
- **自动汇总频度**：每月 1 日 00:00（SY-02 `CMP_REPORT_GEN_DAY=1`）系统**全自动汇总**上月数据，无需人工提交清单（按业务方 Q-11-2 答复"系统全自动汇总"落地）；汇总结果推送至安全部门 + 物资部门 + 财务（用于下游 RPT-CMP-002 跟踪）。
- **反馈状态分布**：未提交 / 已提交 / 已处理 / 已落实 / 已闭环（与 RPT-CMP-002 后评价反馈跟踪表联动）。
- **超期判定**：当月 25 日（SY-02 `CMP_FEEDBACK_DEADLINE_DAY=25`）仍未提交反馈的，自动进入"超期未反馈"列，触发 ALR-CMP-001 预警。
- **政策依据**：政策 01 外委检修管理办法 + 流程调研 11-后评价流程 §1 反馈来源 #4 防爆/煤安按月反馈强制项。

#### RPT-CMP-002 后评价反馈处理跟踪表（业务方 Q-11-5 答复 2026-05-09）

主要展示字段：反馈单号、反馈类型（4 类：供应商 / 产品质量 / 售后服务 / 防爆煤安）、反馈方、反馈对象（供应商 / 物料 / 物资分类）、严重等级、提交时间、企划部受理时间、处理结果、落实情况、闭环时间、对供应商分类影响、闭环成效再反馈状态、`vendor_category_impact_id`（关联供应商重评估实例）。

统计口径：

- **反馈类型**：与流程调研 11-后评价流程 §1 一致（4 类来源）。
- **企划部统计**：按月统计企划部受理与处理时长，输出 P50/P95/P99 处理时长，计算 SLA 合规率。
- **供应商分类联动**：标记每条反馈是否触发供应商分类调整（联动详设 04 §4.1.4 供应商重评估机制，详 RPT-CMP-002 字段 `vendor_category_impact_id`）。
- **闭环成效再反馈**：标记反馈方是否对处理结果二次确认（流程调研 11 §3 闭环关键）。

> - **D-30（第 5 个月）**：ALR-INV-006B 触发二次预警，提醒业务员 + 财务（升级为"重要"级），强制业务员发起供应商对账
> - **D-0（第 6 个月）**：ALR-INV-007 触发阻断；对账完成前**阻断该供应商新采购订单审批**
> - **D+30**：自动进入"暂估超期物资"月度报表，提交财务核对
>
> 三段预警按业务方 Q-05-2 V1.1 答复"D-90 / D-30 二段预警 + D 阻断"落地，与 6 个月暂估窗口（`INVENTORY_TENTATIVE_WINDOW_DAYS=180`）严格对应。

> **超储双维度口径（联动流程调研 Q-10-3）：**
> - 金额维度：`S-13.total_amount > reserve_budget_amount × OVER_STOCK_BUDGET_RATE`，默认 `OVER_STOCK_BUDGET_RATE=110%`。
> - 周转维度：`turnover_days > safety_turnover_days`，`turnover_days` 取 RPT-SUP-001 物资周转率统计表计算结果；`safety_turnover_days` 优先取物料 / 分类 / 组织配置，未维护时默认 180 天。
> - 任一维度满足即触发 ALR-INV-002；预警记录需标明触发维度（金额 / 周转 / 双触发），便于后续延缓采购、调剂或处置。

### 6.2 阈值可配置化

R-04.alert_threshold 存储规则的默认阈值参数（JSON 格式），支持通过 R-04.applicable_org_id 按组织覆盖全局阈值。配置入口通过 SY-02 system_config 管理，config_key 格式：`alert.{rule_code}.threshold`。

## 七、AI Tool 规格

### 7.1 公共约定

- 所有 Tool 调用须携带调用者身份（user_id），在 A-16 ai_call_log 中完整记录。
- 返回结果遵循统一数据权限（A-06），不得超出调用者 org/warehouse 范围。
- 单次返回记录数上限：详情类 200 条；汇总类不限行，但数值聚合须控制在合理计算量内。
- 结果超限时返回 `code: RESULT_TOO_LARGE`，提示缩小查询范围。
- 价格、合同金额、付款金额等敏感字段按 R-07.sensitive_field_mask 脱敏。
- 一期 AI 能力全部只读，不开放写操作。

**通用错误码：**

| 错误码 | 含义 |
| --- | --- |
| PARAM_INVALID | 参数类型/格式错误 |
| PARAM_MISSING | 必填参数缺失 |
| NO_PERMISSION | 权限不足 |
| RESULT_TOO_LARGE | 结果超限，需缩小范围 |
| DATA_NOT_FOUND | 符合条件的数据不存在 |
| TOOL_DISABLED | Tool 已停用 |
| SYSTEM_ERROR | 系统异常 |

**输出结构：** 合同摘要（C-02 关键字段）+ 付款节点列表（C-04）+ 风险标识（逾期/即将到期/超付款比例）。

**敏感字段：** contract_amount、payment_amount 按角色脱敏。

### 7.9 AI-008 设备与设备租赁查询 Tool

**功能：** 查询设备状态、租赁执行情况、费用汇总和租赁异常。

**输入参数：** equipment_ids / equipment_category；org_ids；supplier_id；rental_state；date_from + date_to。

**输出结构：** 设备状态摘要（E-01）+ 租赁执行列表（E-09）+ 费用汇总（E-13）+ 异常提示（超期未退/停租异常）。

## 八、权限、导出与审计控制

### 8.1 报表权限控制

报表和 AI 查询的数据范围由 A-06 data_permission 过滤，维度包括：组织（M-01）、仓库（M-02）、供应商（M-09）、合同（C-02）、物料类别（M-04）。

| 角色 | 报表访问范围 |
| --- | --- |
| 集团领导/管理层 | 集团汇总报表；需授权后可穿透明细 |
| 集团物资管理 | 全集团物资类报表（含明细） |
| 各矿物资管理 | 本单位所有业务报表 |
| 仓库管理员 | 本仓库出入库/库存/盘点台账 |
| 财务人员 | 财务接口类报表（FIN-001~003）、合同付款类报表 |
| 采购人员 | 采购类报表（PUR-001~003）、合同执行台账 |
| 安全部门 | 火工品/安全专项台账 |
| 审计查询角色 | 授权范围内全部报表（只读） |

### 8.2 导出控制

- 导出须具备 R-09 导出模板对应的 `permission_required` 权限。
- 含价格、合同金额、付款金额的导出文件自动启用水印。
- 跨组织、全量明细、含敏感金额字段导出须记录 R-10 export_audit（同时记入 A-18 data_export_log）。
- 超过 5000 行的导出自动触发异步任务，生成完成后通知用户。

### 8.3 AI 调用审计

每次 Tool 调用写入 A-16 ai_call_log，包含：调用人 ID、Tool code、入参摘要、数据范围、结果条数、耗时、异常状态。AI 调用本身不代替受控导出；若需完整列表，应回到业务系统执行导出。

## 十三、一句话结论

09 详设的核心是把"报表、预警、AI 查询"用统一口径（R-01/R-03 指标定义）、统一权限（A-06 数据权限）、统一审计（A-16/R-10 调用日志）串联成一条可信数据链路：业务事实从 S/C/E/F 域流入，经受控统计口径输出报表和预警，再经受控 Tool 接口开放给 AI 平台，全程权限可追、导出可溯、异常可闭环；超储已按储备资金预算和周转周期双维度收口。
