## 三、实体关系（ERD 概要）

```
A-01 user_copy ←── Nova Platform 同步
  └─ A-03 user_role (N) → A-02 business_role
  └─ A-12 user_permission_log (N)
  └─ A-13 operation_log (N)
  └─ A-14 approval_log (N)
  └─ A-15 interface_operation_log (N)
  └─ A-16 ai_call_log (N)
  └─ A-17 data_import_log (N)
  └─ A-18 data_export_log (N)

A-02 business_role
  └─ A-05 role_function_permission (N) → A-04 function_permission
  └─ A-06 data_permission (N)
  └─ A-07 data_permission_rule (N)

A-08 approval_workflow → A-09 approval_node (N) → A-02
  └─ A-10 approval_opinion (N)
  └─ A-14 approval_log (N)

A-11 sensitive_operation （独立维护的高敏感操作枚举表）
A-17 data_import_log (N) → A-19 data_import_failure_row
```

### 5.2 角色互斥规则

以下角色组合不允许同一用户同时拥有（同一业务单据的发起/审批/执行不应由同一人闭环）：

| 互斥场景 | 规则 |
| --- | --- |
| 仓库管理员 + 财务人员 | 不应由同一人既做仓库操作又做接口财务审核 |
| 采购人员 + 财务人员 | 付款申请发起人与财务审核人应分离 |
| 使用单位申请人 + 仓库管理员 | 同一人不应既发起领料申请又执行出库 |
| 仓库管理员 + 盘亏/报废高敏感终审 | 仓库管理员不应拥有盘亏/报废终审权 |

实现方式：A-03 授权时检查 A-02.role_category 的互斥配置；系统管理员授权高敏感组合须触发二次确认和 A-12 日志。

### 5.3 功能权限树（主要分类）

功能权限按 `<模块>:<菜单>:<动作>` 格式编码，主要分类如下：

**库存模块（INV）：**
- `INV:STOCK:view` 库存查询
- `INV:IN:add/submit` 入库单新增/提交
- `INV:OUT:add/submit` 出库单新增/提交
- `INV:COUNT:manage` 盘点任务管理
- `INV:SURPLUS:approve` 盘盈终审
- `INV:SHORTAGE:approve` 盘亏终审（高敏感）
- `INV:SCRAP:approve` 报废/销毁终审（高敏感）
- `INV:TRANSFER:approve` 跨组织调拨审批（高敏感）

**采购模块（PUR）：**
- `PUR:PLAN:add/approve` 采购计划新增/审批
- `PUR:REQUEST:add/approve` 采购申请新增/审批
- `PUR:ORDER:manage` 采购订单管理
- `PUR:RECEIPT:manage` 到货验收管理

**合同模块（CON）：**
- `CON:CONTRACT:approve` 合同审批
- `CON:PAYMENT:approve` 付款审批（高敏感）
- `CON:CHANGE:approve` 合同变更终审（高敏感）

**财务接口模块（FIN）：**
- `FIN:TASK:view` 接口任务查看
- `FIN:TASK:repush` 接口重推（高敏感）
- `FIN:PERIOD:close` 月结操作（高敏感）
- `FIN:PERIOD:reverse` 反结操作（高敏感）

**报表模块（RPT）：**
- `RPT:BASIC:view` 基础报表查看
- `RPT:SENSITIVE:view` 含金额敏感字段报表查看
- `RPT:EXPORT:basic` 基础导出
- `RPT:EXPORT:sensitive` 敏感导出（高敏感）

**系统管理模块（SYS）：**
- `SYS:ROLE:manage` 角色与权限管理
- `SYS:WORKFLOW:config` 审批流配置
- `SYS:ALERT:config` 预警规则配置
- `SYS:INTERFACE:switch` 接口开关管理（高敏感）
- `SYS:LOG:view` 日志查看
- `SYS:IMPORT:execute` 批量导入（高敏感）

## 十三、一句话结论

10 详设的核心是把"谁能看、谁能做、谁来审、做了留痕"四件事用统一的角色—权限—数据范围—审批流—日志链串联起来：角色定义可配置、权限树可分层、审批节点可路由、高敏感操作必须强制留痕，使权限管控在开发阶段可实现、在实施阶段可配置、在运维阶段可审计。
