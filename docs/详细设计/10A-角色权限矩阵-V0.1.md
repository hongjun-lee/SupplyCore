# 10A 角色×功能权限矩阵 V0.1（草稿）

**文档目的：** 详设 10 §5.1 列出 11 个业务角色，§5.3 列出功能权限树（共 32 项），但"哪个角色能用哪些权限"的矩阵未明确（详设 §十占位项 + 10A V0.4 节十 Q7）。本文档由 10A 起草初稿，提交 Spencer + 物资/财务/IT 三方联合评审拍板，最终矩阵作为 ABP `SupplyCoresPermissionDefinitionProvider` 种子。

**起草依据：**
1. 详设 10 §5.1 角色清单 + §5.2 互斥规则 + §5.3 功能权限树 + §六审批流模板（节点角色）
2. 详设 06/05/08（业务流程中各角色出现位置）
3. 概设 01 部署/职责边界

**评审定位：** 本文档**未敲定**；标 ⚠ 处需评审讨论；标"※"处遵循详设 §5.2 互斥规则，不可放开。

## 七、ABP 落地形态（10A 实施侧补充）

矩阵敲定后，落到 ABP `SupplyCoresPermissionDefinitionProvider`：

```csharp
public class SupplyCoresPermissionDefinitionProvider : PermissionDefinitionProvider
{
    public override void Define(IPermissionDefinitionContext context)
    {
        var inv = context.AddGroup(SupplyCoresPermissions.GroupName);
        var stock = inv.AddPermission(SupplyCoresPermissions.Inv.StockView, L("Permission:InvStockView"));
        // ... 32 项功能权限 + 5 项 V1.1 补充权限

        // 11 个角色种子由 SupplyCoresDbMigrator 灌入：
        //   role.AddPermission(InvStockView)
        //   role.AddPermission(...)
        // 数据范围 § 标记不落 ABP（ABP 只管功能权限）；§ 由 A-06 / business_data_scope_grant 表落
        // ! 标记不落 ABP（! 是审批流路由属性）；由 A-08 + A-11 实现
    }
}
```

**要点：**
1. 矩阵中 ✓/§/!/⚠ 在 ABP 层都是"赋权"——`role.AddPermission()`
2. § 数据范围在 A-06 / `a.business_data_scope_grant` 落，不进 ABP
3. ! 审批流约束由 A-08 + A-11 落，不进 ABP
4. ※ 互斥规则由 A-03 授权时校验（详设 §5.2 实现说明）
