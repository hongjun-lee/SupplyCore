## 三、SY 域实体字段详表

### 3.1 SY-01 sequence_generator 序列号生成器

SY-01 字段定义及业务规则已在 `02-基础档案与组织仓库详细设计-v0.6.md` 节 4.9.1 完整展开，本篇不重复。核心约定：

- 格式：`{prefix}{YYYYMMDD}{6位顺序号}`，如 `RC20260502000001`
- 并发控制：行级锁（SELECT FOR UPDATE），与业务写操作在同一事务中完成取号
- 断号策略：不回填，断号仅记录不重用
- 全部前缀已在 02-v0.6 节 4.9.5 集中注册

## 四、部署与兼容约束

### 4.1 部署架构要求

| 项目 | 详细要求 |
| --- | --- |
| 部署方式 | 私有化部署，托管于招标方指定数据中心，不接受 SaaS 公有云形态 |
| 应用服务器 | 主应用服务采用无状态部署，支持横向扩展；至少双节点冗余 |
| 数据库 | PostgreSQL 或兼容 PG 语法的国产信创数据库，独立实例，与集团平台物理隔离 |
| 数据库高可用 | 至少主从热备；具体方案（PG streaming replication / Patroni / 国产 HA 方案）在环境准备阶段选型 `[待环境准备]` |
| 缓存 | Redis（或等效国产缓存），用于 SY-01 序列号热点缓冲、报表缓存、预警去重 |
| 文件存储 | 私有化对象存储或 NFS，用于 SY-04 附件；存储路径在环境准备阶段确定 `[待环境准备]` |
| 反向代理 | Nginx 或等效国产中间件；HTTPS 强制 |
| 操作系统 | 支持至少一类国产 Linux（麒麟/统信/欧拉等） |
| 浏览器 | 兼容 Chrome 110+、Firefox 110+、Edge 110+；兼容主流国产浏览器（360、百度浏览器企业版） |

### 4.2 环境分层

| 环境 | 用途 | 数据库 | 说明 |
| --- | --- | --- | --- |
| DEV | 开发联调 | 独立实例 | 允许脏数据 |
| TEST | 集成测试/UAT | 独立实例 | 使用脱敏生产数据或仿真数据 |
| PRE-PROD | 上线前验证/培训 | 独立实例 | 数据与 PROD 近似 |
| PROD | 生产 | 独立主从 | 唯一权威数据来源 |

## 五、性能与可用性基线

### 5.1 性能指标

| 指标 | 一期基线 | 说明 |
| --- | --- | --- |
| 并发用户数 | ≥ 200 | 需按实际组织规模在上线前复核 sizing |
| 单据操作响应时间 | ≤ 3 秒（P95） | 含新增、提交、审批等写操作 |
| 报表查询响应时间 | ≤ 5 秒（P95） | 台账类报表；复杂统计类允许 10 秒并走异步 |
| AI Tool 响应时间 | ≤ 5 秒（P95） | 超 5 秒转异步返回摘要 |
| 接口推送时效 | ≤ 30 秒（准实时场景） | 批量推送按 F-13 开关控制，允许异步队列 |
| 系统可用性 | ≥ 99.5%（年度） | 非计划停机 ≤ 43.8 小时/年 |
| 批量导入（1000 行） | ≤ 60 秒 | 超 60 秒走异步任务 |
| 大报表导出（5000 行） | ≤ 30 秒 | 超 5000 行强制异步 |

### 5.2 性能关键场景

以下场景须在性能测试阶段重点验证：

| 场景 | 说明 |
| --- | --- |
| 月末暂估/冲销 | S-07 大批量暂估生成；月初批量冲销；并发高峰 |
| 月末对账 | F-06 对账记录批量生成；F-01 任务大批量查询 |
| 盘点任务 | S-15/S-16 大批量创建和更新 |
| 收发存报表 | S-21 库存事务流水全量汇总；月维度跨期间统计 |
| AI Tool 并发 | 多用户同时调用 AI-001/AI-006；不得影响业务单据操作响应 |

### 5.3 接口任务队列策略

F-01 接口任务推送策略（补充 08 详设）：

| 推送模式 | 触发方式 | 队列策略 |
| --- | --- | --- |
| 实时推送 | 单据审批通过后立即入队 | 独立消息队列（如 Redis Stream 或等效中间件） |
| 批量推送 | 月末暂估/对账 | 定时任务批量处理，限流避免压垮 NC |
| 重推 | 人工触发/自动重试 | 最大重试次数 3 次（SY-02 配置）；超次数转人工 |

| 项目 | 要求 |
| --- | --- |

### 6.2 数据安全

| 项目 | 要求 |
| --- | --- |
| 传输加密 | 全链路 HTTPS/TLS 1.2+；内网服务间通信建议加密 |

## 七、数据初始化与迁移方案

### 7.1 初始化对象与顺序

上线前初始化数据按以下顺序导入，顺序不可颠倒（依赖关系决定）：

| 顺序 | 对象 | 来源 | 校验重点 |
| --- | --- | --- | --- |
| 1 | 组织机构（M-01） | Nova Platform 同步 | 层级完整、状态正确 |
| 2 | 仓库/库区/货位（M-02/03A/03B） | 业务梳理 | 归属关系、停用物资清理 |
| 3 | 计量单位（M-07/M-08） | 15-计量单位统一模板 | 换算规则完整 `[待业务确认]` |
| 4 | 成本中心（M-12）+ 映射（M-13） | NC 同步或人工 | NC 未落地先建本地结构 |
| 5 | 物料分类（M-04） | 业务梳理 + 现有编码规范 | 层级完整、停用口径统一 |
| 6 | 物料档案（M-05）+ 属性（M-06） | 现有系统迁移/人工整理 | 重复编码清理、单位统一 |
| 7 | NC 存货映射（M-14） | 14-NC映射模板 | 物料映射完整性 `[待 NC 落地]` |
| 8 | 供应商（M-09）+ 资质（M-10） | 现有系统迁移 | 停用供应商清理、黑名单校对 |
| 9 | 期初库存（S-13 + S-14） | 现有系统导出 | 账实一致、批次完整 |
| 10 | 用户副本（A-01）+ 角色授权（A-03） | Nova 同步 + 人工配置 | 权限完整性、互斥规则 |

### 7.2 导入模板规范

每类数据对象须提供标准 Excel 模板，模板至少包含：

- 字段说明行（字段名、类型、是否必填、取值范围、示例）
- 校验规则（格式、唯一性、外键存在性、值域合法性）
- 错误行高亮和错误信息列（导入失败时自动回填）
- 模板版本号（写入文件头部，A-17 导入日志记录版本）

### 7.3 导入校验规则

| 校验类型 | 说明 |
| --- | --- |
| 格式校验 | 日期格式、数值精度、字符串长度、枚举值 |
| 唯一性校验 | 编码、单据号等 UQ 字段不能重复导入 |
| 外键存在性 | 引用组织/仓库/物料分类等对象须已在系统中存在 |
| 业务规则校验 | 换算比例 > 0；安全库存 ≤ 最高库存；期初数量 ≥ 0 |
| 关联完整性 | 物料档案导入前物料分类须存在；供应商资质导入前供应商须存在 |

导入结果写入 A-17 data_import_log；失败行写入 A-19 data_import_failure_row。

### 7.4 期初库存处理

期初库存（S-13）导入须：

1. 以旧系统库存余额为基准（截至迁移切割日前一天末）
2. 核对账面数量与实物抽盘结果（差异 > 阈值时须人工确认）
3. 导入后生成期初事务流水（S-21，transaction_type = 期初导入）
4. NC 已落地时：与 NC 期初存货对账，差异须财务确认后方可切换
5. NC 未落地时：完成物资系统期初导入即可，NC 对账留待联调阶段

## 九、正式切换步骤框架

### 9.1 切换窗口要求

- 切换窗口选择业务低峰期（非月末、非盘点期），提前 5 个工作日发布切换通知。
- 切换期间旧系统进入只读/暂停新单据模式；新系统完成期初数据最终同步后上线。
- 切换步骤须经项目领导小组审批后方可执行，不得临时加速或跳过步骤。

### 9.2 切换步骤（概要）

| 步骤 | 内容 | 责任方 |
| --- | --- | --- |
| T-1 天 | 旧系统日末库存导出；完成 PROD 期初数据最终校对 | 物资/财务 |
| T 日窗口开始 | 旧系统暂停新单据；通知所有用户 | 网信办 |
| T 日 | 新系统期初库存最终导入；运行导入校验脚本 | 供应商+网信办 |
| T 日 | 抽样核对新旧系统关键数据 | 物资+财务 |
| T 日 | 开放新系统访问权限；切换 SSO 入口 | 供应商+网信办 |
| T 日 | 切换后首批单据试运行（入库/领料/查询） | 仓库管理员+物资 |
| T+1 日 | 确认业务可用；首日数据比对 | 物资+财务+网信办 |
| T+7 日 | 首周问题汇总；切换正式完成评估 | 项目领导小组 |

### 9.3 回退预案

**回退触发条件（任一项即触发评估）：**

1. 核心业务单据（采购入库/领料出库）出现系统级阻断性异常
2. 库存数据与旧系统差异 > 业务可接受范围且无法快速修复
3. 接口数据在 NC 已落地场景下无法推送，影响财务记账
4. 系统可用性低于 95% 且持续超过 2 小时

**回退决策权：** 项目领导小组（网信办 + 物资业务 + 财务共同判断后上报）。

**回退步骤：**

| 步骤 | 内容 |
| --- | --- |
| 1 | 立即暂停新系统新单据录入（切换 F-13 接口开关为全停） |
| 2 | 导出新系统已录入数据清单（新系统运行期间的全部 A-13 操作日志） |
| 3 | 重新开放旧系统（撤销 SSO 入口切换） |
| 4 | 新系统运行期间产生的有效单据由人工补录或转换后导入旧系统 |
| 5 | 项目领导小组审议回退原因、整改方案和再次上线时间 |

## 十六、一句话结论

11 详设的核心不是再列一遍性能指标，而是把"能不能稳、怎么迁、怎么切、怎么验、谁来接"这五件事落到可执行的检查点：并行运行退出条件三级门确保不会带着脏数据上线，切换步骤和回退预案确保项目领导小组决策有依据，验收清单和必交文档确保上线后不失控。
