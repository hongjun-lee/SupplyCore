# 开发进度规划 V0.4（项目内评审版）

**日期：** 2026-05-05
**范围：** **仅一期**（不含二期）；**NC 联调 + Catio 阶段 B/C 算入总工期**（外部触发型模块作为关键路径节点登记）
**估算依据：** 详设 01~11 V1.0+ + 10A V0.6 + 角色权限矩阵 V1.0 + 开发启动门对照表 V1.0 + 行业经验值（以 .NET 应用框架 / Vue Web + 小程序双端作为估算模型；不作为技术栈强制要求）+ 详设复杂度系数
**评审定位：** **内部评审版**——不对外发布，后续与中标供应商技术方案校准后再升 V1.0
**浮动范围：** 实际人天会随技术选型 / 团队成熟度 / 一期边界细节 ±30% 浮动；本文档为**立项申请、预算编制、供应商招标技术问询**的输入参考

---

## 一、文档目的与口径

### 1.1 文档目的

回答四个问题：

1. 完成项目一期总共需要多少人天 / 人月？
2. 按团队规模差异，工期需多久？
3. 关键路径是什么？哪些环节可并行 / 哪些是阻塞？
4. 风险预算占多少？

### 1.2 不做的事

- ❌ 不替代供应商技术方案（供应商对实现细节有自己的工时模型）
- ❌ 不锁定具体上线日（按详设 11 §八 三级退出门驱动，不写固定日期）
- ❌ 不指定具体技术栈（保留 .NET / Java / 前端框架选型空间）
- ❌ 不做团队个体能力评估（按行业平均能力假设）

### 1.3 估算方法

**底层公式：**

```
模块开发工时 = Σ(实体数 × 实体复杂度系数) + Σ(接口数 × 接口复杂度系数)
             + Σ(业务流程数 × 流程复杂度系数) + Σ(前端页面数 × 页面复杂度系数)

总工时 = 模块开发工时合计
       + 测试工时（30%）
       + 项目管理协调（15%）
       + 风险缓冲（30%）

总工期 = 总人月 / 团队规模 + 关键路径串行约束（NC 联调 / OAuth 颁发等）
```

## 二、行业经验值基线（估算模型：.NET 应用框架 / Vue Web + 小程序双端）

### 2.1 后端工时基线

| 任务类型 | 单位工时 | 说明 |
| --- | --- | --- |
| 简单 CRUD 实体 | **2 人天** | 含 EF Core 模型 + Repository + AppService + 单元测试 |
| 复杂业务实体（含状态机 / 业务规则） | **3-5 人天** | 含状态迁移 / 跨表联动 |
| 外部接口（NC / Catio） | **3-5 人天/接口** | 含校验 + 错误处理 + 重试；不含联调时间 |
| 接口实际联调 | **2 人天/接口** | 与外部系统真实交互调试 |
| 复杂业务流程（跨模块） | **5-10 人天** | 如采购下单跨需求/合同/库存/财务 |
| 数据库 schema + migration | **0.5 人天/实体（基础设施层）** | DbContext 多 schema 配置 |

## 五、阶段排期与关键路径

### 5.1 阶段拆分（参考 10A V0.6 §九 + 详设 11 §八）

```
立项 ──→ 阶段 0（招标/选型）──→ 阶段 A（基础设施 + 自建审计表）
   │      ↓
   │      阶段 B1（SSO + 角色权限）—— 等 OAuth 凭据
   │      ↓
   │      阶段 B2（Workflow + 业务模块开发并行）
   │      ↓
   │      阶段 C1（自动留痕，依赖 A）+ C2（高敏感拦截，依赖 A+B）
   │      ↓
   │      NC 联调（依赖业务方签字 + NC 厂商）
   │      ↓
   │      测试 + 验收 → 并行运行（三级退出门）→ 全量上线
   │
   └─→ 关键路径预并行（立项当周即启动，与阶段 0/A 重叠，详见 §5.3）：
        ① NC 厂商签实施协议 + 接口规范交接
        ② 业务方批量签字（NC 联调所需）
        ③ 集团 Nova OAuth 凭据预申请
        ④ 能源集团招采平台技术规范获取 + 沙箱接入测试
        ⑤ 数据清洗 + 试迁移脚本预热
```

### 5.3 阻塞性外部依赖（含立项当周启动的关键路径预并行动作）

## 八、总估算（一句话）

> **一期项目按 10 人推荐团队推进，预计总人月 ≈ 121，工期 ≈ 14-16 个月（含 NC 联调 + Catio 阶段 B/C 关键路径串行约束 + 招投标外部对接缓冲；立项当周即启动 5 项关键路径预并行动作压缩约 1 个月），风险缓冲 30% 已含；按 8/9/10/12 人团队规模可弹性调整工期 12-18 个月区间。**
