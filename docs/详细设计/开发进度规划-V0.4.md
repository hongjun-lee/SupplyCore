## 一、文档目的与口径

### 1.1 文档目的

回答四个问题：

1. 完成项目一期总共需要多少人天 / 人月？
2. 按团队规模差异，工期需多久？
3. 关键路径是什么？哪些环节可并行 / 哪些是阻塞？
4. 风险预算占多少？

### 1.2 不做的事

- ❌ 不替代供应商技术方案（供应商对实现细节有自己的工时模型）
- ❌ 不锁定具体上线日（按详设 11 §八 三级退出门驱动，不写固定日期）
- ❌ 不指定具体技术栈（保留 .NET / Java / 前端框架选型空间）
- ❌ 不做团队个体能力评估（按行业平均能力假设）

### 1.3 估算方法

**底层公式：**

```
模块开发工时 = Σ(实体数 × 实体复杂度系数) + Σ(接口数 × 接口复杂度系数)
             + Σ(业务流程数 × 流程复杂度系数) + Σ(前端页面数 × 页面复杂度系数)

总工时 = 模块开发工时合计
       + 测试工时（30%）
       + 项目管理协调（15%）
       + 风险缓冲（30%）

总工期 = 总人月 / 团队规模 + 关键路径串行约束（NC 联调 / OAuth 颁发等）
```

## 二、行业经验值基线（估算模型：.NET 应用框架 / Vue Web + 小程序双端）

### 2.1 后端工时基线

| 任务类型 | 单位工时 | 说明 |
| --- | --- | --- |
| 简单 CRUD 实体 | **2 人天** | 含 EF Core 模型 + Repository + AppService + 单元测试 |
| 复杂业务实体（含状态机 / 业务规则） | **3-5 人天** | 含状态迁移 / 跨表联动 |
| 外部接口（NC / Catio） | **3-5 人天/接口** | 含校验 + 错误处理 + 重试；不含联调时间 |
| 接口实际联调 | **2 人天/接口** | 与外部系统真实交互调试 |
| 复杂业务流程（跨模块） | **5-10 人天** | 如采购下单跨需求/合同/库存/财务 |
| 数据库 schema + migration | **0.5 人天/实体（基础设施层）** | DbContext 多 schema 配置 |

### 2.2 前端工时基线（Web + 小程序双端）

**Web 端（Vue 假设）：**

| 任务类型 | 单位工时 | 说明 |
| --- | --- | --- |
| 列表 + 表单（CRUD 页） | **2-3 人天/页** | 含查询 / 分页 / 表单校验 |
| 审批中心 / 详情页 | **4-6 人天/页** | 含审批流可视化 / 多状态 UI |
| 看板 / 大屏 | **5-8 人天/看板** | 含数据可视化 / 多卡片联动 |
| AI 交互页 | **4-6 人天/页** | 含 Tool 调用 UI + 结果展示 |
| 二次确认 / 高敏感弹窗 | **0.5 人天/类** | 复用模式 |

**小程序端（uni-app / Taro / 微信原生假设；技能栈与 Web 不同步）：**

| 任务类型 | 单位工时 | 说明 |
| --- | --- | --- |
| 通用框架（SSO 绑定 / 路由 / 组件库 / 网络层） | **15 人天/项目** | 一次性投入 |
| 移动端表单 + 列表 | **1.5-2 人天/页** | 较 Web 简化（少屏宽 / 单列布局）|
| 移动端审批中心 | **3-4 人天/页** | 高频使用，体验要求高 |
| 移动端扫码 / 拍照（库存场景）| **3-5 人天/类** | 含调用相机 + OCR 结果回填 |
| 移动端看板 + 推送 | **4-6 人天/类** | 含订阅消息 / 卡片消息 |
| **覆盖范围（一期）：** 移动端聚焦"高频 + 现场操作"——库存扫码、申请发起、审批中心、看板查看、预警推送、AI 快询；不做完整后台管理（如档案维护 / 报表导出 / 高敏感配置 → 走 Web 端） | — | — |

### 2.3 测试 / 管理 / 缓冲

| 项 | 占开发工时比例 | 说明 |
| --- | --- | --- |
| 测试（单元 + 集成 + E2E） | **30%** | 行业最低标准 |
