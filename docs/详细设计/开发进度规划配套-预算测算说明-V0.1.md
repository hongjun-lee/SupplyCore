## 2. 为什么不是简单按“10 个人 × 工资 × 月数”算

软件项目报价里的“人月单价”，不是开发人员工资。

供应商的人月报价里一般包含：

- 开发人员工资；
- 社保、公积金、奖金、福利等用工附加成本；
- 项目经理、售前、质量管理、行政管理等间接成本；
- 办公、设备、培训、人员空档期成本；
- 税费和财务成本；
- 企业利润；
- 项目风险缓冲。

所以一个人对外报价 **3.5–4.2 万 / 人月**，并不代表这个人每月工资就是 3.5–4.2 万。

通常员工实际税前工资大约占人月报价的 **45%–60%**。

也就是说：

| 对外人月报价 | 反推税前工资大致区间 |
| ---: | ---: |
| 3.0 万 / 人月 | 1.4–1.8 万 / 月 |
| 3.5 万 / 人月 | 1.6–2.1 万 / 月 |
| 4.0 万 / 人月 | 1.8–2.4 万 / 月 |
| 4.5 万 / 人月 | 2.0–2.7 万 / 月 |
| 5.0 万 / 人月 | 2.3–3.0 万 / 月 |

这个比例比较适合解释预算逻辑：

> 项目报价不是单纯发工资，而是供应商完整交付能力的成本。

## 3. 本项目为什么要按中等偏强团队估算

如果只是普通台账系统，沈阳本地普通实施团队报价可以更低。

但这个项目不是普通 CRUD 系统，复杂点比较多：

1. **NC 接口多**  
   详设里涉及 29 项 NC 接口，还包括推送、回执、重试、对账、异常处理、封账、反结等。

2. **集团平台对接多**  
   要对接 Nova / SSO / Workflow / 组织人员权限体系，不是孤立系统。

3. **双端建设**  
   既要 Web 管理端，也要小程序端。小程序还涉及扫码、现场作业、移动审批、AI 快询、预警触达。

4. **库存和财务逻辑复杂**  
   库存有批次账、库存事务、调拨、盘点、废旧、期初导入；财务有 NC、三单匹配、付款、对账。

5. **数据迁移不能低估**  
   老数据清洗、试迁移、正式迁移、切换演练、回退准备都需要专人负责。

6. **审批和审计要求高**  
   有统一权限、审批流、高敏感操作拦截、审计留痕等要求。

7. **报表、预警、AI 能力不是附带项**  
   这部分涉及报表口径、看板、预警规则、AI Tool 权限和审计。

所以建议不要按最低价团队估算，否则后期很容易出现：

- 人员配置不足；
- 开发周期拉长；
- 外部接口联调延期；
- 数据迁移质量不稳；
- 供应商后期通过变更追加费用。

## 6. 沈阳本地市场价口径

按沈阳地区软件实施团队大致情况，可以分几个档位看：

| 团队档位 | 综合人月报价 | 适用情况 |
| --- | ---: | --- |
| 普通本地团队 | 2.5–3.2 万 / 人月 | 一般管理系统、台账、弱集成项目 |
| 中等成熟团队 | 3.2–4.0 万 / 人月 | 有前后端、测试、项目管理，能做一般集成 |
| 中等偏强团队 | 3.5–4.2 万 / 人月 | 适合本项目这种 NC、SSO、双端、数据迁移项目 |
| 较强团队 / 全国型厂商 | 4.5 万以上 / 人月 | 管理成本更高、品牌溢价更高、交付保障更强 |

本项目建议采用：

> **3.5–4.2 万 / 人月**

作为预算测算单价。

按 121 人月计算：

| 综合人月价 | 对应费用 |
| ---: | ---: |
| 3.5 万 | 约 424 万 |
| 3.8 万 | 约 460 万 |
| 4.0 万 | 约 484 万 |
| 4.2 万 | 约 508 万 |

所以预算建议按 **450–520 万** 更合适。

## 7. 费用边界

### 7.1 包含在开发实施费里的内容

一般应包含：

- 系统设计深化；
- 后端开发；
- Web 前端开发；
- 小程序端开发；
- NC 等接口开发与联调配合；
- 权限、审批、审计开发；
- 报表、预警、AI Tool 开发；
- 数据迁移脚本与试迁移；
- 测试、缺陷修复、上线支持；
- 基础培训和交付文档。

### 7.2 不建议包含在开发实施费里的内容

这些建议单独列预算，避免开发费被稀释：

- 服务器、存储、网络、安全设备；
- 国产数据库、中间件、操作系统授权；
- 等保测评、密评、安全加固；
- NC 厂商专项配合费用；
- 能源集团招采平台专项接口费用；
- 长期驻场运维；
- 大规模业务培训；
- 上线后年度维护服务。

## 8. 可以给领导解释的几个要点

这部分只是沟通时可用的解释点，不按正式材料格式展开。

1. **预算不是按工资直接乘出来的。**  
   人月报价包含工资、社保、管理、税费、利润和风险，不等于开发人员实发工资。

2. **本项目不能按普通台账系统估价。**  
   它包含 NC、SSO、审批流、数据迁移、Web + 小程序、报表和 AI，复杂度明显高于普通管理系统。

3. **450–520 万是软件开发实施费，不是完整项目总投资。**  
   硬件、信创软件、测评、安全、第三方接口和运维要另算。

4. **如果预算压到 400 万以下，需要同步压范围或降低交付要求。**  
   否则后期容易通过变更、延期或减少人员投入来平衡成本。

5. **比较稳的预算中位数是 480 万左右。**  
   这个数既不是最低价，也不是高配价，比较适合做立项初算。
