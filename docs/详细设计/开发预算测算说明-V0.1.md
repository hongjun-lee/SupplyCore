## 一、项目建设内容概述

物资供应管理系统一期建设内容覆盖物资供应业务的主要管理链条，包含：

1. 基础档案、组织仓库、物料主数据与编码；
2. 需求计划、采购协同、招投标编排与采购订单；
3. 合同、资金计划、付款申请与三单匹配；
4. 库存入出库、调拨、盘点、废旧处置、库存账与批次账；
5. 设备台账与基础租赁闭环；
6. 财务 NC 接口、对账、封账、反结与异常处理；
7. 报表、预警、管理看板与 AI 快询能力；
8. 统一登录、权限审批流、高敏感操作拦截与审计留痕；
9. Web 管理端 + 小程序端双端建设；
10. 数据迁移、性能优化、上线切换、运维交接等非功能工作。

本项目不是单一 CRUD 管理系统，实施复杂度主要来自：

- 财务 NC 29 项接口及实际联调；
- 与 Nova/SSO/Workflow 等集团平台能力对接；
- Web 管理端与小程序现场操作端双端建设；
- 库存、合同、财务、审批、审计之间的跨模块联动；
- 历史数据清洗、试迁移、上线切换和并行运行要求；
- 报表、预警、AI Tool、管理看板等数据应用能力。

## 三、软件开发实施费用测算

结合沈阳地区软件实施团队市场报价情况，普通管理系统人月报价通常较低，但本项目包含 NC、SSO、审批流、数据迁移、Web + 小程序双端、报表/AI 等复杂内容，建议按**中等偏强实施团队**报价口径测算。

### 3.1 人月单价测算

| 口径 | 综合人月单价 | 对应费用 |
| --- | ---: | ---: |
| 保守低位 | 3.5 万元 / 人月 | 约 424 万元 |
| 中位测算 | 3.8 万元 / 人月 | 约 460 万元 |
| 稳妥测算 | 4.0 万元 / 人月 | 约 484 万元 |
| 偏强团队 | 4.2 万元 / 人月 | 约 508 万元 |
| 优质团队 / 复杂交付 | 4.5 万元 / 人月 | 约 545 万元 |

### 3.2 建议预算口径

综合考虑项目复杂度、外部联调、数据迁移和双端建设要求，建议软件开发实施费用按以下区间控制：

> **软件开发实施费建议预算：450–520 万元。**

其中：

- **450 万元左右**：属于相对经济口径，要求供应商具备较强复用能力，且外部联调较顺畅；
- **480–500 万元**：属于较稳妥口径，更符合本项目当前复杂度；
- **520 万元以上**：适用于供应商团队能力较强、驻场周期较长、交付质量要求较高或包含更多上线保障服务的情况。

建议对领导汇报时采用：

> **一期软件开发实施费用初步测算约 480 万元，建议预算控制区间为 450–520 万元。**

## 四、不包含在软件开发实施费内的事项

上述 450–520 万元主要对应软件开发实施服务，不建议与以下费用混算：

| 类别 | 是否包含 | 说明 |
| --- | :-: | --- |
| 服务器、存储、网络、安全设备 | 不含 | 视集团现有资源和信创环境另行测算 |
| 国产数据库 / 中间件授权 | 不含 | 如使用商业信创数据库、中间件，需单独询价 |
| 等保测评 / 密评 / 安全加固 | 不含 | 一般由第三方测评机构或安全厂商单独报价 |
| NC 厂商专项配合费用 | 不含 | 若 NC 厂商需收费配合，建议单列 |
| 能源集团招采平台专项接口费用 | 不含 | 如平台方或第三方要求收费，需单列 |
| 上线后长期运维服务 | 不含 | 可按年度运维服务单独测算 |
| 大规模业务培训 / 驻场保障扩展 | 部分不含 | 基础培训可纳入实施，长期驻场建议单列 |

因此，如按完整项目预算考虑，建议预留：

> **完整项目总预算建议按 550–700 万元区间统筹考虑。**

其中软件开发实施费约 450–520 万元，其他费用根据硬件、信创软件、测评、安全和第三方接口实际情况另行核定。

## 五、关于“人员工资”与“人月报价”的关系说明

在预算沟通中，容易出现一个误解：认为供应商报价中的“人月单价”就是开发人员实际工资。实际并不是。

供应商的人月报价通常包括：

1. 员工税前工资；
2. 社保、公积金、奖金、福利等用工成本；
3. 管理、项目经理、售前、质量管理等间接成本；
4. 办公、设备、培训、人员空档期成本；
5. 税费、财务成本、商务成本；
6. 企业利润；
7. 项目风险缓冲。

一般情况下，开发人员实际税前工资约占供应商对外人月报价的 **45%–60%**。以本项目测算的人月报价为例：

| 角色 | 对外人月报价参考 | 反推人员税前月薪参考 | 说明 |
| --- | ---: | ---: | --- |
| 架构师 / 技术负责人 | 4.5–6.0 万元 | 2.5–3.5 万元 | 核心架构、平台集成、复杂问题兜底 |
| 后端开发 | 3.2–4.5 万元 | 1.8–2.8 万元 | 业务模块、接口、规则、NC 对接 |
| Web 前端 | 2.8–4.0 万元 | 1.6–2.5 万元 | Web 管理端、复杂表单、报表看板 |
| 小程序前端 | 2.8–4.0 万元 | 1.6–2.5 万元 | 移动审批、扫码、现场作业 |
| DBA / 数据迁移 | 3.5–5.0 万元 | 2.0–3.0 万元 | 数据建模、调优、迁移、切换保障 |
| 测试 | 2.0–3.0 万元 | 1.0–1.6 万元 | 测试执行、回归、接口联调验证 |
| PM / BA | 3.0–4.5 万元 | 1.6–2.6 万元 | 需求澄清、进度管理、沟通协调 |

从综合口径看，若供应商对外报价为 **3.5–4.2 万元 / 人月**，对应团队实际平均税前月薪大致为 **1.6–2.5 万元 / 月**。供应商报价高于员工工资，是因为其还需承担用工附加成本、管理成本、税费、利润和交付风险。

建议在领导汇报中保留本节，但表述为“说明性内容”，主要用于解释预算构成，避免将人月报价简单等同于开发人员工资。

## 六、预算汇报建议表述

建议向领导汇报时采用以下口径：

> 根据当前详细设计成果和开发进度规划测算，物资供应管理系统一期预计软件开发实施工作量约 2548 人天，折合约 121 人月。项目包含 Web 管理端、小程序端、NC 接口、统一登录与审批流、数据迁移、报表预警和 AI 快询等复杂内容，建议按沈阳地区中等偏强软件实施团队报价进行预算测算。
>
> 按 3.5–4.2 万元 / 人月测算，软件开发实施费用约 424–508 万元。综合考虑项目复杂度、外部联调和风险缓冲，建议一期软件开发实施预算按 **450–520 万元** 控制，汇报中位数可按 **480 万元左右**掌握。
>
> 该费用不含服务器、国产数据库/中间件授权、等保测评、安全加固、第三方厂商专项配合费用和长期运维费用。若按完整项目统筹，建议总预算预留 **550–700 万元** 区间，最终以招标结果和供应商技术方案为准。

## 七、结论

1. 本项目一期软件开发实施工作量约 **2548 人天 / 121 人月**；
2. 推荐按 **10 人团队、14–16 个月**实施周期测算；
3. 软件开发实施费建议区间为 **450–520 万元**；
4. 汇报中位预算可按 **480 万元左右**掌握；
5. 完整项目总预算建议预留 **550–700 万元**；
6. 人月报价不等于人员工资，实际工资约占报价的 **45%–60%**，其余为用工附加成本、管理成本、税费、利润和风险成本。
