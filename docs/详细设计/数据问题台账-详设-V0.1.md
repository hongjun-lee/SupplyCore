# 数据问题台账模块详设 V0.1（2026-05-17 起草）

**模块**：DataIssueLog（数据问题台账 / 数据治理核心模块）
**目的**：6 类数据采集 + 系统侧导入校验 + 业务方填报错误的统一记录 + 追责 + 处理流程
**Sprint**：20k T-A2 实施依据（main 主代理 a / 0.5 PD）
**配套**：[`分批上线与基础数据采集计划-V0.1.md`](../上线/分批上线与基础数据采集计划-V0.1.md) §一 / [`数据采集总览-V0.1.md`](../上线/数据采集模板/00-数据采集总览-V0.1.md) §五

## 一、模块定位

- **核心价值**：把数据问题从"散落各处"集中到统一台账 / 业务方有 ownership / 系统侧自动记录 / 处理流程可追溯
- **场景覆盖**：
  - 业务方填模板时缺项 / 重复 / 编码冲突 / 必填缺失 / 枚举不合 / 关联不存在
  - 系统侧导入校验失败自动记录
  - NC 映射不一致
  - 高敏感物资属性错误
  - 期初库存盘点差异
  - 临期资质 / 临期物料
  - cron job 数据异常检测

### 3.2 Domain Service `DataIssueLogManager`

```csharp
public class DataIssueLogManager : DomainService
{
    private readonly IRepository<DataIssueLog, long> _repo;

    public async Task<DataIssueLog> RecordIssueAsync(
        string issueType,
        DataIssueDataType dataType,
        DataIssueSeverity severity,
        string description,
        string responsibleDept,
        string? importBatchId = null,
        int? importRowNo = null,
        object? contextData = null,
        string? linkedEntityType = null,
        long? linkedEntityId = null)
    {
        var slaDeadline = severity switch
        {
            DataIssueSeverity.Critical => Clock.Now.AddHours(24),
            DataIssueSeverity.High => Clock.Now.AddDays(3),
            DataIssueSeverity.Medium => Clock.Now.AddDays(7),
            DataIssueSeverity.Low => Clock.Now.AddDays(30),
            _ => Clock.Now.AddDays(7),
        };

        var log = new DataIssueLog
        {
            IssueType = issueType,
            DataType = dataType,
            Severity = severity,
            Description = description,
            ResponsibleDept = responsibleDept,
            ImportBatchId = importBatchId,
            ImportRowNo = importRowNo,
            ContextData = contextData != null ? JsonSerializer.Serialize(contextData) : null,
            LinkedEntityType = linkedEntityType,
            LinkedEntityId = linkedEntityId,
            Status = DataIssueStatus.Pending,
            SlaDeadline = slaDeadline,
        };

        return await _repo.InsertAsync(log, autoSave: true);
    }
}
```

## 九、关键设计决策

1. **issue_type 字符串而非枚举**：扩展性（新业务类型不需 enum 改动 / 各模板章节定义）
2. **SLA 按严重度自动计算**：业务规则透明 / 超 SLA 自动 alert
3. **ContextData jsonb**：灵活记录上下文 / 不强求结构
4. **import_batch_id GUID 关联**：导入失败时一次记录多条 / 同 batch 统一管理
5. **手动创建支持**：业务方主动反馈数据问题 / 不仅系统侧记录
6. **统计 API + Export**：业务方有可视化 + 可下载分析
