# 数据问题台账模块详设 V0.2（2026-05-17 / cici 反馈 #3+#4+#6 修订）

**模块**：DataIssueLog（数据问题台账 / 数据治理核心模块）+ DataImportBatch（数据导入批次 / V0.2 新增）
**目的**：6 类数据采集 + 系统侧导入校验 + 业务方填报错误的统一记录 + 追责 + 处理流程 + 批次回滚
**Sprint**：20k T-A2 实施依据（main 主代理 a / V0.2 实施 0.8 PD）
**配套**：[`分批上线与基础数据采集计划-V0.2.md`](../上线/分批上线与基础数据采集计划-V0.2.md) §一 / [`数据采集总览-V0.2.md`](../上线/数据采集模板/00-数据采集总览-V0.2.md) §五

**V0.2 修订要点（cici 反馈 6 点）**：
- **#3 sub_group_id 立修**：Entity 改继承 `SupplyCoresFullAuditedAggregateRoot<long>` 自动获得 sub_group_id / created_org_id / delete_reason（与 20+ 业务 entity 对齐）；Manager 走 SetSubGroupId 钩子（参考 DemandRequestManager）
- **#4 DataImportBatch 新增**：完整批次模型（模板类型/版本/SHA256/导入人/sub_group_id/总行成功失败警告/8 状态机/回滚原因）

## 一、模块定位

- **核心价值**：把数据问题从"散落各处"集中到统一台账 / 业务方有 ownership / 系统侧自动记录 / 处理流程可追溯
- **场景覆盖**：
  - 业务方填模板时缺项 / 重复 / 编码冲突 / 必填缺失 / 枚举不合 / 关联不存在
  - 系统侧导入校验失败自动记录
  - NC 映射不一致
  - 高敏感物资属性错误
  - 期初库存盘点差异
  - 临期资质 / 临期物料
  - cron job 数据异常检测

### 2.4 DataImportBatch 状态机（V0.2 #4 新增）

```
                       ┌─→ ValidationFailed（有 Critical / 不可导入）
PendingValidation ─→ Validating ─┤
                                 └─→ PendingConfirm（仅 Medium/Low 警告 / 待业务方确认）
                                            │
                                            ▼
                                         Importing ─→ Imported
                                                   ─→ ImportFailed（事务回滚）

Imported / Importing ─→ RolledBack（业务方主动回滚 / 记录 rollback_reason）
```

8 状态：PendingValidation=0 / Validating=10 / ValidationFailed=20 / PendingConfirm=30 / Importing=40 / Imported=50 / RolledBack=60 / ImportFailed=70

### 3.2 Domain Service `DataIssueLogManager`（V0.2 #3 立修 / 走 SetSubGroupId 钩子）

```csharp
public class DataIssueLogManager : DomainService
{
    private readonly IRepository<DataIssueLog, long> _repo;
    private readonly IRepository<Organization, long> _orgRepo;  // V0.2 新增 / 反查 sub_group_id

    public async Task<DataIssueLog> RecordIssueAsync(
        string issueType,
        DataIssueDataType dataType,
        DataIssueSeverity severity,
        string description,
        string responsibleDept,
        long? createdOrgId = null,  // V0.2 新增
        string? importBatchId = null,
        int? importRowNo = null,
        object? contextData = null,
        string? linkedEntityType = null,
        long? linkedEntityId = null,
        CancellationToken ct = default)
    {
        var slaDeadline = severity switch
        {
            DataIssueSeverity.Critical => Clock.Now.AddHours(24),
            DataIssueSeverity.High => Clock.Now.AddDays(3),
            DataIssueSeverity.Medium => Clock.Now.AddDays(7),
            DataIssueSeverity.Low => Clock.Now.AddDays(30),
            _ => Clock.Now.AddDays(7),
        };

        var log = new DataIssueLog
        {
            IssueType = issueType, DataType = dataType, Severity = severity,
            Description = description, ResponsibleDept = responsibleDept,
            ImportBatchId = importBatchId, ImportRowNo = importRowNo,
            ContextData = contextData != null ? JsonSerializer.Serialize(contextData) : null,
            LinkedEntityType = linkedEntityType, LinkedEntityId = linkedEntityId,
            Status = DataIssueStatus.Pending, SlaDeadline = slaDeadline,
        };

        // V0.2 sub_group_id 写入钩子（参考 DemandRequestManager:42-58 / 评审留痕清单 §修订 #2）
        if (createdOrgId.HasValue)
        {
            var org = await _orgRepo.FindAsync(createdOrgId.Value, cancellationToken: ct);
            log.SetCreatedOrgId(createdOrgId.Value);
            log.SetSubGroupId(org?.SubGroupId);  // 集团根 / 集团级共享允许 null
        }

        return await _repo.InsertAsync(log, autoSave: true, cancellationToken: ct);
    }
}
```
