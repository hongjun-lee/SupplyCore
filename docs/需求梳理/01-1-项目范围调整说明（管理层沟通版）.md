# 项目范围调整说明（管理层沟通版）

## 一、调整结论

经与业务部门进一步沟通确认，**一期建设范围需由原“物资供应管理”扩展为“物资供应 + 设备管理 + 资产管理”一体化建设**。

其中，设备管理和资产管理**不是仅做静态台账**，而是纳入一期整体范围，按**全生命周期管理**要求统筹设计和实施。

## 二、调整后的项目定位

一期建设项目可统一表述为：

**物资供应与设备资产管理系统建设项目**。

项目总体上应覆盖以下三类核心管理对象：

- **物资**：围绕需求、采购、到货、入库、领用、退料、调拨、盘点、处置形成闭环
- **设备**：围绕建档、投用、状态变更、维修保养、调拨、停启用、报废形成闭环
- **资产**：围绕登记、配置、调拨、盘点、处置、报废形成闭环

## 三、一期建设边界

一期不再理解为“先做物资，设备资产以后再说”，而应理解为：

**在一期同步建设物资管理、设备管理、资产管理三部分能力，并实现关键业务环节的衔接贯通。**

重点包括：

1. **物资供应管理**
   - 需求提报与采购计划
   - 采购执行、到货验收、质检、入库
   - 领料、退料、调拨、盘点、废旧处置
   - 合同、供应商、报表预警、财务接口等配套能力

2. **设备管理**
   - 设备分类分级与台账建档
   - 设备领用、投用、停用、启用
   - 设备状态管理
   - 设备维修、保养、点检留痕
   - 设备调拨、报废及全过程追踪

3. **资产管理**
   - 资产分类与资产登记
   - 资产配置、领用、调拨
   - 资产盘点与状态变更
   - 资产处置、报废及全过程追踪

## 四、需要强调的管理口径

本次调整的关键，不是“一期多加两个台账模块”，而是：

- 将设备、资产正式纳入一期建设范围
- 将设备、资产从静态登记对象提升为动态管理对象
- 将物资、设备、资产之间的来源、流转、使用、处置关系纳入统一系统留痕
- 为后续制度执行、统计分析、审计追溯和业财协同打下基础

建议在后续需求、招标和汇报材料中统一使用以下表述：

> 一期建设范围应覆盖物资供应管理、设备管理、资产管理及相关业务协同，其中设备管理、资产管理应覆盖台账建立、状态变更、业务流转、全过程留痕和统计分析等全生命周期管理要求，不限于基础信息登记。

## 五、对后续工作的直接影响

此次范围调整后，后续文档和招标工作应同步按以下原则推进：

- **项目名称和范围口径同步调整**，避免仍按“单一物资系统”描述
- **模块清单同步调整**，设备管理、资产管理应作为一期独立模块列示
- **二期边界同步调整**，二期不再将“是否建设设备资产管理”作为重点，而改为围绕租赁、复杂检修联动、折旧结算等深化场景增强
- **实施与报价同步调整**，要求供应商按一期完整范围进行方案设计和工作量评估

## 六、一句话结论

**一期建设范围已由“物资供应管理”明确调整为“物资供应 + 设备管理 + 资产管理”一体化建设，且设备、资产均按全生命周期管理要求纳入一期统筹实施。**
