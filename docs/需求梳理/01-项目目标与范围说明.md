## 三、项目名称

物资供应管理系统建设项目

## 四、项目背景

当前集团物资管理在实践中普遍存在以下问题：

- 物料分类与编码标准不统一
- 不同单位主数据口径不一致，存在一物多码、多物一码风险
- 入库、领料、退料、调拨、盘点等库存业务缺少统一闭环管理
- 物资业务与财务核算衔接不足，与 NC 财务系统接口规则不清晰
- 管理层难以及时掌握库存结构、物资消耗、呆滞积压和专项物资使用情况

因此，有必要建设一套面向集团管控的物资供应管理系统，统一基础标准，打通业务闭环，并实现与财务系统的规范衔接。

## 五、当前系统存在问题及内部调研情况

前期通过对系统开发端问题清单梳理，并结合 11 名关键岗位使用人员调研，现有系统的主要问题集中在以下四个方面；同时，调研也反映出当前最迫切的建设诉求：

### 1. 流程效率低下

- 现有系统缺少无纸化审批能力
- 电子签章能力缺失，审批流转仍较多依赖线下或人工传递
- 流程办理周期长，人工跑流程成本高，审批滞后现象较为明显

### 2. 数据管理薄弱

- 数据更新不够实时，难以及时反映业务变化
- 缺少跨环节、跨单据的关联分析能力，难以形成完整业务穿透
- 业务人员岗位调整后，系统权限不能同步更新，存在管理滞后
- 系统未有效延伸至基层单位和区队，造成业务执行与管理之间存在断层

### 3. 功能覆盖不足

- 缺少合同账期提醒等管理功能，难以对关键业务节点进行主动预警
- 缺少物资全生命周期管理能力，系统更多停留在局部业务处理层面
- 一些关键业务功能和管理辅助能力尚未覆盖，难以满足当前管理要求

### 4. 合规性不达标

- 现有系统在技术路线和部署适配方面不符合国产化、信创相关政策要求
- 难以满足后续规范建设、统一运维和持续扩展的要求

### 5. 内部调研反映出的核心诉求

结合调研结果，当前业务人员对以下能力的需求最为集中、最为迫切：

- 无纸化审批
- 手机端与 PC 端多端同步使用
- 流程优化与审批提速
- 数据实时更新与业务分析赋能
- 系统向基层单位、区队延伸，减少管理断层

上述情况说明，本次系统建设不能仅停留在“补几个功能”的层面，而应重点围绕**流程优化、数据赋能、基层延伸、关键功能补齐、信创合规**五个方向统筹推进。

## 六、项目建设目标

本项目建设目标可概括为以下四点：

### 1. 统一标准

统一物料分类、物料编码、计量单位、组织、仓库、成本中心等基础数据标准，形成集团统一口径。

### 2. 闭环管理

打通需求提出、采购执行、到货验收、质检、入库、领料、退料、调拨、盘点、废旧处置等核心业务流程，实现业务全过程留痕。

### 3. 业财协同

明确物资系统与 NC 财务系统的主数据映射关系、业务触发规则和凭证推送边界，支撑业务与财务协同。

### 4. 集团可视

形成库存、采购、消耗、盘点、呆滞和专项物资等核心管理视图，为管理层提供真实、及时、可追溯的数据支撑。

综上，本项目核心目标是把物资业务从“分散处理”变成“统一标准、流程闭环、业财协同、集团可视”。

## 七、项目定位

本系统定位为：

可以将本系统理解为一个**以库存实物流转为主线、以财务接口为落点、以权限审批和报表预警为支撑**的物资业务平台。

- **物资业务管理平台**
- **库存实物流转管理平台**
- **财务核算前置业务数据源**

即：

- 物资系统负责业务过程和实物流转
- NC 财务系统负责价值核算和财务凭证管理
- 两者通过标准接口协同，不重复建设、不互相替代

同时，本项目应纳入集团信息化统一治理框架统筹考虑，至少遵循以下要求：

- 接入集团统一权限体系，最低应支持 SSO
- 涉及组织、人员、记账科目、物料编码等核心数据时，应优先引用或同步权威系统数据
- 系统间集成原则上采用 API 通信，接口报文统一使用 JSON
- 如涉及 AI 能力建设，应并入集团统一 AI 入口，并纳入物资管理系统总体设计、总体招标要求和总体实施方案统一考虑，不应作为孤立附件游离于主方案之外

### 技术选型与平台对接要求

本系统在技术选型和部署方式上应满足以下要求：

**1. 数据库与部署**

- 数据库：独立部署，采用 PostgreSQL 兼容的国产信创数据库（如瀚高 HGDB、人大金仓 KingbaseES 等）
- 物资系统拥有独立数据库实例，不与集团统一平台共用数据库

**2. 开发语言与集成方式**

- 开发语言不做强制限定，供应商可根据自身技术能力和项目实际选择
- 若供应商采用 .NET 技术栈，可直接引用集团统一平台（Nova Platform）模块，降低组织、人员、权限等公共服务的集成成本
- 若供应商采用其他技术栈，须通过集团统一平台 API 完成全部平台对接，功能要求不变

**3. 与集团统一平台（Nova Platform）的对接**

集团统一平台（Nova Platform）作为上位系统，提供组织架构、人员信息、权限验证和统一登录等公共服务。物资系统作为独立业务系统，与平台保持低耦合：

- **组织与人员数据**：通过平台获取并本地同步，本地可维护数据副本用于业务关联查询，但权威源在平台侧
- **统一登录（SSO）**：对接平台统一身份认证服务
- **数据权限**：物资系统内部自行实现数据范围控制，但隔离粒度和组织边界须与平台一致（以二级集团为数据隔离边界）
- **审批流程**：物资系统可自带审批引擎，但审批人的组织角色解析应与平台体系兼容
- **物料编码等物资领域主数据**：由本系统作为权威源维护，其他系统通过本系统 API 引用

## 八、一期建设范围

一期建设建议坚持“先跑通核心闭环”的原则，重点覆盖以下内容：

### 1. 基础档案管理

- 组织架构
- 仓库档案
- 物料分类
- 物料主数据
- 编码规则
- 计量单位
- 供应商基础档案
- 成本中心对照
- NC 存货映射关系

### 2. 物资主数据管理

- 物料新增、变更、停用流程
- 一物一码管理
- 分类与属性维护
- 批次/保质期管理规则
- 条码/二维码基础支持
- 历史物料与 NC 编码映射初始化
- 预留与辽宁能源未来统一编码体系对接的字段、映射关系和接口扩展能力

### 3. 需求提报与采购计划管理

- 使用单位/区队需求提报
- 需求汇总、去重与库存校核
- 月度/季度/年度/专项/调整等多类型计划编制
- 采购计划分级审批
- 采购任务分解与下达
- 计划执行跟踪、调整与追加
- 计划外紧急采购留痕管理
- 新品计划线上申报

### 4. 采购执行与核心库存业务

- 采购申请
- 采购订单
- 到货验收与质检
- 采购入库
- 采购退货
- 领料出库
- 退料入库
- 仓库调拨
- 盘点盘盈盘亏
- 废旧物资处置
- 库存权属管理（区分权属归属和保管位置）
- 入库照片拍照保存

### 5. 合同管理

- 合同网上审批会签
- 合同与采购计划/订单/入库数据自动关联
- 合同履行进度实时查询
- 合同到期/账期自动提醒与预警
- 合同变更与终止审批留痕
- 合同全生命周期台账

### 6. 供应商管理

- 供应商分类管理（潜在/合格/负面）
- 供应商资质文件管理与过期提醒
- 合格供应商准入与采购联动
- 供应商考评机制
- 负面供应商（黑名单）管理

### 7. 质量与特殊物资控制

- 到货验收
- 质检结果处理
- 不合格退货与让步接收
- 火工品等特殊物资重点管控
- 安全专项物资标识与台账管理

### 8. 财务接口与对账

- 主数据同步与映射
- 业务单据财务触发规则
- NC 凭证推送
- 异常重推
- 月末对账与差异处理

### 9. 基础管理能力

- 审批流
- 权限管理
- 统一身份接入（至少支持 SSO）
- 操作日志
- 基础报表
- 关键预警
- 统一 AI 入口接入与能力开放支撑

## 九、一期暂不纳入或弱化纳入范围

为控制项目复杂度，一期建议暂缓或仅做预留的内容包括：

- 招投标管理（招标/非招标采购全流程线上管理）——一期采购计划审批通过后的招投标/询比价环节沿用现有线下流程和既有平台完成，中标/成交结果在系统合同模块中录入并与采购计划关联，确保链路可追溯；二期上线招投标模块后由系统接管
- 设备资产全生命周期管理（入账、权属分级、租赁、检修、报废、折旧）
- 能源集团电子商务平台对接（待平台侧对接标准明确后启动）
- 检修计划与物资需求深度联动
- 复杂经营分析驾驶舱
- 全量移动端深度作业
- 智能补货、智能预测等高级能力

## 十、项目边界

### 1. 本系统负责

- 物料主数据管理
- 物资库存实物流转管理
- 业务单据流转与审批
- 与财务系统的业务接口衔接
- 基础统计分析与预警

### 2. 本系统不直接替代

- NC 财务总账、应付、成本核算
- 专业采购招采平台
- 独立合同法务管理平台
- 固定资产全生命周期管理系统
- 集团级 BI 决策平台

## 十一、核心管理原则

项目建设需坚持以下原则：

- 一物一码，统一标准
- 编码稳定，只增不改
- 业务闭环，单据留痕
- 业务驱动财务，不反向篡改业务
- 集团统一规则，矿厂按权限执行
- 遵循集团统一权限、统一数据、统一接口和统一 AI 入口要求
- AI 接入纳入总体设计，不作为孤立附件单独考虑
- 先规范业务，再实现系统
- 以可落地为优先，不追求一步到位

## 十二、项目实施重点

一期最关键的不是界面展示，而是先统一以下内容：

- 物料分类与编码规则
- 主数据责任归口
- 单据体系与状态流转
- 财务口径与 NC 对接规则
- 成本中心对应关系
- 特殊物资管理边界
- 月末结账与对账机制

## 十三、项目成功标准

一期上线后，至少应达到以下效果：

- 物料编码标准统一，新增物料按统一规则生成
- 入库、领料、退料、调拨、盘点业务可在系统中闭环运行
- 与 NC 的主数据及关键业务接口可稳定执行
- 核心财务口径一致，减少手工台账与重复录入
- 能输出库存、收发存、领料、盘点、呆滞等基础报表
- 管理层可查看库存与消耗的核心数据
- 上线前大部分关键待确认事项完成决策，不将争议带入系统

## 十四、当前阶段工作目标

当前阶段不进入编码实施，重点完成以下工作：

- 明确项目范围与阶段目标
- 梳理业务流程与单据清单
- 梳理主数据对象与关键规则
- 梳理业务规则与财务规则
- 汇总待确认事项并明确责任分工

## 十五、一句话总结

本项目一期的本质不是建设一个“大而全”的平台，而是：

**先建立集团统一的物资业务标准，跑通库存业务闭环和 NC 财务联动。**
