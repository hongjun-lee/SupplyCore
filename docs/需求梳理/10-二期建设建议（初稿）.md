# 二期建设建议（初稿）

## 一、文档目的

本文档用于说明物资供应管理系统一期上线并稳定运行后，二期建设可重点推进的内容、边界和启动条件，作为：

- 管理层判断后续投资节奏的参考材料
- 一期招标和实施阶段提前说明扩展方向的依据
- 后续二期立项、需求调研和范围确认的基础文档

## 二、二期建设的总体定位

一期重点是先跑通物资业务核心闭环，解决"有没有、通不通、稳不稳"的问题。

二期重点则应从"基础可用"转向"深度协同、精细管理、作业深化、智能提升"，在不推翻一期成果的前提下，进一步提升集团管控能力和现场应用深度。

二期总体上建议围绕以下方向展开：

- 深协同：将物资系统与更多上游计划、下游平台和集团级能力做深度联动
- 深管理：从核心业务闭环进一步延伸到合同、租赁、经营分析等精细化管理场景
- 深作业：从基础录入和查询，延伸到移动作业、现场执行和智能仓储
- 深智能：在数据质量和流程稳定的前提下，逐步引入预测、补货、分析等智能能力

## 三、二期建设的基本原则

### 1. 不推翻一期主线

二期应基于一期已形成的主数据体系、编码体系、库存业务链路和财务接口能力进行扩展，不应因新增功能推翻一期已确认的核心规则。

### 2. 先稳运行，再扩范围

二期启动前，应以一期已稳定运行、关键数据质量可控、主要接口和审批规则已固化为前提，不建议在一期尚未稳定时同步摊大二期范围。

### 3. 先补管理闭环，再谈高级智能

设备租赁、合同全生命周期、检修联动等管理闭环场景，优先级应高于复杂算法、智能预测和高级驾驶舱。

### 4. 与集团统一治理保持一致

二期新增能力仍须遵循集团统一权限、统一接口、统一 AI 入口、统一数据标准和统一运维要求，不应形成独立烟囱式扩展。

## 四、二期建设优先级总览

根据业务价值、外部依赖和实施可行性，建议二期各内容按以下优先级推进：

| 优先级 | 内容 | 核心理由 |
|--------|------|----------|
| **P0** | 能源集团电子商务平台对接 | 与一期招投标模块配套，实现与集团平台双向数据同步，且为集团层面要求 |
| **P1** | 合同管理增强 | 基于一期运行反馈进行精细化增强，提升管理效率 |
| **P2** | 检修计划与物资需求深度联动 | 依赖检修系统成熟度和数据接口就绪程度 |
| **P2** | 全量移动作业深化与智能仓储 | 基于一期移动端使用反馈，按现场需求深化 |
| **P3** | 复杂经营分析驾驶舱 | 依赖一期数据积累达到一定规模和质量 |
| **P3** | 智能补货、智能预测等能力 | 依赖数据质量和算法验证，需先积累再建设 |
| **P3** | 与更高层集团平台正式双向协同 | 依赖辽宁能源集团后续制度和平台建设进度 |

说明：招投标管理原列为二期 P0，现已移至一期建设范围（一期-B 扩展批），不再纳入二期。P0 仅保留能源集团电子商务平台对接，P1 紧随其后，P2/P3 根据实际条件和资源情况分步推进。

## 五、建议纳入二期的重点内容

### 1. 能源集团电子商务平台对接（P0）

物资公司需求提出与辽宁能源集团电子商务平台深度对接。招投标管理已纳入一期建设范围，但平台侧对接标准和数据格式尚未完全明确，建议待标准确定后在二期实施与平台的深度对接：

- 适配平台数据格式要求
- 支持采购计划、标包、采购文件向平台推送
- 实现招标/非招标采购结果双向自动同步
- 实现物资编码、采购计划等数据的双向同步
- 无需人工二次录入
- 与一期采购计划、合同管理、供应商管理模块联动

### 2. 设备资产全生命周期管理（P1）

鉴于设备管理、资产管理已明确纳入一期，二期重点不再是“是否建设”，而是基于一期上线运行情况，对复杂场景和深化能力进行增强：

- 设备权属分级和复杂权属变更场景增强
- 租赁管理（起租、停租、租金结算，与合同和财务模块联动）
- 检修管理深化（检修申请、停租、检修入库、完成回退全流程）
- 复杂设备调拨及跨组织协同
- 折旧、价值变动等更深层业财联动能力
- 大型、小型设备及重点资产专题分析能力

### 3. 合同管理增强（P1）

一期合同管理定位为"管理闭环"，已实现审批会签、履行跟踪、到期/账期预警、变更管理和合同台账。二期定位为"精细化增强"，在一期闭环稳定运行的基础上，进一步提升合同管理的深度和效率：

- 合同模板库、条款库和版本管理（提高合同编制效率和规范性）
- 电子签约（减少纸质流转，加快签约周期）
- 补充协议和索赔管理（覆盖合同执行中的变更和争议场景）
- 供应商履约深度评分与合同执行偏差分析（为供应商管理和采购决策提供数据支撑）

### 4. 检修计划与物资需求深度联动（P2）

建议推动物资需求与检修计划、维修计划、工程计划之间的前置联动，减少"临时领料、被动补货、计划与库存脱节"等问题。

重点可包括：

- 检修计划驱动物资需求预测
- 检修项目与领料、退料、成本归集联动
- 工程项目、维修项目和专项任务的物资消耗穿透分析

### 5. 全量移动作业深化与智能仓储能力（P2）

一期建议先满足基础移动审批和基础条码能力；二期可进一步推进：

- PDA 全流程仓储作业
- 条码/二维码全链路扫描
- 上架、移库、盘点、复核等现场作业闭环
- 智能仓位、批次管理和现场操作校验

### 6. 复杂经营分析驾驶舱（P3）

在一期报表和基础预警的基础上，二期可逐步建设更高层次的经营分析能力：

- 采购、库存、消耗、呆滞、废旧处置综合驾驶舱
- 多组织、多矿厂、多维度对比分析
- 专项物资、安全物资、重点备件专题分析
- 管理层趋势分析和异常洞察

### 7. 智能补货、智能预测等智能能力（P3）

当一期数据质量、分类口径、消耗记录和计划关联较稳定后，二期可逐步引入：

- 安全库存与补货建议
- 消耗趋势预测
- 呆滞库存识别与处置建议
- 异常消耗识别与预警

### 8. 与更高层集团平台的正式双向协同（P3）

在一期已预留集团未来统一编码扩展能力的前提下，二期可根据辽宁能源集团后续制度和平台建设情况，逐步推进：

- 集团统一编码正式映射与对接
- 更高层集团平台的数据交换与协同
- 跨二级集团口径的主数据、分析或监管能力衔接

## 六、原则上不建议放入二期首批范围的内容

为避免二期再次失控，以下内容建议谨慎推进，除非有明确制度基础、数据基础和管理刚需：

- 过于超前、缺少业务制度支撑的复杂算法能力
- 与当前主线关联不强的大而全门户化建设
- 脱离集团统一治理框架的独立 AI 门户或独立数据平台
- 在一期基础规则尚未稳定时就推动的大规模组织级重构

## 七、二期启动前建议满足的条件

建议至少满足以下条件后，再正式启动二期立项或范围确认：

- 一期核心模块已稳定运行
- 物料主数据、分类和编码质量基本可控
- 与 NC 的核心接口、对账和月结机制已稳定
- 一期关键流程问题和高频遗留问题已基本收敛
- 已形成一期验收总结、遗留问题清单和二期优先级建议

## 八、对一期招标与实施的反向要求

虽然上述内容原则上纳入二期，但一期招标和实施阶段应提前考虑扩展性，不得将系统设计做死。尤其应重点关注：

- 数据模型应预留设备资产、租赁、项目、集团编码、电商平台对接等扩展空间
- 流程引擎、权限模型、接口框架应支持后续扩展
- 移动端、条码、仓储作业能力应具备分阶段深化条件
- 报表与预警框架应支持从基础报表扩展到驾驶舱分析

## 九、建议的二期编制顺序

建议后续如启动二期，按以下顺序组织：

1. 基于一期验收和试运行结果，形成二期候选清单
2. 按 P0→P1→P2→P3 优先级，首批聚焦能源集团电子商务平台对接（与一期招投标模块联动），以及设备资产深化增强和合同管理增强
3. 确认合同增强、检修联动、移动作业等场景的实施深度
4. 再评估驾驶舱、预测和智能能力的可行性
5. 最后形成正式二期立项建议或二期需求说明

## 十、一句话总结

二期不是简单"多做一些功能"，而是在一期核心闭环稳定后，按 P0→P3 优先级，围绕深协同、深管理、深作业和深智能四个方向，分阶段推进物资系统能力升级。
