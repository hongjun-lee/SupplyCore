## 三、8 大业务板块、15 个实施模块与贯穿性专题的对应关系

为兼顾管理沟通和实施落地，建议按以下方式理解当前文档体系：

- “8大业务板块”用于管理沟通和报审对照
- “15个实施模块”用于招标、实施、报价和验收
- “贯穿性专题要求”用于说明跨多个业务板块、贯穿全系统的统一约束

| 8 大业务板块 | 当前对应的实施模块 | 当前处理说明 |
|--------------|--------------------|--------------|
| 库存管理 | 模块五 采购到货与入库管理；模块六 库存领用与退料调拨管理；模块七 盘点与废旧处置管理 | 属于实施级拆解，将”收、发、存、盘、处”分开描述，便于招标和验收 |
| 计划管理 | 模块三 需求提报与采购计划管理 | 已从原”计划管理”细化到”需求提报、计划编制、审批、任务下达、执行跟踪” |
| 招投标管理 | 模块四 招投标管理 | 已纳入一期独立实施模块；覆盖招标/非招标采购全流程线上管理，与采购计划和合同模块联动 |
| 合同管理 | 模块十 合同管理；模块十一 资金计划管理 | 一期已作为独立实施模块纳入；合同条款定义、合同风险点管理由模块十负责，付款计划、付款申请、月度预支付汇总和支付进度管理由模块十一负责 |
| 物资信息库管理 | 模块一 基础档案与组织仓库管理；模块二 物料主数据与编码管理 | 实施层拆分为”基础档案”与”物料主数据”两部分，便于明确主数据边界 |
| 数据统计分析 | 模块十四 报表预警与查询分析 | 已吸收统计分析、预警、追溯、导出等要求 |
| 供应商管理 | 模块十二 供应商管理 | 一期已作为独立实施模块纳入；供应商基础档案由模块一协同支撑 |
| 设备资产管理 | 模块八 设备管理；模块九 设备租赁管理 | 一期强化设备管理，并将设备租赁独立为正式实施模块；资产相关能力不再单列模块，仅保留设备来源关联、权属归属和处置留痕等关联能力 |

当前建议单独作为贯穿性专题要求，而不单独计入 8 大业务板块或 15 个实施模块的内容包括：

- 集团统一平台对接（组织、人员、SSO、数据权限等）
- 财务与 NC 业财协同
- 权限审批、日志审计与导出控制
- 数据安全、脱敏、水印与留痕
- 数据迁移、新旧系统并行运行、上线切换与回退
- 信创部署、国产数据库与运行环境要求
- 移动端、条码、PDA 等跨模块能力
- AI 能力开放与统一入口接入
- 辽宁能源未来统一编码扩展预留

## 六、当前仍需重点关注的事项

虽然来源需求大部分已吸收，但后续仍建议重点盯住以下事项：

1. 顶层口径必须持续统一，避免目录稿、招标正文、对照表再次出现不同模块数或不同一期边界。
2. 招投标管理已纳入一期；一期应明确“采购文件导出/上传、平台招评标、结果导入/附件归档”的协同边界，二期再推进能源集团电子商务平台直连和自动同步，防止后续又被泛化。
3. 财务系统“往来账、进度款、设备折旧”等深度协同内容，后续需与财务部门明确哪些一期做、哪些二期做、哪些仅做接口预留。
4. 新旧系统并行运行的周期、比对规则、回退条件还需形成更细的执行 SOP。
5. 数据导出权限、水印、导出审批和留痕机制，后续可在安全或运维专题中继续细化。

## 七、一句话总结

《报审-物管系统开发项目功能及需求报告》中的主要业务诉求，当前已基本纳入 SupplyCore 文档体系；现阶段更关键的不是再“补新点”，而是持续保持一期、二期、预留和待确认边界的口径稳定。
