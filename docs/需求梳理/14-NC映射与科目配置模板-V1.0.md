# NC 映射与科目配置模板（V1.0）

**版本：** V1.0
**日期：** 2026-04-24
**文档性质：** 项目主文档层实施准备模板
**适用阶段：** 详细设计启动前、实施准备阶段、NC 落地前配置建模阶段

---

## 一、文档目的

本文档用于在 `NC 尚未落地` 或 `NC 尚未完成正式联调` 的阶段，先将物资系统所需的 `NC 存货映射` 和 `凭证科目规则` 固化为可配置模板。

本文档主要解决：

- 当前阶段如何先把映射字段、状态、开关和责任人固定下来
- 哪些映射和科目规则可以先以模板或占位值维护
- 哪些配置缺失时必须拦截正式财务推送
- 后续 NC 落地后，应在本模板基础上补齐哪些正式配置

本文档不替代 `docs/需求梳理/05-财务与NC接口需求说明-V1.0.md` 的需求边界说明，也不替代 `docs/详细规则/物资管理与财务接口规范.md` 中的最终分录和报文细则。本文档的定位是把“可配置”真正落成模板。

## 三、使用原则

### 3.1 可配置，不硬编码

NC 存货映射和凭证科目规则必须通过配置维护，不得在程序中按固定物料、固定组织或固定业务类型写死。

### 3.2 先固化模型，再补正式编码

当前若 NC 尚未落地，则先完成字段、状态、配置入口、审批路径和启停开关设计，不以本阶段形成最终 `NC 存货编码` 和 `正式科目编码` 为前置条件。

### 3.3 业务可运行，财务推送需受控

业务单据可按项目阶段要求继续流转，但凡涉及正式财务推送的场景，只要映射或科目规则不完整，就必须拦截，不得“先推后补”。

### 3.4 状态清晰、责任可追溯

每条映射和每条科目规则均应具备状态、维护人、确认人、生效时间、停用时间和变更记录。

### 3.5 模板服务当前，也预留后续

本模板既服务当前 `NC 未落地` 阶段的占位配置，也应能够平滑过渡到 NC 正式落地后的正式映射和联调，不允许后续推倒重来。

## 四、模板范围与边界

本模板当前覆盖以下两类核心对象：

| 对象 | 当前阶段作用 | 说明 |
| --- | --- | --- |
| NC 存货映射 | 解决物料如何对应 NC 存货口径 | 以 `物料 -> NC 存货` 为核心，支持默认映射和组织级补充映射 |
| 凭证科目规则 | 解决不同业务场景的科目归属和凭证规则如何配置 | 以 `业务类型 + 组织 + 物料分类` 为核心维度 |

本模板暂不单独承担以下内容：

- 集团组织与 NC 核算组织完整映射台账
- 使用单位与成本中心完整对照表
- NC 最终报文样例和错误码字典
- 最终借贷分录细则和特殊财务政策说明

上述内容应分别在组织映射台账、成本中心对照表、接口规范和详细规则文档中继续展开。

## 五、NC 存货映射模板

### 5.1 字段设计说明

| 字段 | 填写要求 | 说明 |
| --- | --- | --- |
| 映射层级 | 必填 | 取值建议：`默认映射`、`组织级映射` |
| 物料编码 | 必填 | 引用物资系统正式物料编码 |
| 物料名称 | 必填 | 与物料主数据一致 |
| 物料分类 | 必填 | 便于后续分类统计和批量校验 |
| 规格型号 | 选填 | 细分物料时保留追溯信息 |
| 主单位 | 必填 | 与单位字典一致 |
| 适用组织 | 条件必填 | 组织级映射时必填；默认映射时可留空或填“全局” |
| 适用仓库 | 选填 | 仅在存在仓库级特殊口径时使用，一期不建议常态化依赖 |
| NC 存货编码 | 当前阶段可占位 | NC 未落地时可暂填占位值或留空，但状态不得标记为“已配置并启用” |
| NC 存货名称 | 当前阶段可占位 | 与上字段配套 |
| 映射状态 | 必填 | 取值见 `5.2` |
| 接口启用状态 | 必填 | 取值建议：`未启用`、`已启用` |
| 生效日期 | 必填 | 记录配置生效时间 |
| 失效日期 | 选填 | 停用时填写 |
| 维护人 | 必填 | 对应网信办或实施维护人 |
| 财务确认人 | 条件必填 | 状态进入“已配置”前必须明确 |
| 备注 | 选填 | 记录特殊口径、占位说明或历史原因 |

### 5.2 映射状态字典

| 状态 | 含义 | 当前阶段允许动作 |
| --- | --- | --- |
| 未配置 | 仅完成物料建档，尚未建立映射关系 | 允许业务建档，不允许正式财务推送 |
| 待确认 | 已提出映射方案，待财务确认 | 允许保存配置，不允许正式财务推送 |
| 已配置 | 映射关系和责任人已确认 | 仅当接口启用状态为“已启用”时才允许正式推送 |
| 已停用 | 历史存在映射，当前已停用 | 不允许继续用于新任务，但需保留历史追溯 |

### 5.4 使用规则

- 默认优先采用 `默认映射`，只有存在财务组织差异时才建立 `组织级映射`。
- 若存在 `组织级映射`，则优先使用组织级映射；否则回退到默认映射。
- 映射缺失时，系统应将接口任务置为 `待映射/待推送拦截`，不得自动生成正式财务任务。
- 映射变更不得直接覆盖历史记录，必须保留变更前后值和生效时点。

### 6.4 使用规则

- 一期科目规则至少支持按 `业务类型 + 组织 + 物料分类` 三个维度配置。
- 若存在更细粒度特殊场景，应在备注中说明，不建议在一期直接扩展过多组合维度。
- 科目规则缺失时，业务单据可继续保留业务状态，但正式凭证推送必须被拦截。
- 科目规则变更后，不得无痕影响历史已推送任务；必要时通过版本或生效日期控制。

## 七、开关控制与拦截规则

### 7.1 建议开关清单

| 开关 | 作用 | 建议责任人 |
| --- | --- | --- |
| NC 接口总开关 | 控制系统是否允许生成正式财务接口任务 | 网信办 |
| 存货映射校验开关 | 控制映射缺失时是否拦截 | 网信办 |
| 科目规则校验开关 | 控制科目缺失时是否拦截 | 网信办 |
| 场景级接口开关 | 按采购入库、领料、调拨等场景逐项控制 | 网信办 + 财务 |
| 重推开关 | 控制异常修复后是否允许人工重推 | 网信办 |

### 7.2 拦截规则

| 场景 | 处理要求 |
| --- | --- |
| 映射状态为 `未配置/待确认` | 不生成正式财务接口任务 |
| 科目规则状态为 `未配置/待确认` | 不生成正式财务接口任务 |
| 配置状态为 `已配置` 但启用状态为 `未启用` | 允许保留占位配置，不允许正式推送 |
| NC 接口总开关关闭 | 所有正式财务推送统一停用 |
| 历史任务已推送成功 | 不允许通过修改模板直接覆盖历史结果 |

## 八、人工替代与留痕模板

当前若 NC 尚未落地，或虽已落地但部分场景暂未启用正式接口，应明确人工替代路径并保留留痕。

| 序号 | 业务场景 | 人工替代方式 | 主责部门 | 配合部门 | 留痕载体 | 后续补录/补联调时点 | 备注 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | 待填写 | 待填写 | 财务部门 | 网信办、业务部门 | 手工台账/审批单/会议纪要 | 待填写 | 待填写 |
| 2 | 待填写 | 待填写 | 财务部门 | 网信办、业务部门 | 手工台账/审批单/会议纪要 | 待填写 | 待填写 |

使用要求：

- 人工替代只解决阶段性过渡，不得长期替代正式接口设计。
- 所有人工处理均应有责任人、处理时间和补录计划。
- 人工台账不得替代系统主数据和接口配置本身，后续仍需回到正式配置。

## 九、NC 落地后需补齐事项

待 NC 账套、存货档案和科目体系正式明确后，应在本模板基础上补齐以下内容：

1. 将占位的 `NC 存货编码/名称` 补齐为正式值。
2. 将占位的 `借方科目/贷方科目` 补齐为正式科目编码和名称。
3. 与组织映射台账、成本中心对照表、单位映射表做交叉校验。
4. 完成关键业务场景联调，并形成回执、错误码和异常补偿记录。
5. 将启用状态由 `未启用` 按阶段门切换为 `已启用`，并保留审批留痕。

---

## 十、模板使用结论

在当前 `NC 未落地` 的项目阶段，真正需要先固化的不是最终编码本身，而是 `映射模型、科目规则、状态字典、开关控制和责任边界`。只有这几项先模板化，后面 NC 一旦落地，正式配置和联调才不会再次失控。
