**版本：** V1.0
**日期：** 2026-04-24
**文档性质：** 项目主文档层实施准备模板
**适用阶段：** 详细设计启动前、主数据准备阶段、实施初始化阶段

---

## 一、文档目的

本文档用于将物资供应管理系统一期所需的 `计量单位字典`、`单位换算规则`、`维护归口` 和 `NC 单位映射` 先统一成可执行模板。

本文档主要解决：

- 单位字典由谁维护、哪些字段必须统一
- 一个物料涉及采购、库存、领用等不同单位时，如何定义换算关系
- 哪些单位可以直接使用，哪些同义叫法必须合并
- 与 `NC` 的单位口径如何预留映射，不在系统中各自乱建

本文档的目标不是扩写一套理论，而是给详细设计和初始化准备提供一张能直接填的模板。

## 三、使用原则

### 3.1 单位字典统一维护

项目范围内所有单位必须引用统一字典，不允许各矿、各仓库、各模块各自新增同义单位。

计量单位字典的业务维护归口应设在 `物资部门`；`财务部门` 负责与 NC 等财务口径相关的映射确认；`网信办` 负责系统配置承载、权限控制和技术支持。

### 3.2 一个物料必须有主单位

每个物料至少应明确 `主单位`，并在需要时扩展 `采购单位`、`库存单位`、`领用单位`。没有主单位的物料不得进入正式初始化。

### 3.3 单位不同必须有换算

采购单位、库存单位、领用单位不一致时，必须维护明确换算比例、精度和舍入规则，不允许依赖人工口头换算。

### 3.4 变更必须审批留痕

单位字典新增、停用、别名合并、换算比例调整、NC 映射调整，都必须留痕，不得直接覆盖历史。

### 3.5 NC 映射可预留，不可乱配

若当前阶段 `NC` 尚未落地，可先预留 `NC 单位编码/名称/映射状态` 字段，但不得因为未落地就放任各系统各自维护不同单位口径。

## 四、模板范围

本模板当前覆盖四类内容：

| 对象 | 主要用途 |
| --- | --- |
| 单位字典 | 统一单位编码、名称、类型、精度、状态 |
| 物料单位配置 | 明确物料主单位、采购单位、库存单位、领用单位 |
| 单位换算规则 | 明确不同单位间的换算比例和舍入方式 |
| NC 单位映射预留 | 为后续财务联动保留单位映射字段和状态 |

本模板暂不单独承担：

- 成本中心映射
- 组织与 NC 核算组织映射
- 凭证科目规则
- 报表统计口径细项

## 六、物料单位配置模板

### 6.1 配置规则

- 每个物料必须有 `主单位`。
- 若 `采购单位`、`库存单位`、`领用单位` 与主单位一致，可直接复用，不必重复派生换算。
- 只有当实际业务存在稳定换算关系时，才允许配置不同单位。
- 不允许同一物料在未定义换算规则的情况下跨单位流转。

## 八、治理与变更规则

| 场景 | 处理要求 | 主责 |
| --- | --- | --- |
| 新增单位 | 先判定是否已有同义单位；确认无重复后方可新增 | 物资部门 |
| 单位停用 | 不删除历史记录，只将状态置为停用 | 物资部门 |
| 别名归并 | 历史别名保留在备注或别名字段，不再作为正式取值 | 物资部门 |
| 换算比例调整 | 必须审批并评估对库存、报表、接口的影响 | 物资部门 + 财务部门 + 网信办 |
| NC 单位映射调整 | 必须与财务共同确认，网信办负责系统配置支持，不得单方修改 | 物资部门 + 财务部门 + 网信办 |

## 九、上线前最低检查项

上线前至少应完成以下检查：

1. 单位字典已形成唯一正式口径，不存在明显同义重复项。
2. 所有一期范围内物料均已配置主单位。
3. 存在多单位使用场景的物料，均已维护换算规则。
4. 数量精度和舍入方式已明确，不依赖人工解释。
5. 若涉及 `NC` 联动，单位映射字段和状态已预留；若 `NC` 未落地，则至少完成占位配置和状态控制。

## 十、一句话结论

计量单位这件事看起来小，实际上会同时影响主数据、库存数量、报表口径和财务接口。当前阶段不需要把它写成大而全的规范，但至少要先把 `字典、换算、归口、映射预留` 这四件事收成一张统一模板。
