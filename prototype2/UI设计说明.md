---

## 一、原型目标

本原型用于物资供应管理系统的早期演示和业务确认，重点展示：

- 系统菜单树与模块组织方式；
- 主要业务页面的列表、查询、状态和操作入口；
- 单据页面的基本信息、明细行、附件、审批轨迹等骨架；
- 典型流程从“列表查看”到“单据处理”的页面关系；
- 与 Nova 平台、NC 接口、审批流、AI 能力的入口和占位方式。

本原型不接真实后端，不承载正式权限逻辑，不代表最终前端代码结构。所有数据均使用 mock 数据。

## 三、总体布局

采用企业管理系统常见的三段式工作台布局：

- 左侧：模块菜单树，支持一级模块与二级功能；
- 顶部：系统标题、当前组织、搜索入口、消息/待办、用户信息；
- 中部：主工作区，展示列表、单据、看板、接口监控等页面；
- 右侧/弹层：用于高级筛选、审批轨迹、AI 助理、操作记录等辅助信息。

首屏不做营销式首页。进入系统后直接展示工作台与业务待办。

## 四、视觉风格

整体风格定位为“企业运营台”：

- 克制、清晰、可扫读；
- 强调业务状态、单据进度、待办数量和异常提示；
- 避免装饰性大色块、渐变背景、营销式 hero；
- 信息密度适中，优先保障表格、查询区、单据明细的阅读效率。

建议视觉基调：

- 背景：浅灰白工作区；
- 主色：稳重蓝或蓝绿，用于主按钮、选中菜单、关键链接；
- 辅色：青绿、橙色、红色分别用于正常/提醒/异常状态；
- 字体：系统默认中文字体优先，数字和英文保持清晰等宽感；
- 圆角：控制在 6px-8px，避免过度圆润；
- 卡片：只用于工作台指标、列表分组、单据区块，不做嵌套卡片。

## 六、页面模板

### 6.1 列表页模板

适用于档案、台账、单据列表、接口任务列表。

标准结构：

- 页面标题与说明；
- 关键统计条，如总数、待审核、异常、今日新增；
- 查询区：关键词、组织、仓库、状态、日期范围；
- 操作区：新增、导入、导出、刷新、更多；
- 数据表格：编号、名称、组织、状态、金额/数量、更新时间、操作；
- 分页区；
- 可选右侧抽屉：高级筛选或详情摘要。

### 6.2 单据页模板

适用于需求计划单、采购订单、合同、入库单、出库单、租赁单。

标准结构：

- 单据标题区：单号、状态、当前环节、主操作按钮；
- 基本信息区：组织、申请人、日期、业务类型、备注；
- 明细行区：物料、规格、单位、数量、金额、仓库、批次等；
- 附件区：上传占位、附件清单；
- 审批轨迹区：发起、审核、会签、退回、完成；
- 操作日志区：关键字段变更、接口推送、异常记录。

### 6.3 监控页模板

适用于 NC 接口任务、AI 调用、预警记录、审计日志。

标准结构：

- 状态概览：成功、失败、待处理、重试中；
- 时间轴或任务链路；
- 异常原因与重推操作；
- 请求/响应摘要；
- 关联业务单据入口。

### 6.4 工作台模板

工作台聚焦“今天要处理什么”：

- 我的待办；
- 近期预警；
- 接口异常；
- 库存关键指标；
- 采购/合同/入库/出库快捷入口；
- AI 助理入口占位。

## 七、组件规则

| 组件 | 设计规则 |
|---|---|
| 状态标签 | 使用明确颜色区分草稿、待审核、审中、已批准、已驳回、异常、冻结等状态 |
| 查询区 | 默认展示高频字段，高级条件放入展开区或抽屉 |
| 表格 | 保持列宽稳定，重要编号和状态靠前，操作列固定在右侧 |
| 主按钮 | 每页最多 1 个主按钮，其他操作使用次按钮或菜单 |
| 明细行 | 支持新增、删除、复制、批量选择等占位操作 |
| 附件区 | 展示文件名、类型、上传人、时间、预览/下载入口 |
| 审批轨迹 | 使用纵向时间线，突出当前节点和处理结果 |
| 异常提示 | 接口失败、库存不足、超期、冻结等异常使用醒目但克制的提示 |
| AI 助理 | 作为辅助入口，不替代主流程操作 |

## 九、交互边界

第一版为静态演示原型，允许：

- 菜单点击切换页面；
- mock 数据列表筛选的视觉反馈；
- 单据页打开/返回；
- 弹窗、抽屉、Tab、状态切换的静态演示；
- 不接真实接口，不保存真实数据。

不做：

- 真实登录；
- 真实权限计算；
- 真实数据库读写；
- 真实审批流流转；
- 真实 NC 推送；
- 真实 AI 调用。

---

## 十、验收标准

第一版原型完成后，应满足：

- 可以通过浏览器打开并演示；
- 菜单结构覆盖主要业务域；
- 代表性列表页和单据页可点击查看；
- 页面术语与详细设计保持一致；
- 状态、字段、按钮能支撑业务讨论；
- 不出现明显遮挡、错位、文字溢出；
- 可作为业务确认会和内部评审的演示材料。
